#### Abstract 原文

> On-policy distillation (OPD) has become a central post-training tool for large language models (LLMs), providing dense per-token teacher supervision along the student's own rollouts. In this work, we identify a common structural cause underlying OPD, which we call prefix failure. Under prefix failure, dense per-token supervision induces a bimodal teacher mixture and fragmented gradients that token-level loss truncation or reweighting fail to address. This observation motivates us to move beyond token-level loss interventions toward trajectory-level output corrections. We thus propose Trajectory-Refined Distillation (TRD), a trajectory-level correction method that revises the student's rollout under the teacher guidance while within on-policy support. By correcting problematic prefixes before distillation, TRD mitigates prefix failure at its source. Moreover, TRD improves the exploration by exposing the student to alternative valid derivations under teacher guidance, even when the original rolls are already correct. TRD can also be applied to on-policy self-distillation (OPSD), a parameter-sharing variant that uses the student model conditioned on privileged informations as the teacher. Across a wide range of benchmarks and base models at multiple scales, TRD consistently outperforms prior baselines, improving single-attempt accuracy and broadening reasoning coverage. Code is available at https://github.com/louieworth/trd

#### 摘要中文翻译

On-policy distillation（OPD，在线策略蒸馏）已经成为 large language models（LLMs，大语言模型）post-training（后训练）中的核心工具，它沿着学生模型自己的 rollouts（采样轨迹）提供 dense per-token teacher supervision（密集逐 token 教师监督）。本文识别出 OPD 背后一个常见的结构性原因，称为 prefix failure（前缀失败）。在 prefix failure 下，密集逐 token 监督会诱导出 bimodal teacher mixture（双峰教师混合分布）和 fragmented gradients（碎片化梯度），而 token-level loss truncation（逐 token 损失截断）或 reweighting（重加权）无法解决这一问题。基于这一观察，本文主张从 token-level loss interventions（逐 token 损失干预）转向 trajectory-level output corrections（轨迹级输出修正）。因此，作者提出 Trajectory-Refined Distillation（TRD），一种 trajectory-level correction method（轨迹级修正方法）：它在 teacher guidance（教师指导）下修订学生 rollout，同时保持在 on-policy support（在线策略支持）内。通过在蒸馏前修正有问题的 prefixes，TRD 从源头缓解 prefix failure。此外，即使原始 rollouts 已经正确，TRD 也能通过在教师指导下暴露 alternative valid derivations（替代的有效推导路径）来改进 exploration（探索）。TRD 也可以应用于 on-policy self-distillation（OPSD），即一种 parameter-sharing（参数共享）变体，其中教师是以 privileged informations（特权信息）条件化的学生模型本身。在多个尺度的 base models 和广泛 benchmarks 上，TRD 一致优于既有 baselines，提高了 single-attempt accuracy（单次尝试准确率）并拓宽了 reasoning coverage（推理覆盖范围）。代码位于 https://github.com/louieworth/trd。

#### 研究背景

这篇论文讨论的是 LLM 后训练中的 On-policy Distillation（OPD）和 On-policy Self-Distillation（OPSD）。OPD 的核心做法是：不是在固定专家数据上蒸馏，而是在学生模型自己生成的 trajectory 上，让教师模型对每个 visited prefix 给出 dense per-token supervision。OPSD 则进一步让教师和学生共享同一 backbone：学生只看到题目，教师分支额外看到 reference solution、ground-truth answer 或其他 privileged information。

作者指出，近期工业和研究系统中 OPD 已经和 SFT、RLVR 一起成为常见后训练阶段，例如 Qwen3、DeepSeek-v4、MiMo-v2、GLM-5 都包含类似 OPD 的阶段。但 vanilla OPD/OPSD 经常不稳定：teacher signal 可能变得 noisy、uninformative，甚至误导学生。已有工作多在 token loss 层面修补，例如 Forward KL clipping、Reverse KL Top-K truncation、token importance reweighting 等。

本文的核心诊断是 prefix failure（前缀失败）：当学生 rollout 进入错误推理路径后，几乎没有后续 continuation 能在不 backtrack 或 reflection 的情况下到达正确答案。此时 teacher 在当前错误 prefix 上会面对两种目标：一方面保持局部 sequence consistency，继续错误路径；另一方面尝试 pivot 到 correction path。因此 teacher distribution 变成双峰混合分布。

这种结构对 KL 方向有不同影响：

- Forward KL 是 mode-covering，容易被 correction-onset tokens（如 “Wait”, “Actually”）主导，可能把学生推向 out-of-distribution correction mode，造成不稳定。
- Reverse KL 是 mode-seeking，由学生高概率区域加权，因此更容易继续优化错误 continuation，而 correction signal 被低估。
- 更根本的问题是 dense per-token KL 总是在 frozen student rollout 上计算；即使 teacher 完美，它也只能在错误轨迹的每个 prefix 上重复提示 correction onset，而不能沿着真正 correction path 展开多步监督。

因此，论文认为问题不是单个 token loss 权重不合适，而是监督发生在错误的 trajectory 上。要修复 prefix failure，需要 trajectory-level correction，而不是只改 token-level loss。

#### Method

本文提出 Trajectory-Refined Distillation（TRD），其核心思想是在执行 KL 蒸馏之前，先让 teacher 对学生原始 rollout 进行 trajectory-level refinement，得到更正确、更短、更接近 reference 的 refined trajectory，然后沿 refined trajectory 进行标准蒸馏。

标准 OPD 的形式是：学生在题目 $x$ 上采样 $y \sim \pi_\theta(\cdot \mid x)$，教师在学生 prefix $y_{<t}$ 上提供分布，训练学生匹配教师：

$$
\mathcal{L}(\theta)
=
\mathbb{E}_{x \sim \mathcal{D},\, y \sim \pi_\theta(\cdot \mid x)}
\left[
\frac{1}{|y|}
\sum_{t=1}^{|y|}
D\left(
\pi_\theta(\cdot \mid x, y_{<t})
\Vert
\pi_T(\cdot \mid x, y_{<t})
\right)
\right].
$$

OPSD 中教师和学生共享参数，但教师分支额外看到 reference solution $y^\star$，并通过 stop-gradient 防止教师分支被直接更新：

$$
\pi_T(\cdot \mid x, y_{<t})
=
\mathrm{sg}\left[
\pi_\theta(\cdot \mid x, y^\star, y_{<t})
\right].
$$

论文首先形式化一个 trajectory construction surrogate：希望构造 verifier-pass rate 更高、同时仍处于当前学生 policy support 内的 trajectory distribution：

$$
\max_q
\mathbb{E}_{(x,y^\star)\sim \mathcal{D}}
\left[
\Pr_{y \sim q(\cdot \mid x,\cdot)}
\left\{
\mathrm{Verify}(y,y^\star)=1
\right\}
\right]
\quad
\text{s.t.}
\quad
\pi_\theta(y \mid x) > 0.
$$

这个目标不是直接用 RLVR 优化，而是作为蒸馏前的 trajectory construction 目标。TRD 的实际算法很简单：

1. 先从学生模型采样原始 rollout：$y_o \sim \pi_\theta(\cdot \mid x)$。
2. 再让教师基于题目和原始 rollout 生成 refined trajectory：$y_r \sim \pi_T(\cdot \mid x, y_o)$。
3. 在 OPSD 中，refinement 使用同一 backbone，但额外条件化 reference solution：$y_r \sim \mathrm{sg}[\pi_\theta(\cdot \mid x, y_o, y^\star)]$。
4. 最后在 $y_r$ 上做 full-vocabulary Forward KL distillation。

TRD 的训练损失为：

$$
\mathcal{L}(\theta)
=
\mathbb{E}_{x, y_o, y_r}
\left[
\frac{1}{|y_r|}
\sum_{t=1}^{|y_r|}
D\left(
\mathrm{sg}\left[
\pi_T(\cdot \mid x, y_o, y_{r,<t})
\right]
\Vert
\pi_\theta(\cdot \mid x, y_{r,<t})
\right)
\right].
$$

这样做有三个关键作用：

- 缓解 prefix failure：错误 prefix 在蒸馏前被改写，teacher 不再被迫在错误轨迹上给出碎片化 correction signal。
- 恢复 ideal supervision pair：监督上下文沿 $y_r$ 展开，而不是沿原始错误 rollout $y_o$ 展开。
- 提升 exploration：即使 $y_o$ 已经正确，teacher 也可能给出更短或不同的 valid derivation，让学生接触更多正确推理路径。

论文还通过 diagnostic metrics 验证 prefix failure：在标准 OPSD 的 $y_o$ 上，correct 和 incorrect rollouts 的 per-token KL 都接近零，teacher-student PPL gap 也接近零，说明 privileged teacher 没有提供增量监督；同时 teacher 对 epistemic onset tokens 的概率质量约为 $6$ 到 $8$‰，符合反复建议 correction onset 的 fragmented signal。TRD 在 $y_r$ 上显著降低这种 epistemic-token mass，并重新打开 PPL gap。

#### 论文自我评价的缺陷

论文明确列出两个 limitations。

第一，TRD 需要额外的 sampling budget 来构造 $y_r$。与 dense-KL baselines 相比，TRD 多了一个 Stage-2 refinement generation，因此推理生成成本更高。作者认为这个开销会被较短 refined trajectories 带来的更快 KL training 部分抵消；在 Qwen3-8B OPSD 上，TRD 总 wall-clock 为 $9{:}20$，vanilla OPSD 为 $9{:}40$，几乎持平。但在较小模型和 OPD 设置下，TRD 仍明显更慢，例如 Qwen3-4B OPD 从 $5{:}20$ 增至 $9{:}00$。

第二，TRD 依赖 teacher 的 refinement 能力。教师必须能够在保持接近学生 on-policy distribution 的同时修正 prefix failure。如果教师不能可靠识别和改写错误推理，或者 refinement 偏离学生 support，TRD 的效果就会下降。论文在 LiveCodeBench 上的结果已经体现了这一点：TRD 在该 harder code benchmark 上没有超过 Base，作者推测当前 teacher 对这些代码任务的 refinement 不够有效。

此外，论文虽然没有在 limitations 中展开，但从实验和方法可见还有一些隐含限制：TRD 需要 reference solution 或强 teacher；OPSD 中 privileged information 的质量会直接影响 refinement；数学任务主要依赖 rule-based boxed-answer verifier，结果可能更适合可验证问题；代码 OPSD 没有作为主表系统报告；实验集中在 Qwen3 family，跨模型家族泛化仍需更多验证。

#### 我的分析：优点

这篇论文的最大优点是把 OPD/OPSD 的失败从 token-level symptom 提升到了 trajectory-level structure。prefix failure 的解释很有说服力：如果学生已经走入错误路径，teacher 在错误 prefix 上的每个 token distribution 都会被局部一致性和全局纠错目标拉扯，导致双峰分布和碎片化梯度。这个分析能够统一解释为什么 Forward KL clipping、Reverse KL Top-K、token reweighting 等看似相反的方法都只能缓解局部问题，却不能根治错误 prefix。

第二，TRD 方法非常简单。它没有引入复杂的 RL objective，也没有改变蒸馏训练主体，只是在蒸馏前增加 trajectory refinement，然后继续用 full-vocabulary Forward KL。这个设计易于集成到现有 OPD/OPSD pipeline 中，也符合工程上的可操作性。

第三，论文的 empirical story 比较完整。它不只给主结果，还提供了机制诊断：per-token KL、PPL gap、epistemic-token mass 都支持 prefix failure；training trajectory analysis 显示 $y_r$ 更短且 verifier-pass rate 更高；AMOBench difficulty bucket analysis 显示 TRD 主要提升 base-unreachable questions 的 Pass@16，这比单纯报告平均分更能说明“coverage expansion”。

第四，TRD 对 OPSD 的适配很自然。用同一个 student backbone 加 privileged reference 作为 teacher，可以减少外部 teacher 成本和 model mismatch，同时实验显示 OPSD TRD 在 Pass@16 上常常强于 OPD TRD。这对自蒸馏、reference-guided post-training、on-policy self-improvement 都有直接启发。

第五，实验结果中 hard benchmark 的提升尤其有意义。AMOBench 上 Qwen3-8B OPSD 的 Pass@16 从 Base 的 $41.0$ 到 TRD 的 $61.5$，比最强 dense-KL baseline 的 $51.3$ 还高很多，说明 trajectory-level refinement 不只是稳定训练，而是真的拓展了模型在采样预算下可达到的解题覆盖面。

#### 我的分析：缺点 / 风险

第一，TRD 的核心收益依赖 teacher refinement 的质量，但论文对 refinement failure 的刻画还不够充分。比如 refined trajectory 可能引入新的错误、过度依赖 reference solution、或者生成与学生原始 policy support 实际不匹配的推理风格。论文用 $y_o$ anchoring 来论证 $y_r$ 仍在 support 附近，但这更多是直觉和经验结果，并不是严格保证。

第二，OPSD 设置中 reference solution 的使用可能使问题变得更接近 “rewrite with answer guidance”。这在训练时合理，但如果 reference 质量参差、只有短答案、或者包含错误 reasoning，TRD 可能会学习到偏差。附录显示 without-CoT reference 也有效，但对低质量 reference、noisy verifier、ambiguous answer 的鲁棒性仍不清楚。

第三，实验主要集中在 competition math，代码结果相对弱。OPD 代码 benchmark 中，HumanEval+ 和 MBPP+ 有小幅提升，但 LiveCodeBench 上 TRD 没有超过 Base。这说明 trajectory refinement 对长程算法代码、隐藏测试、复杂边界条件的帮助可能有限，或者需要更强 code teacher / executor-in-the-loop refinement。

第四，TRD 增加了 pipeline 复杂度和存储/生成成本。虽然 Qwen3-8B OPSD 中 wall-clock 几乎持平，但 OPD 小模型设置下开销明显上升。实际大规模训练中，额外生成 $y_r$、保存 refined trajectories、管理 prompt budgets 和长上下文 teacher inference 都可能成为工程瓶颈。

第五，论文对比的是几类 dense-KL baselines，但没有充分比较更强的 trajectory-level alternatives，例如 best-of-$n$ verified sampling、rejection sampling with verifier、iterative correction、process-level feedback、或者 RLVR + distillation hybrids。TRD 与这些方法的成本-收益边界还需要进一步实验。

第六，prefix failure 的理论分析很有启发，但目前仍偏定性。比如 “teacher distribution becomes bimodal mixture” 和 “fragmented gradient” 是合理抽象，但真实 LLM 分布是否总是按这种模式失败、不同任务中 prefix failure 占失败比例多少、如何自动检测 prefix failure，这些问题还没有完全量化。

第七，TRD 可能带来探索与分布收缩之间的长期风险。虽然论文发现 $y_r$ 更短、更正确，并能提供 alternative derivations，但如果 teacher refinement 总是偏向某些简洁模板，长期训练可能压缩 reasoning diversity，导致模型在开放式任务上过拟合 teacher style。当前实验的 Pass@k 提升说明短期是正向的，但多轮 TRD 或持续自蒸馏下是否会出现 mode collapse，还需要进一步验证。

#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy distillation。

#### Abstract 原文

> On-policy distillation (OPD) is increasingly used to improve large language model reasoning, but its training dynamics remain poorly understood. We characterize the trajectory of OPD updates in parameter space and compare it with supervised fine-tuning (SFT) and reinforcement learning with verifiable rewards (RLVR). A suite of parameter-space diagnostics consistently places OPD in a relaxed off-principal regime: compared with SFT, its updates affect fewer weights and avoid principal directions more strongly, while compared with RLVR, they remain less tightly constrained. Beyond this static localization, OPD exhibits subspace locking: its cumulative updates rapidly enter a narrow low-dimensional channel. Constraining training to the update subspace formed early in training preserves OPD performance but substantially degrades SFT, indicating that the locked subspace is functionally sufficient for OPD. Control experiments further show that sparsifying the update tokens and shifting rollout generation off-policy preserve the rank dynamics, whereas mixing the OPD objective with RLVR changes them. Overall, these results suggest that OPD is not merely an intermediate point between SFT and RLVR, but induces its own update geometry in parameter space.

#### 摘要中文翻译

On-policy distillation (OPD，在线策略蒸馏) 正越来越多地用于提升 large language model reasoning（大语言模型推理能力），但其训练动态仍缺乏充分理解。本文刻画 OPD 更新在 parameter space（参数空间）中的轨迹，并将其与 supervised fine-tuning (SFT，监督微调) 和 reinforcement learning with verifiable rewards (RLVR，带可验证奖励的强化学习) 进行比较。一组 parameter-space diagnostics（参数空间诊断）一致表明，OPD 位于一种 relaxed off-principal regime（宽松的非主方向区域）：与 SFT 相比，它的更新影响更少权重，并且更强地避开 principal directions（主方向）；与 RLVR 相比，它又没有被约束得那么紧。除这种静态定位之外，OPD 还表现出 subspace locking（子空间锁定）：其累积更新会迅速进入一个狭窄的 low-dimensional channel（低维通道）。将训练限制在早期训练形成的 update subspace（更新子空间）中，仍能保持 OPD 性能，但会显著削弱 SFT，这说明该锁定子空间对 OPD 来说在功能上已经足够。进一步的控制实验显示，稀疏化 update tokens（更新 token）以及将 rollout generation（轨迹生成）切换为 off-policy（离策略）仍会保持 rank dynamics（秩动态），而将 OPD objective（目标函数）与 RLVR 混合则会改变这些动态。总体而言，这些结果表明，OPD 并不只是 SFT 与 RLVR 之间的中间点，而是在参数空间中诱导出自身独特的 update geometry（更新几何）。

#### 研究背景

大语言模型的 reasoning（推理）能力主要依赖 post-training（后训练）提升。已有主流范式包括：SFT，即在 offline demonstrations（离线示范）上用 cross-entropy objective（交叉熵目标）训练；RLVR，即用 sparse outcome rewards（稀疏结果奖励）优化策略；以及 OPD，即让 student model（学生模型）在自己生成的 trajectories（轨迹）上接受 stronger teacher（更强教师）的 dense token-level guidance（密集 token 级指导）。

已有研究表明，SFT 和 RLVR 在参数空间中留下截然不同的几何足迹：SFT 往往产生 dense, principal-aligned updates（稠密且沿主方向的更新），会更明显改变 pretrained spectral structure（预训练谱结构）；RLVR 则倾向于 sparse, off-principal updates（稀疏且避开主方向的更新），更能保持原模型的谱结构。OPD 同时具有两类特征：它有类似 SFT 的 dense token-level distillation signal（密集 token 级蒸馏信号），又使用 on-policy sampling（在线策略采样）和 policy-gradient style optimization（类策略梯度优化），因此不能简单从 SFT 或 RLVR 推断其参数轨迹。

论文围绕三个问题展开：OPD 在 SFT-RLVR 参数空间谱系中位于哪里；OPD 训练过程中是否有自身内在的 update trajectory（更新轨迹）；以及 OPD 的哪些组成部分控制这种轨迹。核心结论是：OPD 位于 relaxed off-principal regime，并且在训练早期进入一个稳定、低维、功能上足够的 locked subspace（锁定子空间）；这种锁定主要受 objective composition（目标组成）控制，而不是简单由 token 覆盖率或 rollout policy 决定。

#### Method

论文以 Qwen3-family checkpoints 为主要对象，比较 SFT、OPD 和 RLVR 的参数空间几何。设初始化权重为 `$W_0$`，训练后权重为 `$W_+$`，更新为 `$\Delta W = W_+ - W_0$`。对 OPD 和 RLVR，`$W_0$` 是同一个 Qwen3-8B SFT anchor；对 SFT，`$W_0$` 是 Qwen3-8B-Base。

参数空间诊断包括四类。第一是 bf16-aware update sparsity（bf16 感知更新稀疏性）：若标量权重满足 `$|\hat{w}_i-w_i| \le \eta \max(|w_i|,|\hat{w}_i|)$` 且 `$\eta=10^{-3}$`，则视为未变化；整体稀疏性可写为 `$sp(W_0,W_+) = 1-\frac{1}{n}\sum_{i,j}\mathbf{1}[W_{+,ij}\not\approx_\eta W_{0,ij}]$`。第二是 principal-angle rotation（主角旋转），比较 `$W_0$` 与 `$W_+$` 的 top-`$k$` singular subspaces（奇异子空间），例如 `$ \cos\theta_i(U)=\sigma_i(U_{0,k}^{\top}U_{+,k}) $`，角度越小表示越保持预训练主子空间。第三是 spectral drift（谱漂移），用 normalized spectral shift 表示：`$\mathrm{NSS}(W_0,W_+) = \frac{\|\sigma(W_+)-\sigma(W_0)\|_2}{\|\sigma(W_0)\|_2}$`。第四是 update-mask overlap（更新掩码重叠），比较 bf16-visible update mask（bf16 可见更新掩码）与 principal mask（主方向掩码）和 low-magnitude mask（低幅值掩码）的重叠，判断更新是否避开 principal weights 并集中到低幅值区域。

为解释 OPD 的位置，论文提出 relaxed Three-Gate view（三门控宽松解释）。通用 post-training update 可写成 `$g=\sum_{t=1}^{T}a_t\phi_t$`，其中 `$\phi_t=\nabla_\theta \log p_\theta(y_t\mid x,y_{<t})$` 是 token score。RLVR 中 `$a_t=A(y)$` 是 sequence-level advantage（序列级优势）并在 token 间共享；OPD 中 `$a_t$` 随 token 改变，由 teacher-student disagreement（教师-学生分歧）决定。因此 OPD 保留 on-policy 结构，但把稀疏序列级信号替换为密集 token 级监督。三道门分别是 distributional anchor（分布锚定）、pretrained model geometry（预训练模型几何）和 bf16 precision realization（bf16 精度实现）。OPD 仍受锚定和几何约束，但 dense teacher supervision 会放宽可激活方向，使其比 RLVR 更新更多坐标，又比 SFT 更保守。

为研究训练轨迹，论文跟踪 checkpoint 上的 cumulative update `$\Delta W_t=W_t-W_0$`，并用 stable rank（稳定秩）衡量有效维度：`$\operatorname{srank}(\Delta W_t)=\frac{\|\Delta W_t\|_F^2}{\|\Delta W_t\|_{\mathrm{op}}^2}$`。同时用 Frobenius norm（弗罗贝尼乌斯范数）排除“只是更新很小”的解释，并用 Hill tail estimate（Hill 尾部估计）检查 singular-value profile（奇异值谱形状）。为了验证是否真的锁定到早期子空间，论文计算 early checkpoint 与最终 top-`$K$` right singular subspace 的相似度：`$\mathrm{Sim}_K(t,t_{\mathrm{end}})=\frac{1}{K}\|V_K(t)^\top V_K(t_{\mathrm{end}})\|_F^2$`。进一步，论文把后续梯度限制在早期 rank-16 子空间中，即 `$g \leftarrow gV_{16}V_{16}^{\top}$`，检验该子空间是否对训练功能上足够。

控制实验分别扰动 token supervision density（token 监督密度）、rollout policy（轨迹策略）和 objective composition（目标组成）。token sparsification 保留 top-KL 或随机 token 的 25%/50%；off-policy rollout 使用 teacher 生成样本但仍用 OPD objective；objective mixing 则在 advantage 层混合 OPD 与 RLVR：`$A_i^{(\alpha)}=\alpha A_{i,\mathrm{OPD}}+(1-\alpha)A_{i,\mathrm{RLVR}}$`。这样可以区分 runtime perturbations（运行时扰动）和真正改变 gradient source（梯度来源）的 objective perturbation（目标扰动）。

#### 论文自我评价的缺陷

论文明确列出 Limitations。第一，分析集中在 controlled Qwen3-family reasoning settings。虽然这种设计有助于隔离训练范式差异，但所得 update geometry 可能会随 model family（模型家族）、modality（模态）和 task distribution（任务分布）变化，不能直接保证推广到其他架构、其他多模态任务或更广泛的 instruction-following 场景。

第二，论文的 diagnostics 来自 stored checkpoints（保存的检查点）上的离线参数分析。Three-Gate account 和 covariance analysis 是与证据一致的 mechanistic explanations（机制性解释），但论文承认它们还不是完整的 causal theory（因果理论）或 formal theory（形式化理论）。也就是说，当前结果说明 OPD 的参数轨迹表现出稳定几何模式，但尚未严格证明这些几何门控如何在每一步优化中因果地产生 subspace locking。

#### 我的分析：优点

- 论文把 OPD 从“效果好不好”的行为层面推进到 parameter-space geometry（参数空间几何）层面，问题定义清晰：OPD 究竟像 SFT、像 RLVR，还是有自己的几何轨迹。
- 实验设计较干净：OPD 和 RLVR 使用同一个 SFT anchor 和 math-domain prompt distribution，减少了初始化与数据分布差异带来的混淆。
- 诊断指标覆盖较全面，包括 sparsity、principal-angle rotation、spectral drift、update-mask localization、stable rank、Frobenius norm、Hill estimate 和 subspace similarity，不是只依赖单一谱指标。
- “subspace locking” 的证据链较强：不仅观察 endpoint geometry，还跟踪 trajectory；不仅看 stable rank，还做 early subspace similarity 和 rank-16 projected training，证明该子空间对 OPD 有功能意义。
- 控制实验区分了 token density、rollout policy 和 objective composition，结论对 OPD 算法设计有启发：关键可能不是盲目增加 token-level supervision，而是控制 objective-induced update geometry。
- 对 OPD 与 RLVR/SFT 的关系解释比较细腻：OPD 不是简单中点，而是“geometry-steered like RLVR, relaxed by dense teacher supervision”，这比线性插值式理解更有信息量。

#### 我的分析：缺点 / 风险

- 泛化性仍是最大风险。实验主要在 Qwen3-family 和 reasoning/math settings 上完成，不能确认其他模型家族、代码任务、通用指令任务、多模态模型或更长训练周期也会出现相同 subspace locking。
- 很多结论依赖具体诊断选择，例如 bf16 threshold `$\eta=10^{-3}$`、分析哪些 weight matrices、stable rank 的平均方式、top-`$K$` 子空间里的 `$K=16$` 选择，以及 checkpoint 保存频率；这些超参数可能影响几何判断。
- Three-Gate 和 covariance 解释仍偏 heuristic（启发式）。objective mixing 改变了 gradient source，但也可能同时改变信号尺度、方差、reward normalization 行为和优化稳定性；因此“objective composition 控制锁定”还需要更强的因果干预验证。
- rank-16 projected training 证明了早期子空间对 OPD 足够，但不等于说明这是最优子空间，也不说明监控或约束该子空间一定能提升最终性能；论文更多证明“能保持”，较少证明“能改进”。
- 论文重点是参数几何而不是最终能力、安全性或泛化误差。OPD 的 geometry-preserving 更新可能有利于稳定，但也可能保留 pretrained model 的错误偏见；low-dimensional lock 也可能限制模型适应新分布。
- teacher supervision 本身存在风险：如果 teacher 的 token-level guidance 有系统性错误，OPD 可能在一个低维通道内稳定地复制这些错误；论文没有深入分析 unreliable teacher supervision、teacher calibration 或 token-level filtering 对几何轨迹的影响。
