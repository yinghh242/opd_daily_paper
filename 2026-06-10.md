#### Abstract 原文

> We introduce Kwai Keye-VL-2.0-30B-A3B, an open-source Mixture-of-Experts (MoE) multimodal foundation model designed to advance long-video understanding and agentic intelligence. To address the challenges of ultra-long contexts, information redundancy, and prohibitive computational costs inherent in hour-level videos, Keye-VL-2.0 is the first to adapt DeepSeek Sparse Attention (DSA) to GQA-based multimodal architectures, enabling lossless 256K context processing while capturing critical frames and long-range temporal dependencies. This architecture is underpinned by a highly optimized training and inference infrastructure, including scalable video I/O, heterogeneous ViT-LM parallelism, and custom DSA kernels that significantly maximize throughput and minimize computational overhead. Furthermore, to overcome the algorithmic dilemma of catastrophic forgetting during multi-task alignment, we introduce Cross-Modal Multi-Teacher On-Policy Distillation (MOPD) paired with Context-RL and Video-RL. By distilling dense token-level teacher feedback from on-policy rollouts back into the MoE backbone, which activates only 3B parameters, Keye-VL-2.0 natively empowers advanced agent collaboration across Code, Tool, and Search scenarios with multimodal self-correction. Extensive evaluations across video understanding, temporal grounding, reasoning, STEM, and agent benchmarks demonstrate that Keye-VL-2.0-30B-A3B achieves state-of-the-art performance among models of similar scale, particularly excelling in fine-grained temporal localization on TimeLens and long-video comprehension on Video-MME-v2 and LongVideoBench. We release our model checkpoints to accelerate community progress toward scalable and robust multimodal agentic applications.

#### 摘要中文翻译

本文介绍 Kwai Keye-VL-2.0-30B-A3B，这是一个开源的 Mixture-of-Experts (MoE，混合专家) 多模态基础模型，旨在推进长视频理解和 agentic intelligence（智能体能力）。针对小时级视频中固有的超长上下文、信息冗余和高昂计算成本问题，Keye-VL-2.0 首次将 DeepSeek Sparse Attention (DSA，DeepSeek 稀疏注意力) 适配到基于 GQA 的多模态架构中，使模型能够无损处理 256K 上下文，同时捕获关键帧和长程时间依赖。该架构由高度优化的训练与推理基础设施支撑，包括可扩展的视频 I/O、异构 ViT-LM 并行，以及定制 DSA kernels，用于显著提升吞吐并降低计算开销。

此外，为解决多任务对齐中的 catastrophic forgetting（灾难性遗忘）算法困境，本文提出 Cross-Modal Multi-Teacher On-Policy Distillation (MOPD，跨模态多教师在线策略蒸馏)，并结合 Context-RL 和 Video-RL。通过将来自 on-policy rollouts（在线策略采样轨迹）的 dense token-level teacher feedback（密集 token 级教师反馈）蒸馏回 MoE backbone，而该模型每次仅激活 3B 参数，Keye-VL-2.0 原生支持 Code、Tool 和 Search 场景下的高级 agent collaboration（智能体协作）和多模态 self-correction（自我纠错）。

在视频理解、temporal grounding（时间定位）、推理、STEM 和 agent benchmarks 上的大量评测表明，Keye-VL-2.0-30B-A3B 在同规模模型中达到 state-of-the-art 表现，尤其在 TimeLens 的细粒度时间定位，以及 Video-MME-v2 和 LongVideoBench 的长视频理解上表现突出。作者发布模型 checkpoints，以推动社区向可扩展、鲁棒的多模态智能体应用发展。

#### 研究背景

这篇技术报告的核心问题是：如何把多模态大模型从“短视频/图像理解”推进到“小时级长视频理解 + 多步 agentic task execution”，同时避免成本爆炸和能力互相干扰。

论文认为当前 MLLM 面临两类关键瓶颈：

1. 长上下文视频的基础设施瓶颈
   标准 dense attention（稠密注意力）在序列长度上具有二次复杂度，面对 128K/256K 级别多模态上下文时，KV cache 和计算成本快速膨胀。传统做法往往依赖 aggressive frame subsampling（激进抽帧），但这会损失 temporal continuity（时间连续性）和关键事件证据。

2. 多任务 post-training 的算法冲突
   直接混合视频理解、OCR、STEM、代码、工具调用、搜索、长上下文等任务，容易造成 modality conflict（模态冲突）和 catastrophic forgetting。例如，强化 agent 工具调用能力可能让模型过度输出工具格式；强化 reasoning RL 又可能让模型在简单感知任务中过度冗长。

Keye-VL-2.0 的定位是一个开源 30B-class MoE 多模态基础模型，总参数约 30B、每次仅激活 3B 参数。它继承 Keye-VL 和 Keye-VL-1.5 的视觉语言基础，同时在两个方向上扩展：

- 架构上：将 DeepSeek Sparse Attention (DSA) 引入 GQA-based multimodal backbone，实现 256K multimodal context。
- 训练上：用 staged pre-training、RL、specialized domain experts 和 Cross-Modal MOPD，把不同能力整合回单一 MoE 模型，尽量避免互相遗忘。

#### Method

Keye-VL-2.0-30B-A3B 的方法可以分为 architecture、pre-training、post-training 和 training/inference infrastructure 四个层面。

1. Model Architecture

模型由四个核心组件组成：

- Vision Encoder (ViT)：继承自 Keye-VL-1.5-8B，基于 SigLIP-400M-384-14，用于图像和视频帧特征提取。
- Language Decoder (LLM)：基于 Qwen3-30B-A3B-Thinking-2507，提供语言、知识、指令跟随和推理能力。
- MLP Projector：在 Stage 0 初始化，用于把视觉特征映射到 LLM 表征空间。
- Sparse Attention Module：面向 GQA 的 DSA 设计，用 global MQA-style indexing 加 grouped GQA aggregation 支持长上下文。

视觉侧采用 native-resolution vision encoder（原生分辨率视觉编码器）。相比固定分辨率或 dynamic tiling（动态切块），该设计保留原图长宽比和细粒度局部信息，对 OCR、文档、图表、视频和细粒度识别尤其重要。模型还结合 adaptive position encoding、2D RoPE、NaViT-style Patch n' Pack 和 FlashAttention，以支持不同尺寸样本的高效 packing。

视频和图像采用 unified visual encoding（统一视觉编码）：视频每帧被当作一张高分辨率图像处理，并在每帧视觉 tokens 前加入自然语言 timestamp，使 LLM 在文本空间中获得显式时间锚点。视频还使用 adaptive video pixel budget，根据视频时长分配像素预算：短视频更强压缩，长视频保留更多视觉证据。

2. DSA for Long-Context Multimodal Modeling

论文的关键架构创新是把 DeepSeek Sparse Attention 适配到 GQA-based MLLM 中。传统 full attention 复杂度为 $O(L^2)$，而 Keye-VL-2.0 设置 top-k 为 $k = 2048$，将核心注意力复杂度降为 $O(Lk)$，其中 $k \\ll L$。

DSA 包含两个主要部分：

- MQA-style Lightning Indexer：用共享 key head 计算当前 query token 与历史 tokens 的全局索引分数 $I_{t,s}$，选择 Top-k tokens 形成稀疏集合 $\\Omega_t$。
- GQA Sparse Aggregation：GQA 中多个 query heads 分成 $G$ 个 group，每组共享 group-specific KV head，并使用相同的 $\\Omega_t$ 做稀疏聚合。

核心公式包括全局索引分数：

$$
I_{t,s} = \\sum_{j=1}^{H_I} w^I_{t,j} \\cdot \\mathrm{ReLU}(q^I_{t,j} \\cdot k^I_s)
$$

Top-k 稀疏索引集合：

$$
\\Omega_t = \\{s \\mid I_{t,s} \\in \\mathrm{Top\\text{-}k}(I_{t,:})\\}
$$

第 $g$ 个 GQA group 的 sparse attention 输出：

$$
u_{t,g} = \\mathrm{Attn}(h_{t,g}, \\{c_{s,g} \\mid s \\in \\Omega_t\\})
$$

DSA 训练采用 dense warm-up 和 sparse adaptation 两阶段：

- dense warm-up：主模型保持 dense GQA，大部分参数冻结，让 indexer 学习覆盖原始 dense attention 分布。
- sparse adaptation：全参数解冻并切换到 sparse mode，indexer 继续与主模型注意力分布对齐，但 KL loss 只在 Top-k 集合上计算。

最终目标为：

$$
\\mathcal{L}_{\\mathrm{total}} = \\mathcal{L}_{\\mathrm{NTP}} + \\lambda \\mathcal{L}^{I}_{\\mathrm{sparse}}
$$

3. Pre-Training

预训练采用四阶段 curriculum：

- Stage 0，Projector Initialization：冻结 ViT 和 LLM，仅训练 Projector，使用 caption 和 interleaved image-text 数据，序列长度 8K，约 4B tokens。
- Stage 1，General Multimodal Pre-training：全参数训练，序列长度 32K，约 1T tokens，覆盖 caption、OCR、video mix、image mix、pure-text QA。
- Stage 2，Multi-Task Capability Injection：序列长度扩展到 64K，约 2T tokens，注入 OCR、STEM、GUI、grounding、counting、e-commerce、coding、tool-use/search 等能力。
- Stage 3，Long-Context Extension：序列长度扩展到 256K，约 500B tokens，覆盖 long video、long documents、multi-doc、long multi-image、long code、long agent trajectories。

视频预训练也分阶段扩展：Stage 1 使用 15 秒 clips，Stage 2 扩展到 15 分钟，Stage 3 扩展到 2 小时。论文特别强调 scene-wise dense caption（按场景密集字幕）和 diverse temporal video grounding data，包括 Referred Action Recognition、Video Highlight Detection、Extractive Video Summarization、Temporal Event Matching 等。

4. Post-Training

post-training 包含 SFT、Synthetic CoT、RL、specialized RL、Video RL、Agentic RL 和 MOPD。

SFT 数据约 500B tokens，包含 Text NLP、Video、Perception、Reasoning、Agent、Long-context，其中约 40% 是 text-only data，用于保持语言和推理能力。Synthetic CoT 用强教师模型生成 reasoning traces，再经 query-level、response-level、process-level checks 过滤；数学任务还使用 Doubt2Clean 二次清洗。

RL 主要分为：

- Synthetic-Data RL：构造可程序验证的 difference-recognition tasks，例如图像差异定位、几何、化学式、电路图 DSL 输出。奖励可由规则自动计算。
- General RL：使用 GSPO，在 VQA、STEM、chart、math、logic、text-only QA 等可验证任务上提升可靠性。
- Specialized RL：训练 grounding、spatial、math、counting、OCR 等 domain experts。
- Video RL：约 31K video samples，冻结 ViT 和 projector，用 GSPO 优化 temporal alignment、event tracking 和 long-range aggregation。
- Agentic RL：覆盖 Coding RL、Tool Use RL、Search RL，用环境执行结果、工具状态、测试结果或最终答案正确性作为 reward。

GSPO 的 sequence-level ratio 定义为：

$$
s_i(\\theta) = \\exp \\left(
\\frac{1}{|y_i|} \\sum_{t=1}^{|y_i|}
\\log \\frac{\\pi_\\theta(y_{i,t} \\mid x, y_{i,<t})}
{\\pi_{\\theta_{\\mathrm{old}}}(y_{i,t} \\mid x, y_{i,<t})}
\\right)
$$

advantage 标准化为：

$$
\\hat{A}_i =
\\frac{
R(x, y_i) - \\mathrm{mean}(\\{R(x, y_j)\\}_{j=1}^{G})
}{
\\mathrm{std}(\\{R(x, y_j)\\}_{j=1}^{G}) + \\delta
}
$$

5. Cross-Modal Multi-Teacher On-Policy Distillation

MOPD 是全文最重要的 post-training consolidation 方法。论文维护 13 个 RL-trained domain teachers，覆盖 safety、pure-text math、instruction following、code、visual STEM、OCR、grounding、counting、video、tool use 等。每个样本根据 modality 和 task type 路由到最合适的 teacher。

学生先生成 on-policy response：

$$
y_i = (y_{i,1}, \\ldots, y_{i,T}) \\sim \\pi_\\theta(\\cdot \\mid x_i)
$$

教师在学生真实访问过的 state 上提供 token-level feedback。为避免 full-vocabulary distillation 的不稳定性，论文定义 teacher 和 student Top-k 的 overlap set：

$$
\\Omega_{i,t} =
\\mathrm{TopK}(\\pi_T^{r(i)}(\\cdot \\mid s_{i,t}))
\\cap
\\mathrm{TopK}(\\pi_\\theta(\\cdot \\mid s_{i,t}))
$$

当 $\\Omega_{i,t}$ 非空时，raw advantage 为：

$$
A_{i,t} =
\\sum_{v \\in \\Omega_{i,t}}
\\bar{\\pi}_\\theta(v \\mid s_{i,t})
\\left[
\\log \\pi_T^{r(i)}(v \\mid s_{i,t})
-
\\log \\pi_\\theta(v \\mid s_{i,t})
\\right]
$$

其中：

$$
\\bar{\\pi}_\\theta(v \\mid s_{i,t}) =
\\frac{
\\pi_\\theta(v \\mid s_{i,t})
}{
\\sum_{u \\in \\Omega_{i,t}} \\pi_\\theta(u \\mid s_{i,t})
}
$$

MOPD objective 为：

$$
\\mathcal{L}_{\\mathrm{MOPD}} =
-
\\mathbb{E}
\\left[
\\frac{1}{|M_i|}
\\sum_{t \\in M_i}
\\hat{A}_{i,t}
\\log \\pi_\\theta(y_{i,t} \\mid x_i, y_{i,<t})
\\right]
$$

此外，论文使用 Segmented Prompt-Response Re-tokenization (SPRR) 保证 teacher/student token 对齐，并使用 token-category-aware advantage scaling 降低格式 token 权重、提高感知或推理 token 权重。对 repetition collapse，则只在崩溃点之后施加 localized repetition penalty：

$$
\\hat{A}_{i,t} = \\tilde{A}_{i,t} - \\lambda_{\\mathrm{rep}} \\cdot \\mathbf{1}[t \\ge \\tau_i]
$$

6. Efficient Training and Inference Infrastructure

系统侧贡献包括：

- ExtraIO：异步视频 I/O 服务，解除视频解码和抽帧瓶颈。
- ViT-LM heterogeneous parallelism：ViT 和 LM 共置但采用不同并行切分策略，并通过 recompute/offload 降低 ViT activation memory。
- two-level load balancing：在 multimodal-token level 和 LM-sample level 同时平衡负载，端到端训练吞吐提升约 20%。
- DSA variable-length optimization：用 FlashInfer 和 TileLang 优化 DSA，相比开源 baseline 超过 $2\\times$ speedup。
- inference optimization：Chunk ViT、Top-k set deduplication、DSA decode optimization，使 128K context 下 prefill cost 约为 full attention 的 $0.32\\times$，decode cost 约为 $0.20\\times$。

| DSA inference at 128K context | Prefill cost | full attention 的 $0.32\\times$ | H800 GPU，按每 GPU-hour 2 美元估算 |
| DSA inference at 128K context | Decode cost | full attention 的 $0.20\\times$ | 论文称 DSA-specific decode optimization 使 prefill 降低超过 $3\\times$、decode 降低超过 $5\\times$ |

#### 论文自我评价的缺陷

论文未单列 limitations，但从 conclusion and future work 以及实验呈现方式可以看出若干自我承认或隐含限制：

1. 当前仍偏 leaderboard-oriented optimization
   作者在 future work 中明确说，后续会“move beyond leaderboard-oriented optimization toward deeper deployment in real business scenarios”。这说明当前结果主要还是围绕 benchmark 证明能力，真实业务闭环、线上稳定性、长周期反馈和用户体验还不是本文充分验证的对象。

2. 真实业务落地仍是未来工作
   论文计划把 fine-grained long-video perception 和 image-text understanding 融入推荐、内容生态治理、商业投放等产品 pipeline。这意味着模型虽然在 benchmark 上表现强，但在真实业务中的收益、延迟、成本、鲁棒性和风险控制尚未被系统性报告。

3. Video-Agent workflow 仍未完成
   论文把 “Video-Agent workflows” 放在 future work：从被动理解视频，发展到自动化协调视频检索、highlight extraction、automated editing、packaging、marketing-copy generation。这说明当前模型已经展示 agentic capability，但完整视频生产链路中的自主编排、错误恢复和多工具协作仍处于规划阶段。

4. native multimodal modeling 和 deeper end-to-end fusion 仍是未来方向
   作者提到将继续从 DSA compute optimization、data flywheels、Context-RL post-training 走向 native multimodal modeling 和 deeper end-to-end fusion。换言之，当前方案仍较依赖“视频帧作为图像 + timestamp 文本注入 + LLM 聚合”的统一管线，而不是完全端到端的视频原生建模。

5. 若干关键组件缺少独立消融
   论文报告了 DSA、Video RL、MOPD、数据课程、系统优化等综合收益，但没有充分给出每个组件对最终 benchmark 的逐项 ablation。例如，MOPD 相比简单 multi-task SFT/RL mixing 的定量收益、13 个 teacher 的贡献拆分、top-k overlap estimator 的替代实验等没有详细表格。

6. 评测公平性仍有复杂因素
   TimeLens 中不同模型存在 2 FPS 和 4 FPS 设置差异，Video-MME-v2 中 Gemini-3-Flash 也采用不同采样设置。论文已说明这些细节，但这也意味着跨模型对比仍受到 frame sampling、closed-source API、上下文长度和推理预算差异影响。

#### 我的分析：优点

1. 问题定义清晰：同时抓住“长视频成本”和“多任务能力冲突”
   很多 MLLM 技术报告只堆 benchmark，而这篇文章把两个核心瓶颈讲得比较清楚：长视频需要 sparse long-context modeling，多能力 post-training 需要解决 catastrophic forgetting。DSA 和 MOPD 分别对应这两个瓶颈，主线较完整。

2. DSA 适配 GQA-based multimodal backbone 有工程价值
   把 DSA 用到 GQA 多模态架构中，比单纯宣称支持 256K context 更有实用意义。论文还讨论了 indexer warm-up、sparse adaptation、variable-length sequence optimization、RL 中 deterministic Top-k 等细节，说明作者确实面对了训练和推理落地问题。

3. Cross-Modal MOPD 对 post-training 冲突的处理值得关注
   MOPD 不是简单 teacher distillation，而是 on-policy、multi-teacher、cross-modal、token-level feedback。尤其 top-k overlap estimator 只在 teacher 和 student 都认为 plausible 的局部分布上比较，能减少 full-vocabulary KL 中低概率 token 的噪声。这对多领域 RL teacher consolidation 很有启发。

4. 训练 curriculum 完整，覆盖从 projector 到 256K long context
   Stage 0 到 Stage 3 的设计比较系统：先稳定视觉语言接口，再做 32K 对齐，再注入 64K 多任务能力，最后扩展到 256K 长上下文。视频 curriculum 也从 15s 到 15min 再到 2h，符合能力渐进学习逻辑。

5. 长视频 temporal grounding 结果突出
   TimeLens 三个子集都领先，说明模型不只是“看更多帧”，而是在 temporal localization 上确实有收益。scene-wise dense caption、TVG data 和 tIoU-centered Video RL 的组合比较针对性。

6. 系统工程描述比较扎实
   ExtraIO、ViT-LM heterogeneous parallelism、load balancing、FlashInfer/TileLang DSA kernels、Chunk ViT、Top-k deduplication 等内容说明这不是单纯算法报告，而是围绕大规模长视频训练和推理做了系统协同优化。

7. 30B-A3B MoE 的部署效率定位明确
   总参数 30B、激活 3B，使它在同规模开源模型中有较好性价比。对于长视频和 agent 场景，激活参数少带来的推理成本优势很重要。

8. 保留 text-only 数据和 domain experts 有助于避免多模态训练损害语言能力
   SFT 中约 40% text-only data，以及 pure-text math、code、instruction 等 teacher 的 MOPD consolidation，体现了作者对 language-side capability preservation 的重视。

#### 我的分析：缺点 / 风险

1. 缺少关键消融，难以判断 DSA、MOPD、Video RL 各自贡献
   论文强调很多组件都有效，但没有系统回答：如果不用 MOPD 会掉多少？如果只用 single teacher distillation 会怎样？top-k overlap 是否优于 teacher-only top-k 或 student-only top-k？DSA 对最终长视频准确率的独立贡献是多少？这会影响方法可复现和学术说服力。

2. 数据规模极大，复现门槛很高
   预训练涉及 4B、1T、2T、500B tokens，SFT 又有约 500B tokens，还包含大量内部数据、业务数据、专家模型生成数据和合成数据。即使模型开源，完整训练 recipe 对学术社区仍难复现。

3. benchmark 对比可能受推理预算和采样策略影响
   长视频 benchmark 对 frame sampling、context length、FPS、是否使用字幕、是否允许更多帧非常敏感。论文已披露部分设置，但不同模型之间仍可能不是完全同等预算。尤其 closed-source 模型的内部采样和上下文策略不可控。

4. MOPD 的 teacher routing 和 token-level feedback 复杂度高
   13 个 domain teachers、SPRR 对齐、Top-k distillation、token-category scaling、repetition penalty 都会增加系统复杂度。实际部署或复现时，任何 tokenizer、视觉 token count、chat template、mRoPE position 的错位都可能让 KL 信号错误传播，论文自己也强调严格 alignment verification 是必要的。

5. On-policy distillation 可能继承学生探索分布的盲区
   MOPD 在学生自己生成的 on-policy states 上提供 teacher feedback，这有利于贴近学生真实错误，但如果学生很少探索到某些高质量推理路径，teacher 也难以在这些未访问状态上提供监督。论文没有充分讨论 exploration coverage 问题。

6. Long-context “lossless 256K” 的表述需要谨慎理解
   论文说 DSA 支持 lossless 256K context processing，但 sparse attention 本质上仍选择 Top-k 证据。这里的 “lossless” 更可能指输入上下文不被截断或强抽帧，而不是注意力计算完全等价于 full attention。对于需要密集全局交互的任务，Top-k 稀疏选择仍可能遗漏弱信号。

7. 真实视频 agent 风险尚未充分评估
   future work 指向自动剪辑、内容治理、广告标签和商业投放，这些场景对误判、偏见、隐私、版权和商业安全非常敏感。论文主要报告能力 benchmark，没有充分展开安全治理、错误代价、人工审核和可解释性机制。

8. 视觉帧加 timestamp 的统一建模简单有效，但可能不是视频原生最优
   把每帧作为 image token 并注入 timestamp 有工程简洁性，但对 motion dynamics、帧间连续动作、低级时序信号可能不如视频原生 encoder 或更深的 temporal module。作者在 future work 中提到 native multimodal modeling，也间接说明当前方案仍有提升空间。
