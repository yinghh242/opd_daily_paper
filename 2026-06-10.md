### 1. When Context Returns: Toward Robust Internalization in On-Policy Distillation

- arXiv: https://arxiv.org/abs/2606.11627

#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy distillation。

#### Abstract 原文

> Recent work has shown that on-policy distillation can internalize privileged context, such as system prompts or task hints, into a student model so that the context is no longer needed at inference time. Although this approach successfully improves the student's no-context performance, we identify an interesting and previously unstudied phenomenon: in many settings, reintroducing the original privileged context to the distilled student actually degrades its performance, even on instances it already solves correctly without context. We term this context-induced degradation and argue that robust internalization demands not only matching the teacher's context-conditioned behavior, but also remaining stable when the context is reintroduced, a property we call context removability. Motivated by this observation, we propose a lightweight consistency regularizer that first anchors the student's no-context output via stop-gradient, then penalizes the context-conditioned output for deviating from it via forward KL divergence. This simple addition requires only one extra forward pass per training step, yet it effectively mitigates context-induced degradation and, in many cases, even improves no-context performance. Across 12 configurations spanning diverse domains and model families, our method improves context-conditioned accuracy in the majority of settings, reduces context-induced harm in 11 out of 12 settings, and effectively eliminates response-length inflation. A mechanistic case study further confirms that context removability is achieved at the representation level, with hidden states remaining nearly identical regardless of whether the context is present.

#### 摘要中文翻译

近期研究表明，on-policy distillation（在线策略蒸馏）可以把 privileged context（特权上下文），例如 system prompts（系统提示词）或 task hints（任务提示），内化到 student model（学生模型）中，使推理时不再需要该 context。尽管这种方法能成功提升 student 的 no-context performance（无上下文性能），本文发现了一个有趣且此前未被研究的现象：在许多设置中，把原始 privileged context 重新提供给已蒸馏的 student，反而会降低其性能，甚至会让模型在原本无 context 时已经答对的样本上出错。作者将其称为 context-induced degradation（上下文诱发退化），并认为 robust internalization（鲁棒内化）不仅要求 student 匹配 teacher 的 context-conditioned behavior（带上下文行为），还要求当 context 被重新引入时模型保持稳定；作者把这一性质称为 context removability（上下文可移除性）。基于这一观察，本文提出一个 lightweight consistency regularizer（轻量一致性正则项）：先通过 stop-gradient（停止梯度）锚定 student 的 no-context output，再用 forward KL divergence（前向 KL 散度）惩罚 context-conditioned output 偏离该锚点。这个简单附加项每个 training step 只需要一次额外 forward pass，却能有效缓解 context-induced degradation，并且在许多情况下还能提升 no-context performance。在覆盖多种领域和模型族的 12 个配置中，该方法在多数设置中提升 context-conditioned accuracy，在 12 个设置中的 11 个降低 context-induced harm，并有效消除 response-length inflation（回答长度膨胀）。进一步的 mechanistic case study（机制案例研究）证实，context removability 也在 representation level（表征层面）实现：无论 context 是否存在，hidden states 几乎保持一致。

#### 研究背景

大语言模型经常依赖 privileged context，例如 system prompt、chain-of-thought traces、task hints 或 text game 中的 game-state scaffold。这类 context 能通过 in-context learning 提升表现，但也带来三类实际问题：推理延迟增加、服务成本上升、敏感 prompt 或策略信息可能暴露给用户。

已有 on-policy distillation / on-policy context distillation 的核心目标是把 teacher 在带 context 条件下的行为蒸馏到 student 的无 context 路径中，使部署时无需再提供 context。这个目标可以理解为 privileged fidelity：student without context 应该匹配 teacher with context。

本文指出，仅优化 privileged fidelity 不够。一个真正“内化”了 context 的 student，在重新看到同一 context 时不应变差；换言之，重新提供已经内化的信息应近似是 idempotent operation（幂等操作）。但作者发现，多数设置中 OPD 后的 student 在 context reintroduction 时会出现：

- accuracy / win rate 下降；
- 原本无 context 答对的样本被 context 翻转为错误，即 Harm 上升；
- QA 任务中 response length 明显膨胀。

作者将这一现象定义为 context-induced degradation，并提出新的鲁棒性目标 context removability：如果模型已经把 context 吸收到参数中，则 no-context view 和 context-conditioned view 应该一致，即 `$q_x \approx q_c$`。论文进一步把不同任务中的 context interaction 分为三类：

- Regime A：context 变成扰动，student 无 context 已表现良好，但重新引入 context 会导致 accuracy drop 和 high Harm。这是最常见情形。
- Regime B：context 仍是有用 scaffold，说明 internalization 不完全；强制 removability 可能牺牲一部分带 context 性能。
- Regime C：两种 view 已基本一致，Harm 很低，NCA 影响较小。

#### Method

论文设 student model 为 `$g_\theta$`，输入为 `$x$`，固定 privileged context 为 `$c$`。student 的两种 view 定义为：

$$
q_x \triangleq g_\theta(y \mid x), \quad q_c \triangleq g_\theta(y \mid x,c).
$$

标准 OPD with privileged context 最小化：

$$
\mathcal{L}_{\mathrm{OPD}}
=
\mathbb{E}_{x \sim \mathcal{D}}
\left[
\mathrm{KL}\left(q_x \parallel f(y \mid x,c)\right)
\right],
$$

其中 `$f(y \mid x,c)$` 是 teacher distribution。实际实现中 KL 在 token level 上计算并沿序列求和。

关键问题是：这个目标只约束 `$q_x$`，即 student 的 no-context pathway；它不直接约束 `$q_c$`。因此，student 可能在无 context 时模仿 teacher，但在重新看到 context 时进入另一个行为模式，导致 context-induced degradation。

作者首先把 context removability 写成约束优化：

$$
\min_\theta \mathcal{L}_{\mathrm{OPD}}(\theta)
\quad
\mathrm{s.t.}
\quad
D(q_x,q_c) \le \epsilon.
$$

实际训练中将其放松为带权正则项：

$$
\mathcal{L}
=
\mathcal{L}_{\mathrm{OPD}}
+
\beta \mathbb{E}_{x \sim \mathcal{D}}
\left[
D(q_x,q_c)
\right].
$$

本文选择 forward KL divergence，并对 no-context view 使用 stop-gradient，得到 No-Context Anchoring (NCA)：

$$
\mathcal{L}_{\mathrm{NCA}}
\triangleq
\mathrm{KL}
\left(
\mathrm{sg}\left[g_\theta(\cdot \mid x)\right]
\parallel
g_\theta(\cdot \mid x,c)
\right).
$$

最终目标为：

$$
\mathcal{L}
=
\mathcal{L}_{\mathrm{OPD}}
+
\beta \mathcal{L}_{\mathrm{NCA}}.
$$

NCA 的设计动机有三点：

- forward KL 是 mode-covering，有助于防止 context-conditioned view `$q_c$` 丢失 no-context view `$q_x$` 的模式；
- 期望在 `$q_x$` 下计算，因此可以复用 OPD 已生成的 on-policy samples，不需要从 `$q_c$` 额外 rollout；
- stop-gradient 让 `$q_x$` 只接受 OPD loss 的梯度，让 `$q_c$` 只接受 NCA loss 的梯度，减少两个目标之间的 gradient competition。

计算成本方面，NCA 每个 training step 只增加一次 `$g_\theta(\cdot \mid x,c)$` 的 forward pass。OPD 原本已经计算 `$g_\theta(\cdot \mid x)$`，NCA 只需 detach 这一路 logits 作为 anchor，再对带 context 路径计算 KL。

总体实验结论是：context-induced degradation 在 OPD 后相当普遍，尤其 Regime A 中 context 会从有益 scaffold 变成扰动源。NCA 作为一个很小的 consistency regularizer，通常能同时降低 Harm、提高 BothCorrect / BothWon、减少 response-length inflation，并且经常提升 no-context performance。但在 Regime B 中，如果 context reintroduction 仍提供额外有效信息，固定地把 `$q_c$` 拉向 `$q_x$` 可能牺牲 with-context performance。

#### 论文自我评价的缺陷

论文在 Limitations 中明确承认三点：

- 由于计算资源限制，所有实验只使用 single training seed；多 seed 验证会让结论更稳健。
- mechanistic analysis 只在一个 Sokoban case study、一个模型上进行；如果扩展到更多 instances、tasks 和 architectures，会更能支撑 representation-level 结论。
- 方法主要针对 OPD setting 设计，若用于其他 distillation paradigms，可能需要适配。

此外，论文在实验讨论中也承认一个重要边界现象：Qwen3-4B-Instruct-2507 到 Qwen3-1.7B 的 FrozenLake cross-model distillation 中，OPD student with context 的表现异常地超过 teacher with context，而 NCA 反而压制了这种收益并提高 Harm。作者明确表示该机制仍不清楚，并提出未来可研究 dynamic anchor，即根据 internalization 程度动态决定锚定 no-context view 还是 context-conditioned view。

#### 我的分析：优点

- 问题定义有价值：论文没有只看 OPD 后 student 无 context 的性能，而是提出“context 被重新引入时是否仍稳定”这一部署中很实际的问题。对 system prompts、policy prompts、安全规则、工具使用提示等场景都很相关。
- context removability 是一个清晰的新评价维度：privileged fidelity 只要求 `$q_x$` 模仿 teacher with context，而 context removability 要求 `$q_x \approx q_c$`。这个区分抓住了 OPD 的盲点。
- 方法简单且工程成本低：NCA 只加一个 forward KL consistency regularizer 和一次额外 forward pass，不改变 OPD rollout 流程，也不需要复杂 reward 或额外标注。
- stop-gradient 设计合理：把 no-context output 作为 anchor，避免 NCA 反向拉动 `$q_x$` 偏离 teacher distillation 目标，设计上比 SymKL 更克制。
- 实验覆盖面相对完整：包含 QA 和 text games 两类任务，多个模型族，self-distillation 和 cross-model distillation，并报告 Harm、BothCorrect / BothWon、response length 等更能刻画稳定性的指标。
- 机制分析增强说服力：hidden-state similarity、gradient alignment、parameter update distribution 都支持 NCA 不是只在输出格式上“修补”，而是在内部表征上减少 context / no-context 分歧。
- 对 OPD 研究有启发：论文说明“蒸馏成功”不应只用无 context accuracy 判断，还应测试 context reintroduction robustness。这一点可以直接影响后续 OPD benchmark 的设计。

#### 我的分析：缺点 / 风险

- single seed 是较大统计风险：很多结果差异只有 0.6 到 2 points，若无多 seed 或置信区间，难判断小幅提升是否稳定。
- 任务规模偏小：QA 只用 500 左右 test samples，text games 只评估 128 instances；对更大规模、更复杂、多轮交互或真实用户分布的泛化仍不明确。
- privileged context 多为固定 context：论文设定中 `$c$` 对每个任务基本固定或模板化。如果真实部署中 context 是动态的、用户相关的、时间更新的，强制 context removability 可能不合适。
- NCA 的核心假设并非总成立：它假设 reintroduced context 应该是 redundant information。但在 Regime B 或现实场景中，context 可能仍含有 student 未完全内化的新信息；此时把 `$q_c$` 拉向 `$q_x$` 可能压制有益上下文。
- FrozenLake cross-model 失败案例值得警惕：唯一 Harm 变差的设置并非微小异常，而是 `$WR_{x,c}$` 从 86.5 降到 70.1。这说明 NCA 可能破坏某些 on-policy training 中偶然形成的强策略模式。
- `$\beta$` 仍需调参：虽然论文显示 `$\beta$` 有一定鲁棒性，但不同模型和任务仍可能需要不同强度。过小不能抑制 drift，过大可能损害 no-context 学习或有用 context 利用。
- 额外 forward pass 对大模型并非免费：论文称 lightweight 是相对 OPD 而言；在大规模训练或长序列任务中，每步多一次 context-conditioned forward 仍可能显著增加成本。
- 机制分析样本太少：hidden-state 和 gradient 分析只用 16 个 Sokoban instances，且只覆盖一个模型。它能提供解释线索，但不足以证明所有模型中的通用机制。
- 评价集中在 accuracy / win rate：对 generation quality、calibration、factuality、安全拒答边界、长文本任务中的语义一致性等没有深入验证。
- response-length inflation 被消除未必总是好事：长度变短可能意味着更简洁，也可能意味着 reasoning 被压缩或解释减少。论文主要从稳定性角度解释，没有充分分析回答质量和可解释性变化。
- 对安全类任务需谨慎：如果 system prompt 包含最新安全策略或部署约束，强行让模型在有无 prompt 时一致，可能使模型忽略显式安全更新；实际应用中需要区分“应内化的稳定规则”和“应实时服从的动态规则”。

### 2. Rubric-Guided Self-Distillation: Post-Training Without Rubric Verifiers

- arXiv: https://arxiv.org/abs/2606.12507
- PDF: https://arxiv.org/pdf/2606.12507
- Authors: MohammadHossein Rezaei, Anas Mahmoud, Zihao Wang, Utkarsh Tyagi, Advait Gosai, Razvan-Gabriel Dumitru, Aakash Sabharwal, Bing Liu, Yunzhong He
- 作者单位：Scale AI（所有作者均标注为同一单位）。
- arXiv published: 2026-06-10
- arXiv updated: 2026-06-10
- Categories: cs.LG

#### 与 OPD / OPSD 的相关性依据

标题或摘要同时包含 on-policy/on policy 与 distillation/self-distillation。

#### Abstract 原文

> Rubrics have emerged as an alternative to RLVR in open-ended domains where a single ground-truth final answer is not available. Existing rubric-based training methods rely on an LLM verifier that scores each rollout against rubrics. This introduces substantial training-time overhead, exposes optimization to verifier-specific biases, and reduces rubric feedback to a sparse end-of-trajectory signal. We propose Rubric-Guided Self-Distillation (RGSD), a verifier-free training method in which the base policy, conditioned on the rubric, serves as the teacher for the unconditioned student. RGSD distills the rubric-conditioned teacher distribution into the student token-by-token, replacing sparse trajectory-level rewards with dense per-token learning signals and removing the LLM judge from the training loop entirely. Across Qwen-2.5 (3B, 7B) and Qwen3-Thinking (4B, 8B) models on medical and science domains, RGSD achieves rubric satisfaction comparable to judge-based GRPO while using one on-policy rollout per prompt and no training-time verifier calls. Ablations show that raw rubrics provide a stronger teacher enrichment signal than self-generated reference responses, while a stronger GRPO judge can outperform RGSD in some settings, positioning RGSD as a complementary verifier-free alternative when verifier cost or reliability is the bottleneck.

#### 摘要中文翻译

Rubrics（评分规程/评价准则）已成为 RLVR 在开放式领域中的一种替代方案，因为这些领域通常不存在单一的 ground-truth final answer（标准最终答案）。现有 rubric-based training（基于评分规程的训练）方法依赖 LLM verifier（大语言模型验证器）根据 rubrics 对每个 rollout（生成轨迹）打分。这会带来显著的训练时开销，使优化暴露于 verifier-specific biases（验证器特定偏差），并把 rubric feedback（评分规程反馈）压缩为稀疏的 end-of-trajectory signal（轨迹末端信号）。本文提出 Rubric-Guided Self-Distillation（RGSD），一种 verifier-free（无验证器）训练方法：base policy（基础策略）在 rubric 条件下作为 teacher（教师），而不带 rubric 条件的模型作为 student（学生）。RGSD 逐 token 地将 rubric-conditioned teacher distribution（基于评分规程条件化的教师分布）蒸馏到 student 中，用 dense per-token learning signals（密集逐 token 学习信号）替代稀疏的 trajectory-level rewards（轨迹级奖励），并完全移除训练循环中的 LLM judge（大模型裁判）。在 medical 和 science 领域，跨 Qwen-2.5（3B、7B）与 Qwen3-Thinking（4B、8B）模型，RGSD 在 rubric satisfaction（评分规程满足度）上达到与 judge-based GRPO（基于裁判的 GRPO）相近的表现，同时每个 prompt 只使用一次 on-policy rollout，并且没有训练时 verifier calls（验证器调用）。消融实验表明，raw rubrics（原始评分规程）比 self-generated reference responses（自生成参考回答）提供更强的 teacher enrichment signal（教师增强信号）；但在某些设置中，更强的 GRPO judge 可以超过 RGSD。因此，RGSD 被定位为一种 complementary verifier-free alternative（互补的无验证器替代方案），适用于 verifier cost（验证器成本）或 reliability（可靠性）成为瓶颈的场景。

#### 研究背景

开放式任务如 clinical advice（临床建议）、scientific exposition（科学阐释）、instruction following（指令遵循）和 creative writing（创意写作）很难像数学、代码或选择题那样使用 programmatic verifiers（程序化验证器）。因此，rubrics 逐渐成为这类任务的核心评价工具：每个 prompt 配有一组带权重的 criteria（评价准则），LLM judge 根据这些 criteria 对回答逐项判定并聚合成 scalar reward（标量奖励）。

论文指出，rubric-based RL 虽然有效，但有三个核心问题：

1. computational cost（计算成本）：GRPO 等方法通常需要对每个 prompt 采样多个 rollouts，并对每个 rollout 调用 LLM judge。本文默认 GRPO 使用 $G=16$ 个 rollouts，五个 epoch 下每个训练 prompt 需要 $80$ 次 batched judge calls。
2. verifier blind spots（验证器盲点）：LLM judge 的偏好、偏差和漏洞会进入 reward signal，导致 reward hacking（奖励黑客）或 judge-specific optimization（针对裁判的优化）。
3. credit assignment（信用分配）困难：rubric 判断最终被压缩为一个 trajectory-level scalar reward，无法清楚告诉模型哪些 token 或局部决策真正帮助满足了哪些 criteria。

本文的关键观察是：同一个 base model 如果在上下文中看到 prompt-specific rubric，其回答的 rubric satisfaction 会显著提升。例如在 RubricHub-medical 上，Qwen-2.5-7B 从 $33.0$ 提升到 $78.0$，增幅 $45.0$ pp；Qwen3-4B-Thinking 从 $43.6$ 提升到 $89.8$，增幅 $46.2$ pp。RGSD 的核心想法就是把这种 rubric-conditioning gap（评分规程条件化增益）蒸馏进不带 rubric 的 student，使 student 在推理时不需要看到 rubric，也能部分内化 rubric 对回答形态的指导。

相关工作上，论文把自己放在两条线之间：

- rubric-based RL / Rubrics as Rewards：使用 LLM judge 将 rubric satisfaction 转成 reward，再用 GRPO 等 RL 方法优化。
- On-Policy Distillation / OPSD（on-policy self-distillation）：student 生成 on-policy rollout，teacher 在更强信息条件下给出逐 token 分布。RGSD 继承 dense per-token supervision（密集逐 token 监督），但 teacher 的 privileged information（特权信息）不是 ground truth answer 或 reference response，而是 rubrics。

#### Method

设一个训练样本为 $(q,R)$，其中 $q$ 是 prompt，$R=\{(c_i,w_i)\}_{i=1}^{K}$ 是 rubric set，包含 criterion $c_i$ 和权重 $w_i$。传统 rubric judge $M_J$ 对候选回答 $y$ 的每个 criterion 给出二值判定 $v_i(q,c_i,y)\in\{0,1\}$，并聚合为：

$$
s_J(q,R,y)=
\frac{\sum_{i=1}^{K} w_i v_i(q,c_i,y)}
{\sum_{i=1}^{K} w_i}
\in [0,1].
$$

rubric-RL 的目标是最大化：

$$
\max_{\theta}
\mathbb{E}_{(q,R)\sim D,\ y\sim \pi_{\theta}(\cdot\mid q)}
\left[
s_J(q,R,y)
\right].
$$

GRPO 的做法是：对每个 prompt 采样 $G$ 个 rollouts，用 LLM judge 给每个 rollout 打分，再通过 group-relative advantage（组相对优势）更新 policy。这带来大量 verifier calls，并且 reward 仍然是 sparse trajectory-level signal。

RGSD 则完全移除训练循环中的 judge。它从同一个 base checkpoint 初始化两个模型：

- student：参数为 $\theta_S$，可训练，只看到 prompt $q$，定义 policy $\pi_S(\cdot\mid q)$；
- teacher：参数为 $\theta_T=\theta_{\mathrm{base}}$，冻结，看到 prompt $q$、rubric set $R$，以及 student rollout 的 prefix $y_{<t}$，定义分布 $\pi_T(\cdot\mid q,R,y_{<t})$。

训练步骤如下：

1. student 根据 prompt $q$ 采样 on-policy rollout $y\sim\pi_S(\cdot\mid q)$；
2. frozen teacher 在每个 token prefix $y_{<t}$ 上，结合 rubric $R$ 给出 next-token distribution；
3. student 用 clipped Jensen-Shannon divergence（截断 Jensen-Shannon 散度）逐 token 匹配 teacher distribution。

论文给出的 RGSD loss 为：

$$
\mathcal{L}_{\mathrm{RGSD}}(\theta_S)
=
\mathbb{E}_{(q,R)\sim D,\ y\sim \pi_S(\cdot\mid q)}
\left[
\frac{1}{T}
\sum_{t=1}^{T}
D^{\mathrm{clip}}_{\beta}
\left(
\pi_S(\cdot\mid q,y_{<t})
\ \|\
\pi_T(\cdot\mid q,R,y_{<t})
\right)
\right].
$$

其中 $D^{\mathrm{clip}}_{\beta}$ 是 clipped Jensen-Shannon divergence，$\beta=0$ 接近 forward KL，$\beta=1$ 接近 reverse KL；本文统一使用 $\beta=0.5$，JSD clip 为 $\tau=0.05$。只有 student 接收梯度，teacher 全程冻结。

对 Qwen3-Thinking 这类 reasoning-trace model，论文发现 teacher 的 `` 之间的 tokens 做 loss mask，只蒸馏 final answer tokens。附录显示，这一 masking 对 Qwen3-Thinking 带来约 $2$--$4$ pp 改善。

#### 论文自我评价的缺陷

论文明确单列了 Limitations，主要包括：

1. replication cost（复现实验成本）高。GRPO 五个 epochs、每个 prompt $16$ rollouts，会导致 medical 约 $1.0$M、science 约 $1.6$M 次 judge calls。因此主表大多数 runs 是 single-seed。作者只在 headline setting，即 Qwen-2.5-7B medical 上做了 $3$-seed envelope，并观察到 peak 附近 $\sigma \approx 0.4$ pp。因此论文强调小于 $1$ pp 的差距应解释为 parity，而不是显著胜利。
2. RGSD 依赖 meaningful rubric-conditioning response。RGSD 只能转移 base model 在看到 rubric 时已经能表达出的行为。如果 rubric-conditioning gap 很小，teacher 本身没有提供足够新增信息，RGSD 的收益会有限。
3. RGSD 不是 verifier-based RL 的通用替代品。当有 substantially stronger verifier 且成本可接受时，judge-based GRPO 可能更好。论文的 judge-strength ablation 中，gpt-oss-120b judge 让 GRPO 在 Qwen-2.5-7B science 上超过 RGSD。
4. evaluation 仍然依赖 LLM judges。RGSD 移除了 training loop 中的 LLM verifier，但 rubric satisfaction 的评价仍由 gpt-5.4 judge 完成，HealthBench factual-claim audit 也使用 LLM extract-and-verify pipeline。因此绝对分数仍可能继承 judge-specific biases。
5. reasoning-trace models 需要额外实现处理。Qwen3-Thinking 的 teacher 在 `` 中会泄露 rubric criteria；论文通过 masking thinking-block loss 缓解。这说明 RGSD 在 reasoning model 上不是完全 plug-and-play，需要针对 trace leakage 做工程处理。
6. cross-family generalization 不完整。附录展示 OLMo 和 Mistral 上 RGSD 也能提升，但由于 judge cost 没有跑 matching GRPO baselines，所以不能证明在这些模型族上 RGSD 与 GRPO 的相对关系。

#### 我的分析：优点

1. 核心想法简单且抓住了 rubric-based RL 的真实瓶颈。论文没有试图发明更复杂的 reward model，而是直接利用“base model 看到 rubric 后会变好”这个现象，把 rubrics 从 external grading instrument（外部评分工具）改造成 privileged teacher context（教师特权上下文）。
2. 对 open-ended post-training 很实用。许多开放式任务没有 verifiable reward，但有 rubrics；RGSD 正好适用于 verifier calls 昂贵、延迟高、不可批量稳定调用或 judge reliability 存疑的场景。
3. dense per-token supervision 比 sparse scalar reward 更适合长回答。GRPO 的 trajectory-level reward 很难做 token-level credit assignment，而 RGSD 每个 token prefix 都能从 teacher distribution 得到学习信号。
4. 方法 operationally simpler。RGSD 不需要训练时 LLM judge，不需要每个 rollout 外部评分，也避免了 judge API rate limit、failed calls、latency 等工程问题。
5. 论文的定位比较诚实。作者没有声称 RGSD 全面支配 GRPO，而是明确说二者是 complementary regimes：RGSD 适合 verifier bottleneck，stronger-judge GRPO 适合愿意付出更高 verifier cost 以追求更高 rubric score 的场景。
6. 消融实验有说服力。raw rubric vs self-golden、gpt-4o-mini vs gpt-oss-120b、thinking-token mask、factual-claim audit 都直接回应了方法有效性和适用边界。
7. 对 reward hacking 风险有实证触及。论文不仅看 rubric score，还检查了 length inflation 和 false-claim rate，发现 GRPO 的额外长度可能带来更多 unsupported claims。这比只报 rubric satisfaction 更有价值。
8. 对 reasoning-trace leakage 的处理细致。作者没有忽略 Qwen3-Thinking teacher 在 `` 中显式引用 rubric 的问题，而是用数据统计和 mask ablation 证明了 masking 的必要性。

#### 我的分析：缺点 / 风险

1. RGSD 的上限受 base teacher 限制。teacher 与 student 来自同一个 base checkpoint，只是 teacher 多看了 rubric；如果 base model 本身知识不足、推理错误或容易 hallucinate，RGSD 可能只是把这些错误以更符合 rubric 的形式蒸馏给 student。
2. rubric satisfaction 不等于真实质量。rubric-based evaluation 仍可能偏向 coverage、keyword inclusion 或格式完整性。即使 RGSD 避免了直接优化 judge reward，它仍然在学习 rubric-conditioned distribution，可能强化 rubric 本身的偏差。
3. 评价仍是 judge-mediated。训练时 verifier-free 不代表最终结论完全摆脱 LLM judge。主结果由 gpt-5.4 打分，factual audit 也由 LLM 完成，因此分数仍可能受到 evaluation judge 偏好的影响。
4. single-seed 主结果限制显著性。虽然作者给出一个 $3$-seed sanity check，但主表大部分配置仍是 single-seed。很多 RGSD 与 GRPO 的差距很小，应谨慎解读。
5. stronger judge 可以改变结论。gpt-oss-120b judge 的 GRPO 在 science 上明显超过 RGSD，说明当 verifier 足够强时，dense self-distillation 未必能替代高质量 reward optimization。
6. Thinking model 的 trace masking 是额外复杂性。对于带 hidden reasoning trace 的模型，teacher 可能在内部推理中直接使用 rubric 编号或内容。masking 虽然有效，但也意味着 RGSD 对不同模型输出格式、chat template 和 trace boundary 有依赖。
7. top-k vocabulary approximation 可能影响 distillation fidelity。为适配 Qwen3-8B-Thinking 的大词表，论文将 JSD loss 限制在 teacher top-$128$ vocabulary entries；这有工程必要性，但也可能丢失 tail distribution 信息。
8. factuality 风险没有完全解决。RGSD 的 false-claim rate 从约 $30.5\%$ 上升到 $35.1\%$，只是比 GRPO 的 $45.1\%$ 更低，并不是消除了 unsupported claims。尤其在 medical domain，这仍然是很大的风险。
9. RGSD 可能学习“如何看起来满足 rubric”，而不是学到更可靠的 domain knowledge。因为 teacher signal 来自 rubric-conditioned base distribution，而不是外部事实验证或人类专家反馈。
10. 适用前需要先测 rubric-conditioning gap。论文也承认 gap 是 RGSD 的 operating regime。如果某个任务中 base model 看到 rubric 后提升不大，RGSD 可能几乎没有可转移信号，甚至不如直接做 better judge-based RL。

### 3. RLCSD: Reinforcement Learning with Contrastive On-Policy Self-Distillation

- arXiv: https://arxiv.org/abs/2606.11709
- PDF: https://arxiv.org/pdf/2606.11709
- Authors: Leyi Pan, Shuchang Tao, Yunpeng Zhai, Lingzhe Zhang, Zhaoyang Liu, Bolin Ding, Aiwei Liu, Lijie Wen
- 作者单位：Tsinghua University; Tongyi Lab, Alibaba Group; Peking University
- arXiv published: 2026-06-10
- arXiv updated: 2026-06-10
- Categories: cs.LG, cs.CL

#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy distillation。

#### Abstract 原文

> On-policy self-distillation (OPSD) provides dense, token-level supervision for reasoning models by aligning a model's own distribution with the distribution it produces under privileged context, typically a verified solution. However, we show that the learning signal drawn from this distributional gap concentrates on style tokens rather than task-bearing ones, as the hinted model tends to produce more direct, shorter outputs. We term this pathology \emph{privilege-induced style drift}, which destabilizes training or causes response length to shrink. To address this, we propose \textbf{RLCSD} (Reinforcement Learning with Contrastive on-policy Self-Distillation), which mitigates this drift by contrasting the teacher-student gap under a correct hint against that under a wrong hint, suppressing the style shift that conditioning on a hint tends to induce regardless of correctness, and yielding a signal that is more concentrated on task-bearing tokens. Experiments on Qwen3 (1.7B/4B/8B) and Olmo-3-7B-Think across mathematical and logical reasoning show that RLCSD consistently outperforms GRPO and prior OPSD methods. We further show that the contrastive principle is general: it plugs into existing OPSD methods to improve them, and its underlying insight extends to the broader cross-model on-policy distillation setting.

#### 摘要中文翻译

On-policy self-distillation (OPSD，在策略自蒸馏) 通过将模型自身的分布与其在 privileged context（特权上下文，通常是 verified solution）下产生的分布对齐，为 reasoning models 提供 dense, token-level supervision（密集的 token 级监督）。然而，作者发现，这种分布差异中的学习信号会集中在 style tokens（风格 token）而非 task-bearing tokens（承载任务信息的 token）上，因为带有 hint 的模型倾向于生成更直接、更短的输出。作者将这一问题称为 privilege-induced style drift（特权诱发的风格漂移），它会导致训练不稳定或响应长度收缩。

为了解决这一问题，论文提出 **RLCSD**，即 Reinforcement Learning with Contrastive on-policy Self-Distillation（带对比式在策略自蒸馏的强化学习）。RLCSD 通过对比 correct hint（正确提示）下的 teacher-student gap 与 wrong hint（错误提示）下的 gap，抑制由 hint conditioning 本身引起、但与正确性无关的 style shift，从而得到更集中于 task-bearing tokens 的信号。论文在 Qwen3 1.7B/4B/8B 和 Olmo-3-7B-Think 上，针对 mathematical reasoning 与 logical reasoning 任务进行实验，结果显示 RLCSD 持续优于 GRPO 和已有 OPSD 方法。作者还进一步表明，contrastive principle（对比原则）具有通用性：它可以插入已有 OPSD 方法中提升效果，其核心洞察也可扩展到更广义的 cross-model on-policy distillation 场景。

#### 研究背景

RLVR（Reinforcement Learning with Verifiable Rewards）尤其是 GRPO，已经成为训练 reasoning models 的重要范式。它依赖 verifier 给出的 outcome-level reward（结果级奖励），例如答案是否正确。但这类信号非常稀疏：长链式推理可能包含上千个 token，GRPO 却只能把同一个 scalar advantage 广播到所有 token，无法细粒度区分哪些中间步骤真正导致了正确或错误。

OPD（On-Policy Distillation，在策略蒸馏）试图解决这一问题：让 student 在自己的 on-policy rollouts 上，利用 teacher 的 token-level logits 产生 dense supervision。其典型信号来自 teacher 与 student 在每个 token 上的 distribution gap，因此比单纯 outcome reward 更细。但 cross-model OPD 有实际限制：需要 white-box teacher logits、teacher 与 student 通常要共享 vocabulary，还需要同时服务一个更大的 teacher，带来内存和延迟开销。

OPSD（On-Policy Self-Distillation，在策略自蒸馏）进一步把 teacher 和 student 设为同一个模型，只是 teacher 得到 privileged context，例如 ground-truth solution 或 correct rollout。这样避免了独立 teacher 的部署成本，也绕开了跨模型 vocabulary 不一致问题。已有 OPSD 方法通常使用 one-sided positive privileged context，即只给 teacher 正确参考解，然后构造 token-level signal：

$$
e_{c,t} = \log \pi_T(y_t \mid x, y_c^*, y_{<t}) - \log \pi_S(y_t \mid x, y_{<t})
$$

论文指出，这个信号并不干净。带有正确 hint 的 teacher 不只是更懂题，也会系统性改变生成风格：更短、更笃定、更少探索性表达，并把概率质量转移到 discourse tokens、format tokens、connectives 等风格相关 token 上。作者称之为 privilege-induced style drift。实验中，数学任务上 style-token 的平均 $|e_{c,t}|$ 为 0.263，而 task-token 仅为 0.083，约相差 3 倍。这意味着 OPSD 的 token-level signal 可能主要在教模型“怎么说”，而不是“怎么推理”。

这种偏移带来两个训练病症：一类 OPSD 方法会出现 entropy explosion 和 response length explosion，导致训练崩溃；另一类如 RLSD 虽然稳定，但会出现 premature response-length shrinkage，使长程推理能力受限。RLCSD 的核心动机就是从 token-level signal 中抵消这类与正确性无关的风格成分。

#### Method

RLCSD 的核心思想是：不要只看 correct hint 下 teacher 与 student 的差异，而是同时构造 correct hint 与 wrong hint，在完全相同的 prompt template 下让 teacher 读取二者，然后相减。由于 correct hint 和 wrong hint 都会诱发类似的“被参考答案引导后的风格漂移”，相减后共享的 style component 被部分抵消，剩下的信号更可能与任务正确性相关。

方法分三步。

1. Rollout sampling and partitioning。对每个 query $x$，按照 GRPO 方式从 student policy $\pi_S$ 采样一组 $G$ 个 rollouts。rule-based verifier 给每个 rollout 二值奖励：正确为 1，错误为 0。于是得到 correct subset $\mathcal{G}^+$ 与 incorrect subset $\mathcal{G}^-$。如果一个 group 全对或全错，则丢弃，因为这类 group 在 GRPO 的 group-relative normalization 下本来也没有有效 advantage。

2. Contrastive token-level signal。最朴素的 one-sided OPSD 使用 correct hint：

$$
e_{c,t} = \log \pi_T(y_t \mid x, y_c^*, y_{<t}) - \log \pi_S(y_t \mid x, y_{<t})
$$

RLCSD 同时引入 wrong hint：

$$
e_{w,t} = \log \pi_T(y_t \mid x, y_w^*, y_{<t}) - \log \pi_S(y_t \mid x, y_{<t})
$$

两者相减得到：

$$
e_{\mathrm{ctr},t} = e_{c,t} - e_{w,t}
$$

由于 student 项相消，最终信号可写为：

$$
e_{\mathrm{ctr},t}
=
\log \pi_T(y_t \mid x, y_c^*, y_{<t})
-
\log \frac{1}{K} \sum_{k=1}^{K}
\pi_T(y_t \mid x, y_{w,k}^*, y_{<t})
$$

其中 $y_c^* \sim \mathcal{G}^+ \setminus \{y\}$，$y_{w,k}^* \sim \mathcal{G}^- \setminus \{y\}$。这里有两个关键设计：

- K-marginalized negative hints：错误解的 error modes 很分散，单个 wrong hint 可能与目标错误 rollout 的错误类型完全不同，导致负分支不稳定。因此论文用 $K$ 个 negative hints 的概率均值来近似错误分布，主实验中 $K=4$。
- Excluding the target rollout from the hint pool：如果把目标 rollout 本身作为 hint，teacher 会对该 rollout 过度自信，破坏 token-level signal 的区分性。因此 positive/negative hints 都从 sibling rollouts 中抽取，不使用目标 rollout 自身。

3. Verifier-anchored modulation and two-path loss。RLCSD 不直接用 $e_{\mathrm{ctr},t}$ 作为 advantage，因为即使 contrastive signal 更干净，也仍可能存在 wrong-sign token。论文保留 GRPO 的 verifier-derived advantage $A_{\mathrm{ORM}}$ 作为更新方向锚点，只让 contrastive signal 调制 token-level 更新幅度。

首先将原始 contrastive signal 经过 tanh 压缩：

$$
r_t = \lambda \tanh \left( \frac{e_{\mathrm{ctr},t}}{\tau} \right)
$$

再用 mask 选出强信号 token：

$$
m_t = \mathbf{1}(|r_t| > \delta)
$$

实验中约 20%–30% 的 token 会被选中。对被选中的 token，RLCSD 将 $r_t$ 加到 $A_{\mathrm{ORM}}$ 上，但用 sign-preserving clamp 防止方向反转：

$$
\tilde{A}_t =
\begin{cases}
\max(0, A_{\mathrm{ORM}} + r_t), & A_{\mathrm{ORM}} \ge 0, \\
\min(0, A_{\mathrm{ORM}} + r_t), & A_{\mathrm{ORM}} < 0.
\end{cases}
$$

最终 loss 是 two-path PPO-style clipped objective：对 $m_t=0$ 的 token 使用原始 $A_{\mathrm{ORM}}$，对 $m_t=1$ 的 token 使用 $\tilde{A}_t$。两个路径分别归一化，而不是对所有 token 做单一平均。这样可以避免 modulated tokens 因只占 20%–30% 而被 unmodulated tokens 稀释。

论文还把 contrastive principle 扩展到其他 OPSD 方法。对 RLSD/RLCSD 这类 advantage-modulation 方法，直接用 $e_{\mathrm{ctr},t}$ 替代 one-sided $e_{c,t}$。对 OPSD 这类 full-distribution distillation 方法，论文使用 classifier-free-guidance 风格的 contrastive target：

$$
q_{\mathrm{target}} =
\mathrm{softmax}((1+\alpha)\ell_c - \alpha \ell_w)
$$

其中 $\ell_c$ 与 $\ell_w$ 分别是 correct hint 与 wrong hint 下的 teacher logits。

#### 论文自我评价的缺陷

论文未单列 limitations，但从全文可见以下限制与边界条件：

1. 依赖 verifier 和 binary rewards。RLCSD 的 rollout partitioning 需要把同一 query 的 rollouts 分成 $\mathcal{G}^+$ 与 $\mathcal{G}^-$，这要求任务有可靠的 rule-based verifier。论文实验集中在 math reasoning 和 Knight-and-Knaves，这些任务都容易验证；对开放式写作、对话、代码风格偏好、多目标任务等无法精确验证的场景，方法不能直接套用。

2. 需要同一 group 中同时存在 correct 与 incorrect rollouts。若模型太弱，group 中可能全错；若模型太强，group 中可能全对。论文会丢弃这类 group，并认为它们在 GRPO 下本来也给不出 group-relative advantage。但这仍意味着 RLCSD 的有效训练信号依赖模型处于“有对有错”的能力区间。

3. 负样本 hint 的质量仍是潜在瓶颈。论文用 $K$-marginalization 缓解错误解 error modes 过于分散的问题，但这只是近似。如果 negative rollouts 的错误类型高度异质，或者错误解本身包含大量偶然正确的中间步骤，contrastive signal 仍可能带噪。

4. 计算成本高于 GRPO 和 RLSD。Appendix D 显示 RLCSD 的 teacher log prob time 为 14.210 s/step，高于 RLSD 的 9.485 s/step；total step time 为 891.55 s/step，高于 GRPO 的 759.01 和 RLSD 的 812.84。虽然它仍快于 dense-distillation baselines，但 contrastive hints 确实引入额外 teacher forwards。

5. 方法不直接适用于 cross-model OPD。论文在 Section 4.5 指出，cross-model OPD 中也存在 task-bearing vs surface-style 的分解，但由于 OPD 没有 privileged hint，RLCSD 的 correct/wrong hint contrast 不能直接应用。作者只提出 per-token KL ranking 可作为 teacher selection 的 training-free diagnostic。

6. 评估任务和模型仍有限。实验覆盖 Qwen3 三个规模和 Olmo-3-7B，但任务主要是数学与逻辑谜题，且都具备明确答案验证。论文没有证明 RLCSD 在代码生成、复杂 agent tasks、多轮交互、自然语言开放问答或 preference-based alignment 中同样有效。

7. 超参数和训练设置可能影响结论。RLCSD 依赖 $K$, $\tau$, $\lambda$, $\delta$, $\eta$、teacher refresh interval、mask ratio 等设计。Appendix 还指出 logical reasoning 中 RLCSD 使用较高 learning rate，而其他方法在该学习率下更易 collapse，因此使用较低 learning rate；这虽有实践合理性，但也提示比较结果与调参策略有关。

#### 我的分析：优点

1. 问题诊断清楚。论文没有只提出一个新 loss，而是先定义 privilege-induced style drift，并用 token-level statistics 证明 one-sided OPSD signal 确实更多落在 style tokens 上。这比单纯报告 benchmark 提升更有说服力。

2. 方法简单且针对性强。RLCSD 的核心是 correct hint 与 wrong hint 的对比相减，本质上是去掉 hint conditioning 共同诱发的 style component。这个设计与诊断高度对应，不是任意堆叠技巧。

3. 保留了 GRPO 的稳定性。论文没有让 noisy token-level signal 直接决定更新方向，而是用 $A_{\mathrm{ORM}}$ anchoring 和 sign-preserving clamp 保证 verifier 仍控制 direction。这一点很重要，因为 token-level distillation signal 即使经过 contrast 也不可能完全无噪。

4. 对关键组件做了充分消融。K-marginal negative hints、排除 self-rollout、$A_{\mathrm{ORM}}$ anchoring、two-path aggregation 都有单独 ablation，且退化方向符合方法解释，增强了因果可信度。

5. 不只对 RLCSD 有效。论文把 contrastive hints 插入 OPSD 和 RLSD，显示它是 general-purpose component。这使贡献从“一个新算法”上升为“如何净化 privileged token-level signal 的原则”。

6. 实验覆盖性能、稳定性和效率。主结果展示最终 accuracy；training dynamics 展示 entropy、length、reward、validation curves；Appendix D 展示 step time。这使读者能同时判断效果、稳定性和实用成本。

7. 对 OPD 的 broader insight 有价值。Section 4.5 把 self-distillation 中的 style drift 推广到 cross-model OPD，指出强 teacher 如果风格距离过大，KL signal 可能主要传递 register 而非 capability。这对 teacher selection 很有启发。

#### 我的分析：缺点 / 风险

1. 对 verifiable tasks 的依赖很强。RLCSD 当前最适合 math、logic 这类可自动验证任务。若没有可靠 verifier，就无法稳定构造 correct/incorrect rollout subsets，也无法得到可信的 $A_{\mathrm{ORM}}$ anchor。

2. “style token vs task token”的划分可能过于粗糙。论文在数学任务中用规则把数字、运算符、LaTeX command 等定义为 task tokens，把 discourse markers 定义为 style tokens。这能支持诊断，但真实 reasoning 中，一些自然语言 token 也可能承载关键推理信息；一些数学符号也可能只是格式噪声。

3. contrastive cancellation 的假设不一定总成立。RLCSD 假设 correct hint 和 wrong hint 在相同 template 下共享类似 style drift，因此相减可以抵消风格项。但如果模型对正确参考和错误参考的反应风格本身不同，或者错误参考诱导出特殊 artefacts，则 subtraction 可能残留非任务成分。

4. negative hints 可能引入错误知识污染。论文故意把 wrong hint 也放在同样的 “Correct final answer” framing 下，以保证 template 对称。这有利于 style cancellation，但也意味着 teacher 被要求认真条件化在错误参考上。若错误参考非常迷惑，负分支可能产生复杂且不可控的分布变化。

5. 对模型能力区间敏感。方法需要 group 内同时有正例和负例。训练早期模型太弱时，有效 groups 可能不足；训练后期模型太强时，全对 groups 增多，也可能减少 contrastive signal 的覆盖。

6. 超参数和实现复杂度不低。相比 GRPO，RLCSD 需要维护 teacher snapshot、构造 sibling hints、做 multiple teacher logprob forwards、mask token、two-path aggregation。虽然比 dense OPSD 便宜，但工程复杂度明显高于普通 RLVR。

7. 统计显著性还不够清楚。论文给出大量 benchmark 数字，但未见多随机种子、置信区间或显著性检验。部分 math gain 较小，例如 Olmo math Avg. 79.1 vs GRPO 79.0，这类差异是否稳定仍需更多重复实验确认。

8. 与 baseline 的学习率设置存在解释风险。Appendix 说明 logical reasoning 中 RLCSD 用 $5 \times 10^{-6}$，其他方法因更易 collapse 使用 $1 \times 10^{-6}$。这可以被理解为 RLCSD 更稳定、能承受更大学习率；但也可能让部分读者担心 baseline tuning 是否充分公平。

### 4. InternVideo3: Agentify Foundation Models with Multimodal Contextual Reasoning

- arXiv: https://arxiv.org/abs/2606.12195
- PDF: https://arxiv.org/pdf/2606.12195
- Authors: Ziang Yan, Sheng Xia, Jiashuo Yu, Yue Wu, Tianxiang Jiang, Songze Li, Kanghui Tian, Yicheng Xu, Yinan He, Kai Chen, Limin Wang, Yu Qiao, Yi Wang
- 作者单位：Shanghai AI Laboratory; Shanghai Innovation Institute; Nanjing University
- arXiv published: 2026-06-10
- arXiv updated: 2026-06-10
- Categories: cs.CV

#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy distillation。

#### Abstract 原文

> Recent progress in foundation models has shifted toward agentic behavior involving multi-step reasoning and tool use. However, open-source efforts largely focus on text-dominant settings, leaving long-horizon multimodal tasks underexplored. This gap is evident in video tasks requiring sustained temporal understanding and iterative interaction. We present InternVideo3, a framework enhancing these capabilities via Multimodal Contextual Reasoning (MCR). MCR treats understanding as a closed-loop process over a shared, evolving context containing observations, instructions, reasoning, tool actions, and memory. This frames long-video understanding as evidence accumulation and verification. To ensure efficiency, we introduce Multimodal Multi-head Latent Attention (M^2LA), a token-preserving reparameterization compressing KV-cache states while retaining the full token stream. Our staged training includes continued pretraining, short-to-long supervised fine-tuning, rule-based reinforcement learning, and on-policy distillation. Experiments show InternVideo3 achieves strong performance on benchmarks like Video-MME, MLVU, and EgoSchema. We further instantiate the model as a video agent with retrieval tools, demonstrating robust evidence-grounded behavior. Our results suggest that efficient context handling and closed-loop reasoning are vital for adapting open multimodal models toward long-horizon visually grounded agency.

#### 摘要中文翻译

基础模型（foundation models）的最新进展正从一次性预测（one-shot prediction）转向具备多步推理（multi-step reasoning）和工具使用（tool use）的智能体行为（agentic behavior）。然而，开源方向主要集中在以文本为主的场景中，使得长时程多模态任务（long-horizon multimodal tasks）仍未得到充分探索。这个缺口在视频任务中尤其明显，因为视频任务通常需要持续的时间理解（sustained temporal understanding）和迭代式交互（iterative interaction）。本文提出 InternVideo3，这是一个通过 Multimodal Contextual Reasoning（MCR，多模态上下文推理）增强这些能力的框架。MCR 将理解视为一个围绕共享且不断演化的上下文（shared, evolving context）的闭环过程，该上下文包含观察、指令、推理、工具动作和记忆。这将长视频理解建模为证据积累（evidence accumulation）与验证（verification）的过程。为保证效率，本文提出 Multimodal Multi-head Latent Attention（$M^2LA$），一种 token-preserving reparameterization（保留 token 流的重参数化方法），它在保留完整 token stream 的同时压缩 KV-cache 状态。本文的分阶段训练包括 continued pretraining（持续预训练）、short-to-long supervised fine-tuning（从短到长的监督微调）、rule-based reinforcement learning（基于规则的强化学习）和 on-policy distillation（在线策略蒸馏）。实验表明，InternVideo3 在 Video-MME、MLVU 和 EgoSchema 等 benchmark 上取得强表现。作者进一步将该模型实例化为带有检索工具的视频智能体（video agent），展示了稳健的、基于证据的行为。结果表明，高效上下文处理和闭环推理对于将开源多模态模型适配到长时程、视觉 grounded agency（视觉接地智能体能力）非常关键。

#### 研究背景

这篇论文的核心背景是：基础模型的能力评估正在从“能否回答一个问题”转向“能否在长时程任务中维护状态、调用工具、反复验证并更新判断”。在文本场景中，agentic AI 已经围绕 coding agents、browser/search agents、function calling 和 long-context language tasks 快速发展；但在视频等多模态场景中，开源模型仍多停留在 single-pass QA（单次问答）范式。

作者认为，长视频任务天然需要更强的 long-horizon reasoning（长时程推理）：模型不仅要看见内容，还要知道哪些证据已经观察到、哪些信息仍不确定、下一步该看哪个时间片段或调用哪个工具，以及当前答案是否有足够证据支持。现实应用包括 long-form video analysis（长视频分析）、instructional understanding（教学/说明视频理解）、surveillance review（监控回放）、egocentric perception（第一视角理解）和 evidence-grounded temporal reasoning（证据接地的时间推理）。

现有 multimodal large language models（MLLMs）在图像理解和短视频 QA 上已有明显进展，但在以下能力上仍不足：持续时间理解、证据定位、空间推理、迭代证据收集、外部工具或记忆交互。作者因此提出将视频理解从“静态输入到答案”的映射，改写为 Multimodal Contextual Reasoning（MCR）：模型在共享上下文中持续记录观察、推理、工具动作、反馈和记忆，并在闭环中更新自己的 belief state（信念状态）。

这篇论文还面对一个工程瓶颈：长视频和 agentic rollout 会产生巨大的 KV-cache。常见方法是 frame subsampling（帧采样）、retrieval（检索）或 summarization（摘要压缩），但这些方法可能丢失后续验证所需信息。InternVideo3 选择互补路线：尽量保留完整 multimodal token stream，同时通过 $M^2LA$ 压缩 attention 内部缓存状态，以降低长上下文推理成本。

#### Method

InternVideo3 由三个关键部分组成：MCR 推理框架、$M^2LA$ 长上下文高效 attention，以及分阶段训练 recipe。

第一，Multimodal Contextual Reasoning（MCR）把长视频理解建模为闭环过程。给定用户 query $Q$ 和初始多模态观察 $V_{\mathrm{init}}$，模型在第 $i$ 步基于当前视觉证据、动作轨迹、工具反馈和上下文产生中间推理状态：

$$
R_i \sim f_\theta(Q \mid V_i, A_i, F_i, C_i), \quad i = 1,2,\ldots,N.
$$

随后模型选择下一步动作、调用工具或请求新的感知证据：

$$
A_{i+1} \sim \rho(A_i \mid R_i, F_i, Q),
$$

$$
F_{i+1} \sim \pi(A_{i+1} \mid R_i, Q),
$$

$$
V_{i+1} \sim \delta(V_i \mid C_{i+1}).
$$

新产生的 reasoning、action 和 feedback 会被追加进上下文：

$$
C_{i+1} = C_i \oplus (R_i, A_{i+1}, F_{i+1}),
$$

$$
C_1 = T_{\mathrm{sys}}, \quad (V_1,A_1,F_1) = (V_{\mathrm{init}}, \varnothing, \varnothing).
$$

这里，$\rho$ 是动作选择 policy，$\pi$ 是环境或工具执行函数，$\delta$ 是感知更新算子，$\oplus$ 表示上下文聚合。MCR 不试图训练一个完整的 action-conditioned world simulator（动作条件世界模拟器），而是把 MLLM 的上下文当作 task-grounded belief state（任务接地信念状态），用于记录模型看到了什么、推断了什么、采取了什么动作、还存在哪些不确定性。

第二，$M^2LA$ 解决长上下文 rollout 的 KV-cache 成本。标准 GQA-style attention 需要为每个 token 存储较完整的 per-head keys 和 values。$M^2LA$ 改为为每个 token 缓存 compact latent vector（紧凑潜变量），并在 attention 计算时动态恢复 head-specific keys 和 values。这样做的目标不是删除 token，而是在保留完整 multimodal token stream 的同时压缩缓存状态。

$M^2LA$ 包含几个设计点：RoPE-aware positional aggregation（面向 RoPE 的位置聚合）用于避免压缩 key 时破坏 rotary positional embedding 的几何结构；low-rank latent factorization（低秩潜变量分解）用于将 content keys/values 投影到低维 latent space；modality-aware latent adaptation（模态感知潜变量适配）为文本和视觉 token 使用轻量 adapter，以处理两种模态分布差异；dynamic latent budgeting（动态潜变量预算）则允许按层、按模态调整 latent rank，在 memory 和 accuracy 之间折中。

第三，作者还处理了 Qwen3-style head-wise QK-Norm 与共享 latent aggregation 的兼容问题。论文指出，对于某个 head $h$，RMSNorm 可以写作：

$$
\mathrm{RMSNorm}(x_h) = \frac{x_h}{\sigma(x_h)}.
$$

如果 $\sigma(x_h)$ 在高维 pretrained model 中高度集中在均值 $\mu$ 附近，则可一阶近似为：

$$
\mathrm{RMSNorm}(x_h) \approx \frac{1}{\mu}x_h.
$$

因此，head-wise RMS normalization 可近似为常数线性缩放，从而恢复 shared latent reparameterization 的可行性。作者通过 calibration set 验证了这种近似，并使用 Global-Norm-Linear approximation 进行 M2LA conversion。

第四，训练 recipe 分为四步。首先，在 M2LA conversion 后进行 continued pretraining（CPT），用 16M multimodal samples、约 13.5B tokens 来恢复语言能力和多模态对齐。其次，进行 short-to-long SFT：Stage 1 使用 2 fps、最多 512 frames、约 32K tokens；Stage 2 提升到 4 fps、最多 2048 frames、最高 256K tokens。第三，使用 rule-based group sequence policy optimization（GSPO）做强化学习，任务包括 video QA 和 temporal grounding，奖励来自答案正确性或 temporal IoU。第四，进行 video on-policy distillation，教师模型为 Qwen3-235B，学生在自己的 sampled trajectories 上接受教师分布监督，目标为 reverse-KL：

$$
\mathcal{L}_{\mathrm{OPD}} =
\mathbb{E}_{Y \sim \pi_\theta(\cdot \mid V,Q)}
\left[
\frac{1}{T}
\sum_{t=1}^{T}
\left(
\log \pi_\theta(y_t \mid V,Q,Y_{<t})
-
\log \pi_{\mathrm{teacher}}(y_t \mid V,Q,Y_{<t})
\right)
\right].
$$

最后，作者将 InternVideo3 实例化为 video agent。该 agent 构建 hierarchical memory（分层记忆），包含 sampled frames、scene boundaries、clip-level captions、timestamps、subtitle/OCR signals 等；根据问题类型做 question routing；可调用 video_segmentation、ASR、web_search、temporal_grounding，以及 summarize、verify、answer 等内部动作；在最终回答前做 recursive verification（递归验证），检查答案是否有充分证据支持，若不足则继续检索或重新感知。

#### 论文自我评价的缺陷

论文明确列出了 limitations，并且讨论得比较具体。

1. open-weight frontier 演化太快。作者承认，在项目进行过程中，开源多模态生态快速发展，更新模型家族如 Qwen3.5/3.6/3.7、GLM、Kimi、StepFun、DeepSeek、MiniMax 等已经在不少 benchmark 上超过 InternVideo3，并且很多新模型已有更强 native long-context architectures。因此本文不应被理解为宣称 state-of-the-art multimodal agency 或整体 frontier open-model performance。

2. M2LA 是一条特定效率路径，不一定适合所有新模型。本文的长上下文路线是把 pretrained GQA backbone 转换为 latent-attention form。这个路线在项目开始时实用，但对已经原生采用 MLA、linear attention 或 hierarchical compressed attention 的新模型不一定直接适用。作者认为更一般的结论是：long-horizon multimodal reasoning 需要 efficient context handling。

3. MCR 的概念新颖性有限。closed-loop reasoning、tool use、memory、self-correction、iterative evidence gathering 等思想已经广泛出现在 agent 论文和系统综述中。本文贡献更多是把这些要素整理成一个适用于 long-horizon multimodal reasoning，尤其是 video-centric settings 的清晰框架，而不是从零发明这些概念。

4. MCR 不是完整的 predictive world model。作者将 MCR 定位为 contextual belief-state modeling，而不是完整的 action-conditioned environment simulator。因此它适合多模态推理任务，但不能像 predictive world model 那样显式建模未来环境动态。未来方向是将 contextual reasoning 与 learned predictive models 更紧密结合。

5. agent 行为依赖外部工具，存在系统脆弱性。ASR、temporal grounding、segmentation、retrieval、search 等工具的错误会被写入共享上下文，并可能传播到后续推理。当前工具接口较轻量，也没有与 base model 联合优化。作者建议未来研究 tighter model-tool integration、uncertainty-aware tool use 和更好的 tool-derived evidence calibration。

6. 当前评估范围仍有限。实验主要集中在 long-video understanding、short-video reasoning、spatiotemporal intelligence 和一个 video-agent setup，没有广泛评估 GUI agents、browser tasks、mobile agents、embodied robotics 或 persistent multi-session assistants。因此论文不能证明 MCR 已经泛化到所有 multimodal agent 场景。

7. 公共 benchmark 对 multimodal agency 的测量不足。作者指出，当前公开评测仍主要是 direct QA，不能充分衡量 recursive evidence gathering、memory management、verification quality 和 closed-loop decision making。Video-MME 上 agentic inference 有 +2.7 提升，但其他 benchmark 并不稳定，因此作者只把它作为 proof of concept。

#### 我的分析：优点

1. 论文的问题定义比较清楚：它没有只把长视频理解视为“塞更多帧进上下文”，而是强调 evolving context、belief update、tool feedback 和 verification。这比单纯扩大 context window 更接近实际视频 agent 的需求。

2. MCR 虽然不是全新概念，但作为视频多模态场景的统一抽象是有价值的。它把 observation、reasoning trace、tool action、feedback、memory 都放入同一上下文，使 long-video QA、temporal grounding、agentic inference 可以被放在一个统一框架下讨论。

3. $M^2LA$ 的动机很实际。长视频 agent 不仅 prefill 长，decode 也长，因为中间推理、工具输出和 memory summary 会不断累积。相比直接丢帧或摘要压缩，压缩 KV-cache 但保留 token stream 的思路更适合 evidence-grounded verification。

4. 训练 recipe 比较完整：M2LA conversion 后先 CPT 恢复能力，再做 short-to-long SFT，之后用 rule-based RL 优化可验证任务，最后用 on-policy distillation 从强 teacher 转移复杂推理行为。这个 pipeline 与论文目标一致，不是单点技巧。

5. 数据构造与任务目标匹配。长视频 SFT 数据平均 15.8 分钟，总量约 100K 小时，并通过 hierarchical narrative annotation 同时保留局部 clip-level evidence 和全局 temporal coherence，这对长视频理解比普通短视频 QA 数据更合理。

6. 实验覆盖面较广。论文不仅报告 long-video benchmark，还报告 short-video benchmark、temporal grounding、spatial intelligence、efficiency、ablation 和 preliminary agentic inference。尤其 temporal grounding 和 spatial intelligence 对“答案是否有证据支持”更有诊断意义。

7. 结果不是只靠一个 benchmark。InternVideo3 在 Video-MME、MLVU、VRBench、EgoSchema、PerceptionTest、TOMATO、QVHighlights、Charades-STA、ANet、VSIBench 等多个任务上都有强表现，说明提升不完全是单一数据集过拟合。

8. 作者对负面结果比较诚实。例如 VideoMME-v2 略低于 Qwen3-VL-8B，LVBench 低于 Qwen3-VL-8B，MVBench 和 MotionBench 未超过 InternVideo2.5，TempCompass 不是最强；agentic inference 也承认只在 Video-MME 上稳定展示提升，没有夸大为普适结论。

9. 效率实验有实际价值。M2LA 在长 context decode 中达到 1.84 倍到 5.01 倍 throughput 提升，并使原始模型 OOM 的 512K+ context 可执行，这直接支撑长视频和 agentic rollout 的可部署性。

#### 我的分析：缺点 / 风险

1. MCR 的概念贡献偏框架化，真正技术增量主要来自 $M^2LA$、数据和训练 recipe。closed-loop reasoning、tool use、memory、verification 都是 agent 文献中已有要素，论文的新意更多是组合与视频化落地。

2. agentic evaluation 仍然偏弱。论文展示了 qualitative demos 和 Video-MME 上 Direct QA 73.1 到 Agentic inference 75.8 的初步提升，但没有大规模、系统化的 agent benchmark，也没有严格比较不同工具调用策略、交互预算和失败模式。

3. 工具依赖可能带来 error amplification。MCR 将 tool output 写入共享上下文，如果 ASR、temporal grounding 或 retrieval 错误，模型后续可能把错误当作证据继续推理。论文提出 recursive verification，但没有充分量化 verification 对错误传播的抑制能力。

4. on-policy distillation 依赖强 teacher Qwen3-235B，开源复现成本可能较高。虽然学生模型是 7/8B 级别，但数据筛选和教师评估需要强大 teacher，实际复现 InternVideo3 的完整 pipeline 可能不容易。

5. 数据合成质量可能影响结论。长视频 SFT 依赖 teacher 生成 clip captions、hierarchical narratives 和 QA pairs。若 teacher 对长视频的局部细节、实体一致性或跨场景关系存在错误，这些错误可能进入训练数据。论文虽然强调筛选和层次化标注，但对 annotation noise 的定量分析不充分。

6. 部分 benchmark 提升可能来自数据覆盖和 post-training mixture，而不完全来自 MCR。消融显示 CPT、Long Ctx.、LV Data 都重要，但 MCR 本身作为推理框架与数据/架构贡献之间的因果分离还不够清楚。

7. $M^2LA$ 的适用性可能被新架构削弱。作者也承认，若新模型已经原生使用 MLA、linear attention 或 hierarchical compressed attention，那么 GQA-to-M2LA conversion 的工程价值会下降。长期价值可能更多是“上下文效率原则”，而不是具体实现。

8. 论文仍主要在 video-centric setting 中验证。它声称 long-horizon visually grounded agency 是更广泛方向，但没有覆盖 GUI、browser、mobile、robotics、multi-session assistants 等复杂智能体环境，因此不能直接推断 MCR 在这些任务中有效。

9. benchmark 与真实 agent 能力之间存在 gap。许多公开视频 benchmark 仍是 multiple-choice 或 direct QA，即使模型得分高，也不代表它真的具备稳定的 evidence gathering、memory management、self-correction 和 tool-use planning 能力。论文对此有认识，但实验上还没有完全解决。

10. “超过 human score”这类结果需要谨慎。EgoSchema 上 76.6 略高于 reported human score 76，但单个 benchmark 的 human comparison 容易受评测协议、数据泄漏、题型偏差和样本代表性影响，不应过度解读为模型在真实长视频理解上超过人类。

### 5. HERO: Hindsight-Enhanced Reflection from Environment Observations for Agentic Self-Distillation

- arXiv: https://arxiv.org/abs/2606.11559
- PDF: https://arxiv.org/pdf/2606.11559
- Authors: Haoran Liu, Yuwei Zhang, Xiyao Li, Bohan Lyu, Jingbo Shang
- 作者单位：University of California, San Diego; Independent Researcher; University of California, Berkeley.
- arXiv published: 2026-06-10
- arXiv updated: 2026-06-10
- Categories: cs.AI

#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy self-distillation。

#### Abstract 原文

> Reinforcement learning typically improves multi-turn agent capabilities through the terminal outcome of the trajectories, which makes it difficult to determine credit assignments for each intermediate turns. Recent on-policy self-distillation methods offer a promising alternative by converting privileged feedback into dense token-level supervision through a self-teacher. Our study is motivated by the unexpected performance degradation observed when naively extending this paradigm to multi-turn settings, which we attribute to a lack of alignment between privileged feedback, such as successful trajectories or terminal outcomes, and the student's current decision context. We introduce HERO, a hindsight-enhanced self-distillation framework that uses next environment observations as locally aligned feedback. After each rollout, HERO reflects on the completed interaction to convert each observation into a compact turn-level diagnosis, that captures actionable feedback about the original action such as its necessity, validity or failure cause. On TauBench and WebShop, HERO improves task success and reduces unnecessary turns over environment-feedback-only self-distillation and GRPO. It is especially effective under limited training turn budgets, where successful rollouts are rare and GRPO provides weak reward-contrast signals.

#### 摘要中文翻译

强化学习通常通过轨迹的 terminal outcome（终局结果）来提升 multi-turn agent（多轮智能体）能力，这使得很难为每个 intermediate turn（中间轮次）进行 credit assignment（信用分配）。近期的 on-policy self-distillation（在线策略自蒸馏）方法提供了一种有前景的替代方案：通过 self-teacher（自教师）把 privileged feedback（特权反馈）转换为 dense token-level supervision（密集 token 级监督）。本文的动机来自一个意外现象：当把这一范式朴素扩展到 multi-turn 设置时，性能反而下降。作者认为原因在于 privileged feedback，例如 successful trajectories（成功轨迹）或 terminal outcomes，与 student（学生模型）当前 decision context（决策上下文）缺乏对齐。作者提出 HERO，即 Hindsight-Enhanced Reflection from Environment Observations，一个 hindsight-enhanced self-distillation（后见增强自蒸馏）框架，它使用 next environment observations（下一步环境观测）作为 locally aligned feedback（局部对齐反馈）。每次 rollout 后，HERO 会反思已完成的交互，把每个 observation 转换成紧凑的 turn-level diagnosis（轮次级诊断），捕捉关于原始 action 的可操作反馈，例如其必要性、有效性或失败原因。在 TauBench 和 WebShop 上，HERO 相比 environment-feedback-only self-distillation 和 GRPO 提升了 task success，并减少了不必要的 turns。它在训练 turn budget 受限时尤其有效，因为此时 successful rollouts 很少，GRPO 能获得的 reward-contrast signals 较弱。

#### 研究背景

- 多轮 tool-use agent 的 post-training 难点在于：最终 reward 往往只在完整 trajectory 结束后给出，但中间每个 tool call 或 action 的贡献并不清楚，导致 credit assignment 很弱。
- PPO / GRPO 等 outcome-based RL 方法可以利用自动 verifier 或 judge model 的 scalar reward，但在 long-horizon interaction 中容易出现两个问题：
  - 成功轨迹稀少，尤其在 strict turn budget 下，大多数 rollouts 都失败，GRPO 的 group-relative advantage 可能几乎为零。
  - 即便最终成功，trajectory 中也可能包含 redundant tool calls（冗余工具调用），scalar reward 难以指出哪一步多余或错误。
- SDPO、OPSD 等 on-policy self-distillation 方法在 single-turn setting 中有效，因为 privileged feedback 与当前决策通常天然对齐；但在 multi-turn agent 任务中，如果 teacher 直接看到完整 successful trajectory 或 corrected demonstration，早期决策会接触 student 当时不可见的 future actions / outcomes，造成 teacher-student context mismatch。
- 论文的核心观察是：multi-turn setting 中最自然、局部对齐的反馈不是完整未来轨迹，而是每个 action 之后立即返回的 next environment observation。HERO 进一步用 hindsight reflection 把这个 observation 和完整交互结果压缩成局部可用的 diagnosis，从而提供 dense token-level supervision。

#### Method

HERO 面向 multi-turn tool-use 环境。给定初始任务 `$x$`，agent 最多交互 `$T$` 轮。第 `$t$` 轮的历史为：

$$
H_t = (x, y_1, o_1, \ldots, y_{t-1}, o_{t-1}),
$$

其中 `$y_i$` 是 assistant action，`$o_i$` 是 environment observation。student policy 采样：

$$
y_t \sim \pi_\theta(\cdot \mid H_t).
$$

完整 trajectory 写作：

$$
\tau = (x, y_1, o_1, \ldots, y_T, o_T).
$$

HERO 的流程分三步：

1. On-policy rollout：student 在正常 causal interaction history 下生成多轮 trajectory。
2. Hindsight reflection：rollout 完成后，reflector 读取原始任务、完整 trajectory、tool observations、turn-level rewards 和 final outcome，生成每个 assistant turn 的结构化 hint：

$$
h_{1:T} = R(x,\tau,R(\tau)),
$$

其中单个 hint 抽象为：

$$
h_t = (d_t,\hat{a}_t),
$$

`$d_t$` 是自然语言 diagnosis，`$\hat{a}_t$` 是可选的 corrected local action 或 target behavior。若没有可操作信号，则设为 empty hint。论文实现中 confidence 字段只作为文本元数据，不作为数值权重、过滤条件或优化目标。

3. Turn-level privileged self-teacher：teacher 不直接读取 raw future trajectory，而只接收当前 turn 的原始上下文、下一步 environment observation、trajectory outcome 和 reflection hint。其 hindsight-augmented context 可概括为：

$$
\tilde{H}_t = P(x, r_{<t}, o_t, R(\tau), h_t),
$$

其中 `$P$` 是固定 teacher prompt template。teacher 在这个上下文下重新评估 student 原始 action tokens，而不是直接生成并监督一个新的 response。

对 assistant response `$y_t = (y_{t,1}, \ldots, y_{t,|y_t|})$`，第 `$k$` 个 token 的 student 与 self-teacher 分布为：

$$
p^{\mathrm{stu}}_{t,k} = \pi_\theta(\cdot \mid H_t, y_{t,<k}),
$$

$$
p^{\mathrm{tea}}_{t,k} =
\mathrm{sg}\left[
\bar{\pi}_\theta(\cdot \mid \tilde{H}_t, y_{t,<k})
\right],
$$

其中 `$\mathrm{sg}$` 表示 stop-gradient，`$\bar{\pi}_\theta$` 可以是当前模型、delayed copy 或 EMA teacher。

HERO 使用 Jensen-Shannon divergence 作为 distributional distance。若：

$$
m_t = \frac{1}{2}(p_t + q_t),
$$

则：

$$
D_{\mathrm{JSD}}(p_t,q_t)
=
\frac{1}{2}D_{\mathrm{KL}}(p_t \Vert m_t)
+
\frac{1}{2}D_{\mathrm{KL}}(q_t \Vert m_t).
$$

最终 HERO objective 对有非空、可解析 reflection hint 的 assistant turns 计算 token-level self-distillation loss：

$$
\mathcal{L}_{\mathrm{HERO}}(\theta)
=
\mathbb{E}_{\tau \sim \pi_\theta}
\left[
\frac{1}{|I(\tau)|}
\sum_{t \in I(\tau)}
\ell_t
\right],
$$

$$
\ell_t
=
\frac{1}{|y_t|}
\sum_{k=1}^{|y_t|}
D_{\mathrm{JSD}}
\left(
p^{\mathrm{stu}}_{t,k},
p^{\mathrm{tea}}_{t,k}
\right).
$$

关键设计点是：HERO 可以从 failed trajectories 中学习。若 GRPO 的一个 group 内所有 rollouts 都失败，则每条 trajectory 的 reward 都相同，group-relative advantage 为零，基本没有学习信号；但 HERO 只要某些 turn 的 reflection hint 非空，就能产生非零 token-level supervision，例如纠正 tool-call schema、避免 fabricated arguments、减少 repeated API calls。

#### 论文自我评价的缺陷

- 论文明确指出，HERO 依赖模型自身对 trajectory 的 reflection 能力。它最适合那些 hindsight 中容易识别的 agentic errors，例如 malformed tool calls、over-specified arguments、repeated API calls、incorrect interaction patterns。对于复杂数学推理或深层 multi-step problem solving，如果模型自己无法诊断根因，HERO 的机制可能不足。
- HERO 依赖 instruction-following 和 in-context learning 能力。self-teacher 必须正确理解 reflection hint，并把它转化为 token-level preference changes。因此 HERO 更适合 instruction-tuned models，而不一定适合 base models；论文实验也显示 larger / stronger instruct models 受益更多。
- Appendix 的 reflection quality analysis 显示 reflection 本身并不完美。Self-reflector 只有 58.9% hints 被标为 Correct，另有 17.9% Incorrect、13.8% Vacuous / non-actionable。即使 GPT-4o reflector 也仍有 12.0% Incorrect 和 12.4% Vacuous / non-actionable。
- 作者总结了三个 recurrent failure modes：hindsight leakage，即 reflection 使用了 later turns 才可见的信息；recency bias，即把失败归因给最近一轮而不是更早的 forking point；over-confident corrections，即提出看似合理但缺乏当前状态支持的 next action。
- HERO 虽然 wall-clock 上比 GRPO 省，因为只需单 rollout，但仍引入额外 reflection step 和 teacher forward pass。若真实环境 API 很昂贵、reflection prompt 很长，或 teacher logits 计算成本高，实际成本优势需要重新评估。

#### 我的分析：优点

- 核心问题定义清楚：论文没有简单说“dense feedback 更好”，而是指出 multi-turn self-distillation 的关键是 local alignment。这个切入点比直接把 SDPO / OPSD 扩展到完整 trajectory 更合理。
- 方法设计克制：HERO 不需要额外 value model、process reward model 或外部专家 teacher；默认用 policy 自身做 reflector 和 self-teacher，只把 hindsight 压缩成 turn-level hint，避免 teacher 直接看到完整未来轨迹。
- 对 failed trajectories 的利用很有价值：在 strict turn budget 或 sparse reward 场景下，GRPO 的 reward contrast 可能消失，而 HERO 仍能从局部诊断中获得 token-level supervision，这一点对真实 agent 训练很实用。
- 实验覆盖了 success rate、turn efficiency、OOD transfer、general capability retention、prompt ablation、single-rollout learning、wall-clock efficiency 和 reflection quality，证据链相对完整。
- Full-Demo Privileged Teacher 的负结果很重要：它直接支持论文主张，即“更多 privileged context”不一定更好，尤其当 context 与 student 当前可见信息不一致时会产生 harmful supervision。
- Case study 展示了 token-level JSD loss 如何集中到错误 tool name 和 fabricated arguments 上，使 credit assignment 的改善更可解释，而不是只报告 aggregate score。
- 对 OPD / SDPO 方向有启发：它提示 token-level filtering / calibration 不只应考虑 teacher confidence，还要考虑 feedback 是否与 student 的 local decision context 因果对齐。

#### 我的分析：缺点 / 风险

- 主结果提升幅度在部分 benchmark 上并不大。例如 Qwen3-4B 上 TauBench-Retail 相比 GRPO 只提升 1.4 个 SR 点，TauBench-Airline 提升 1.5 个点；需要更多 seeds、置信区间或显著性检验来判断稳健性。
- Reflection hint 质量是潜在瓶颈。Self-reflector 的 incorrect 和 vacuous 比例不低，如果错误 hint 被 teacher 放大，可能导致自蒸馏强化错误归因，尤其在更复杂任务中风险更高。
- HERO 的“压缩 hindsight”仍可能泄漏未来信息。虽然 prompt 要求不 verbatim 暴露 future actions / observations，但诊断本身可能隐含 later evidence；这会削弱 teacher signal 与 student local context 的严格因果对齐。
- 论文主要评测 tool-use / shopping / customer-service 类任务，这些任务很多错误是 schema、参数、重复调用、信息收集策略等局部执行错误。对于代码生成、数学证明、科学推理等需要深层语义归因的任务，HERO 是否同样有效尚不明确。
- Teacher 与 reflector 都来自同一模型时，存在 self-confirmation bias。模型可能把自己偏好的错误策略解释为合理策略，或者给出貌似有道理但不可执行的 correction。
- 训练细节较复杂：需要 rollout、reflection、teacher distribution、EMA teacher、top-token approximation、clipped importance weighting 等组件。虽然概念上简洁，但工程实现比普通 GRPO 或 SFT 更复杂。
- 与 GRPO 的比较虽然调了 GRPO learning rate，但两个目标的 compute pattern 很不同；HERO 用 `$G=1$`、GRPO 用 `$G=8$` 是合理设定，但在不同环境交互成本、并行度或 reward density 下，效率结论可能变化。
- 如果环境 observation 本身错误、过于稀疏或不可解释，HERO 的 reflection 可能只能生成空泛诊断；论文中的 WebShop invalid action 示例说明，hint 甚至可能建议当前 clickable set 中不存在的 action。

#### 摘要中文翻译

本文介绍 Kwai Keye-VL-2.0-30B-A3B，这是一个开源的 Mixture-of-Experts (MoE，混合专家) 多模态基础模型，旨在推进长视频理解和 agentic intelligence（智能体能力）。针对小时级视频中固有的超长上下文、信息冗余和高昂计算成本问题，Keye-VL-2.0 首次将 DeepSeek Sparse Attention (DSA，DeepSeek 稀疏注意力) 适配到基于 GQA 的多模态架构中，使模型能够无损处理 256K 上下文，同时捕获关键帧和长程时间依赖。该架构由高度优化的训练与推理基础设施支撑，包括可扩展的视频 I/O、异构 ViT-LM 并行，以及定制 DSA kernels，用于显著提升吞吐并降低计算开销。

此外，为解决多任务对齐中的 catastrophic forgetting（灾难性遗忘）算法困境，本文提出 Cross-Modal Multi-Teacher On-Policy Distillation (MOPD，跨模态多教师在线策略蒸馏)，并结合 Context-RL 和 Video-RL。通过将来自 on-policy rollouts（在线策略采样轨迹）的 dense token-level teacher feedback（密集 token 级教师反馈）蒸馏回 MoE backbone，而该模型每次仅激活 3B 参数，Keye-VL-2.0 原生支持 Code、Tool 和 Search 场景下的高级 agent collaboration（智能体协作）和多模态 self-correction（自我纠错）。

在视频理解、temporal grounding（时间定位）、推理、STEM 和 agent benchmarks 上的大量评测表明，Keye-VL-2.0-30B-A3B 在同规模模型中达到 state-of-the-art 表现，尤其在 TimeLens 的细粒度时间定位，以及 Video-MME-v2 和 LongVideoBench 的长视频理解上表现突出。作者发布模型 checkpoints，以推动社区向可扩展、鲁棒的多模态智能体应用发展。

#### 研究背景

这篇技术报告的核心问题是：如何把多模态大模型从“短视频/图像理解”推进到“小时级长视频理解 + 多步 agentic task execution”，同时避免成本爆炸和能力互相干扰。

论文认为当前 MLLM 面临两类关键瓶颈：

1. 长上下文视频的基础设施瓶颈
   标准 dense attention（稠密注意力）在序列长度上具有二次复杂度，面对 128K/256K 级别多模态上下文时，KV cache 和计算成本快速膨胀。传统做法往往依赖 aggressive frame subsampling（激进抽帧），但这会损失 temporal continuity（时间连续性）和关键事件证据。

2. 多任务 post-training 的算法冲突
   直接混合视频理解、OCR、STEM、代码、工具调用、搜索、长上下文等任务，容易造成 modality conflict（模态冲突）和 catastrophic forgetting。例如，强化 agent 工具调用能力可能让模型过度输出工具格式；强化 reasoning RL 又可能让模型在简单感知任务中过度冗长。

Keye-VL-2.0 的定位是一个开源 30B-class MoE 多模态基础模型，总参数约 30B、每次仅激活 3B 参数。它继承 Keye-VL 和 Keye-VL-1.5 的视觉语言基础，同时在两个方向上扩展：

- 架构上：将 DeepSeek Sparse Attention (DSA) 引入 GQA-based multimodal backbone，实现 256K multimodal context。
- 训练上：用 staged pre-training、RL、specialized domain experts 和 Cross-Modal MOPD，把不同能力整合回单一 MoE 模型，尽量避免互相遗忘。

#### Method

Keye-VL-2.0-30B-A3B 的方法可以分为 architecture、pre-training、post-training 和 training/inference infrastructure 四个层面。

1. Model Architecture

模型由四个核心组件组成：

- Vision Encoder (ViT)：继承自 Keye-VL-1.5-8B，基于 SigLIP-400M-384-14，用于图像和视频帧特征提取。
- Language Decoder (LLM)：基于 Qwen3-30B-A3B-Thinking-2507，提供语言、知识、指令跟随和推理能力。
- MLP Projector：在 Stage 0 初始化，用于把视觉特征映射到 LLM 表征空间。
- Sparse Attention Module：面向 GQA 的 DSA 设计，用 global MQA-style indexing 加 grouped GQA aggregation 支持长上下文。

视觉侧采用 native-resolution vision encoder（原生分辨率视觉编码器）。相比固定分辨率或 dynamic tiling（动态切块），该设计保留原图长宽比和细粒度局部信息，对 OCR、文档、图表、视频和细粒度识别尤其重要。模型还结合 adaptive position encoding、2D RoPE、NaViT-style Patch n' Pack 和 FlashAttention，以支持不同尺寸样本的高效 packing。

视频和图像采用 unified visual encoding（统一视觉编码）：视频每帧被当作一张高分辨率图像处理，并在每帧视觉 tokens 前加入自然语言 timestamp，使 LLM 在文本空间中获得显式时间锚点。视频还使用 adaptive video pixel budget，根据视频时长分配像素预算：短视频更强压缩，长视频保留更多视觉证据。

2. DSA for Long-Context Multimodal Modeling

论文的关键架构创新是把 DeepSeek Sparse Attention 适配到 GQA-based MLLM 中。传统 full attention 复杂度为 $O(L^2)$，而 Keye-VL-2.0 设置 top-k 为 $k = 2048$，将核心注意力复杂度降为 $O(Lk)$，其中 $k \\ll L$。

DSA 包含两个主要部分：

- MQA-style Lightning Indexer：用共享 key head 计算当前 query token 与历史 tokens 的全局索引分数 $I_{t,s}$，选择 Top-k tokens 形成稀疏集合 $\\Omega_t$。
- GQA Sparse Aggregation：GQA 中多个 query heads 分成 $G$ 个 group，每组共享 group-specific KV head，并使用相同的 $\\Omega_t$ 做稀疏聚合。

核心公式包括全局索引分数：

$$
I_{t,s} = \\sum_{j=1}^{H_I} w^I_{t,j} \\cdot \\mathrm{ReLU}(q^I_{t,j} \\cdot k^I_s)
$$

Top-k 稀疏索引集合：

$$
\\Omega_t = \\{s \\mid I_{t,s} \\in \\mathrm{Top\\text{-}k}(I_{t,:})\\}
$$

第 $g$ 个 GQA group 的 sparse attention 输出：

$$
u_{t,g} = \\mathrm{Attn}(h_{t,g}, \\{c_{s,g} \\mid s \\in \\Omega_t\\})
$$

DSA 训练采用 dense warm-up 和 sparse adaptation 两阶段：

- dense warm-up：主模型保持 dense GQA，大部分参数冻结，让 indexer 学习覆盖原始 dense attention 分布。
- sparse adaptation：全参数解冻并切换到 sparse mode，indexer 继续与主模型注意力分布对齐，但 KL loss 只在 Top-k 集合上计算。

最终目标为：

$$
\\mathcal{L}_{\\mathrm{total}} = \\mathcal{L}_{\\mathrm{NTP}} + \\lambda \\mathcal{L}^{I}_{\\mathrm{sparse}}
$$

3. Pre-Training

预训练采用四阶段 curriculum：

- Stage 0，Projector Initialization：冻结 ViT 和 LLM，仅训练 Projector，使用 caption 和 interleaved image-text 数据，序列长度 8K，约 4B tokens。
- Stage 1，General Multimodal Pre-training：全参数训练，序列长度 32K，约 1T tokens，覆盖 caption、OCR、video mix、image mix、pure-text QA。
- Stage 2，Multi-Task Capability Injection：序列长度扩展到 64K，约 2T tokens，注入 OCR、STEM、GUI、grounding、counting、e-commerce、coding、tool-use/search 等能力。
- Stage 3，Long-Context Extension：序列长度扩展到 256K，约 500B tokens，覆盖 long video、long documents、multi-doc、long multi-image、long code、long agent trajectories。

视频预训练也分阶段扩展：Stage 1 使用 15 秒 clips，Stage 2 扩展到 15 分钟，Stage 3 扩展到 2 小时。论文特别强调 scene-wise dense caption（按场景密集字幕）和 diverse temporal video grounding data，包括 Referred Action Recognition、Video Highlight Detection、Extractive Video Summarization、Temporal Event Matching 等。

4. Post-Training

post-training 包含 SFT、Synthetic CoT、RL、specialized RL、Video RL、Agentic RL 和 MOPD。

SFT 数据约 500B tokens，包含 Text NLP、Video、Perception、Reasoning、Agent、Long-context，其中约 40% 是 text-only data，用于保持语言和推理能力。Synthetic CoT 用强教师模型生成 reasoning traces，再经 query-level、response-level、process-level checks 过滤；数学任务还使用 Doubt2Clean 二次清洗。

RL 主要分为：

- Synthetic-Data RL：构造可程序验证的 difference-recognition tasks，例如图像差异定位、几何、化学式、电路图 DSL 输出。奖励可由规则自动计算。
- General RL：使用 GSPO，在 VQA、STEM、chart、math、logic、text-only QA 等可验证任务上提升可靠性。
- Specialized RL：训练 grounding、spatial、math、counting、OCR 等 domain experts。
- Video RL：约 31K video samples，冻结 ViT 和 projector，用 GSPO 优化 temporal alignment、event tracking 和 long-range aggregation。
- Agentic RL：覆盖 Coding RL、Tool Use RL、Search RL，用环境执行结果、工具状态、测试结果或最终答案正确性作为 reward。

GSPO 的 sequence-level ratio 定义为：

$$
s_i(\\theta) = \\exp \\left(
\\frac{1}{|y_i|} \\sum_{t=1}^{|y_i|}
\\log \\frac{\\pi_\\theta(y_{i,t} \\mid x, y_{i,<t})}
{\\pi_{\\theta_{\\mathrm{old}}}(y_{i,t} \\mid x, y_{i,<t})}
\\right)
$$

advantage 标准化为：

$$
\\hat{A}_i =
\\frac{
R(x, y_i) - \\mathrm{mean}(\\{R(x, y_j)\\}_{j=1}^{G})
}{
\\mathrm{std}(\\{R(x, y_j)\\}_{j=1}^{G}) + \\delta
}
$$

5. Cross-Modal Multi-Teacher On-Policy Distillation

MOPD 是全文最重要的 post-training consolidation 方法。论文维护 13 个 RL-trained domain teachers，覆盖 safety、pure-text math、instruction following、code、visual STEM、OCR、grounding、counting、video、tool use 等。每个样本根据 modality 和 task type 路由到最合适的 teacher。

学生先生成 on-policy response：

$$
y_i = (y_{i,1}, \\ldots, y_{i,T}) \\sim \\pi_\\theta(\\cdot \\mid x_i)
$$

教师在学生真实访问过的 state 上提供 token-level feedback。为避免 full-vocabulary distillation 的不稳定性，论文定义 teacher 和 student Top-k 的 overlap set：

$$
\\Omega_{i,t} =
\\mathrm{TopK}(\\pi_T^{r(i)}(\\cdot \\mid s_{i,t}))
\\cap
\\mathrm{TopK}(\\pi_\\theta(\\cdot \\mid s_{i,t}))
$$

当 $\\Omega_{i,t}$ 非空时，raw advantage 为：

$$
A_{i,t} =
\\sum_{v \\in \\Omega_{i,t}}
\\bar{\\pi}_\\theta(v \\mid s_{i,t})
\\left[
\\log \\pi_T^{r(i)}(v \\mid s_{i,t})
-
\\log \\pi_\\theta(v \\mid s_{i,t})
\\right]
$$

其中：

$$
\\bar{\\pi}_\\theta(v \\mid s_{i,t}) =
\\frac{
\\pi_\\theta(v \\mid s_{i,t})
}{
\\sum_{u \\in \\Omega_{i,t}} \\pi_\\theta(u \\mid s_{i,t})
}
$$

MOPD objective 为：

$$
\\mathcal{L}_{\\mathrm{MOPD}} =
-
\\mathbb{E}
\\left[
\\frac{1}{|M_i|}
\\sum_{t \\in M_i}
\\hat{A}_{i,t}
\\log \\pi_\\theta(y_{i,t} \\mid x_i, y_{i,<t})
\\right]
$$

此外，论文使用 Segmented Prompt-Response Re-tokenization (SPRR) 保证 teacher/student token 对齐，并使用 token-category-aware advantage scaling 降低格式 token 权重、提高感知或推理 token 权重。对 repetition collapse，则只在崩溃点之后施加 localized repetition penalty：

$$
\\hat{A}_{i,t} = \\tilde{A}_{i,t} - \\lambda_{\\mathrm{rep}} \\cdot \\mathbf{1}[t \\ge \\tau_i]
$$

6. Efficient Training and Inference Infrastructure

系统侧贡献包括：

- ExtraIO：异步视频 I/O 服务，解除视频解码和抽帧瓶颈。
- ViT-LM heterogeneous parallelism：ViT 和 LM 共置但采用不同并行切分策略，并通过 recompute/offload 降低 ViT activation memory。
- two-level load balancing：在 multimodal-token level 和 LM-sample level 同时平衡负载，端到端训练吞吐提升约 20%。
- DSA variable-length optimization：用 FlashInfer 和 TileLang 优化 DSA，相比开源 baseline 超过 $2\\times$ speedup。
- inference optimization：Chunk ViT、Top-k set deduplication、DSA decode optimization，使 128K context 下 prefill cost 约为 full attention 的 $0.32\\times$，decode cost 约为 $0.20\\times$。

#### 论文自我评价的缺陷

论文未单列 limitations，但从 conclusion and future work 以及实验呈现方式可以看出若干自我承认或隐含限制：

1. 当前仍偏 leaderboard-oriented optimization
   作者在 future work 中明确说，后续会“move beyond leaderboard-oriented optimization toward deeper deployment in real business scenarios”。这说明当前结果主要还是围绕 benchmark 证明能力，真实业务闭环、线上稳定性、长周期反馈和用户体验还不是本文充分验证的对象。

2. 真实业务落地仍是未来工作
   论文计划把 fine-grained long-video perception 和 image-text understanding 融入推荐、内容生态治理、商业投放等产品 pipeline。这意味着模型虽然在 benchmark 上表现强，但在真实业务中的收益、延迟、成本、鲁棒性和风险控制尚未被系统性报告。

3. Video-Agent workflow 仍未完成
   论文把 “Video-Agent workflows” 放在 future work：从被动理解视频，发展到自动化协调视频检索、highlight extraction、automated editing、packaging、marketing-copy generation。这说明当前模型已经展示 agentic capability，但完整视频生产链路中的自主编排、错误恢复和多工具协作仍处于规划阶段。

4. native multimodal modeling 和 deeper end-to-end fusion 仍是未来方向
   作者提到将继续从 DSA compute optimization、data flywheels、Context-RL post-training 走向 native multimodal modeling 和 deeper end-to-end fusion。换言之，当前方案仍较依赖“视频帧作为图像 + timestamp 文本注入 + LLM 聚合”的统一管线，而不是完全端到端的视频原生建模。

5. 若干关键组件缺少独立消融
   论文报告了 DSA、Video RL、MOPD、数据课程、系统优化等综合收益，但没有充分给出每个组件对最终 benchmark 的逐项 ablation。例如，MOPD 相比简单 multi-task SFT/RL mixing 的定量收益、13 个 teacher 的贡献拆分、top-k overlap estimator 的替代实验等没有详细表格。

6. 评测公平性仍有复杂因素
   TimeLens 中不同模型存在 2 FPS 和 4 FPS 设置差异，Video-MME-v2 中 Gemini-3-Flash 也采用不同采样设置。论文已说明这些细节，但这也意味着跨模型对比仍受到 frame sampling、closed-source API、上下文长度和推理预算差异影响。

#### 我的分析：优点

1. 问题定义清晰：同时抓住“长视频成本”和“多任务能力冲突”
   很多 MLLM 技术报告只堆 benchmark，而这篇文章把两个核心瓶颈讲得比较清楚：长视频需要 sparse long-context modeling，多能力 post-training 需要解决 catastrophic forgetting。DSA 和 MOPD 分别对应这两个瓶颈，主线较完整。

2. DSA 适配 GQA-based multimodal backbone 有工程价值
   把 DSA 用到 GQA 多模态架构中，比单纯宣称支持 256K context 更有实用意义。论文还讨论了 indexer warm-up、sparse adaptation、variable-length sequence optimization、RL 中 deterministic Top-k 等细节，说明作者确实面对了训练和推理落地问题。

3. Cross-Modal MOPD 对 post-training 冲突的处理值得关注
   MOPD 不是简单 teacher distillation，而是 on-policy、multi-teacher、cross-modal、token-level feedback。尤其 top-k overlap estimator 只在 teacher 和 student 都认为 plausible 的局部分布上比较，能减少 full-vocabulary KL 中低概率 token 的噪声。这对多领域 RL teacher consolidation 很有启发。

4. 训练 curriculum 完整，覆盖从 projector 到 256K long context
   Stage 0 到 Stage 3 的设计比较系统：先稳定视觉语言接口，再做 32K 对齐，再注入 64K 多任务能力，最后扩展到 256K 长上下文。视频 curriculum 也从 15s 到 15min 再到 2h，符合能力渐进学习逻辑。

5. 长视频 temporal grounding 结果突出
   TimeLens 三个子集都领先，说明模型不只是“看更多帧”，而是在 temporal localization 上确实有收益。scene-wise dense caption、TVG data 和 tIoU-centered Video RL 的组合比较针对性。

6. 系统工程描述比较扎实
   ExtraIO、ViT-LM heterogeneous parallelism、load balancing、FlashInfer/TileLang DSA kernels、Chunk ViT、Top-k deduplication 等内容说明这不是单纯算法报告，而是围绕大规模长视频训练和推理做了系统协同优化。

7. 30B-A3B MoE 的部署效率定位明确
   总参数 30B、激活 3B，使它在同规模开源模型中有较好性价比。对于长视频和 agent 场景，激活参数少带来的推理成本优势很重要。

8. 保留 text-only 数据和 domain experts 有助于避免多模态训练损害语言能力
   SFT 中约 40% text-only data，以及 pure-text math、code、instruction 等 teacher 的 MOPD consolidation，体现了作者对 language-side capability preservation 的重视。

#### 我的分析：缺点 / 风险

1. 缺少关键消融，难以判断 DSA、MOPD、Video RL 各自贡献
   论文强调很多组件都有效，但没有系统回答：如果不用 MOPD 会掉多少？如果只用 single teacher distillation 会怎样？top-k overlap 是否优于 teacher-only top-k 或 student-only top-k？DSA 对最终长视频准确率的独立贡献是多少？这会影响方法可复现和学术说服力。

2. 数据规模极大，复现门槛很高
   预训练涉及 4B、1T、2T、500B tokens，SFT 又有约 500B tokens，还包含大量内部数据、业务数据、专家模型生成数据和合成数据。即使模型开源，完整训练 recipe 对学术社区仍难复现。

3. benchmark 对比可能受推理预算和采样策略影响
   长视频 benchmark 对 frame sampling、context length、FPS、是否使用字幕、是否允许更多帧非常敏感。论文已披露部分设置，但不同模型之间仍可能不是完全同等预算。尤其 closed-source 模型的内部采样和上下文策略不可控。

4. MOPD 的 teacher routing 和 token-level feedback 复杂度高
   13 个 domain teachers、SPRR 对齐、Top-k distillation、token-category scaling、repetition penalty 都会增加系统复杂度。实际部署或复现时，任何 tokenizer、视觉 token count、chat template、mRoPE position 的错位都可能让 KL 信号错误传播，论文自己也强调严格 alignment verification 是必要的。

5. On-policy distillation 可能继承学生探索分布的盲区
   MOPD 在学生自己生成的 on-policy states 上提供 teacher feedback，这有利于贴近学生真实错误，但如果学生很少探索到某些高质量推理路径，teacher 也难以在这些未访问状态上提供监督。论文没有充分讨论 exploration coverage 问题。

6. Long-context “lossless 256K” 的表述需要谨慎理解
   论文说 DSA 支持 lossless 256K context processing，但 sparse attention 本质上仍选择 Top-k 证据。这里的 “lossless” 更可能指输入上下文不被截断或强抽帧，而不是注意力计算完全等价于 full attention。对于需要密集全局交互的任务，Top-k 稀疏选择仍可能遗漏弱信号。

7. 真实视频 agent 风险尚未充分评估
   future work 指向自动剪辑、内容治理、广告标签和商业投放，这些场景对误判、偏见、隐私、版权和商业安全非常敏感。论文主要报告能力 benchmark，没有充分展开安全治理、错误代价、人工审核和可解释性机制。

8. 视觉帧加 timestamp 的统一建模简单有效，但可能不是视频原生最优
   把每帧作为 image token 并注入 timestamp 有工程简洁性，但对 motion dynamics、帧间连续动作、低级时序信号可能不如视频原生 encoder 或更深的 temporal module。作者在 future work 中提到 native multimodal modeling，也间接说明当前方案仍有提升空间。
