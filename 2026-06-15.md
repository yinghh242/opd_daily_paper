#### Abstract 原文

> Sparse reward reinforcement learning (RL) has become a standard tool for improving LLM reasoning, but its success depends critically on the coverage present in the base model. In practice, models are often primed for RL through \emph{mid-training} on curated reasoning traces that teach useful primitive skills such as decomposition, verification, or self-correction. Although effective, this strategy requires manually specifying what the model should learn, and it remains unclear whether such primitive coverage is enough for much harder problems, which require combining these skills into broader solution strategies. We study a more automated approach: \emph{RL-based mid-training} using large corpora of human-written question-answer data. Rather than treating reference solutions as targets to imitate, our method, ExpRL, uses them as \emph{reward scaffolds}: references are hidden from the policy and used only to construct problem-specific grading rubrics for judging on-policy reasoning traces. The policy samples from the original problem prompt, while an LLM judge compares the sampled reasoning trace against the reference solution and assigns outcome-level or process-level dense rewards. This lets ExpRL reinforce partial progress, useful intermediate reductions, and productive reasoning behaviors that sparse final-answer rewards often fail to upweight. On challenging math reasoning tasks, ExpRL yields stronger RL priming than SFT, sparse-reward GRPO, and self-distillation, and provides a better initialization for subsequent sparse-reward RL. Additional mixed-domain experiments further suggest that ExpRL can extend beyond the original math-only setting.

#### 摘要中文翻译

稀疏奖励 reinforcement learning（RL，强化学习）已经成为提升 LLM reasoning（大语言模型推理能力）的标准工具，但其成功高度依赖 base model（基础模型）中已有的 coverage（覆盖度）。实践中，模型通常会先通过 curated reasoning traces（人工筛选的推理轨迹）进行 mid-training（中间训练），以学习 decomposition（分解）、verification（验证）或 self-correction（自我纠错）等有用的 primitive skills（基础技能）。这种策略虽然有效，但需要人工指定模型应该学习什么；同时，对于更难的问题，仅有这些 primitive coverage（基础技能覆盖）是否足够仍不清楚，因为更难的问题需要把这些技能组合成更广泛的 solution strategies（解题策略）。

本文研究一种更自动化的方法：使用大规模 human-written question-answer data（人类撰写的问答数据）进行 RL-based mid-training（基于 RL 的中间训练）。与把 reference solutions（参考解答）当作 imitation（模仿）目标不同，本文方法 ExpRL 将它们用作 reward scaffolds（奖励脚手架）：参考解答对 policy（策略）隐藏，只用于构造 problem-specific grading rubrics（特定问题的评分准则），以评判 on-policy reasoning traces（在线策略采样的推理轨迹）。policy 只从原始 problem prompt（问题提示）中采样，而 LLM judge（大语言模型裁判）会将采样到的 reasoning trace 与 reference solution 比较，并分配 outcome-level（结果级）或 process-level（过程级）的 dense rewards（稠密奖励）。

这使得 ExpRL 能强化 partial progress（部分进展）、有用的 intermediate reductions（中间化简）以及 productive reasoning behaviors（有效推理行为），而这些往往是 sparse final-answer rewards（稀疏最终答案奖励）无法充分提升权重的。在具有挑战性的 math reasoning tasks（数学推理任务）上，ExpRL 相比 SFT、sparse-reward GRPO 和 self-distillation 能产生更强的 RL priming（RL 预热/初始化效果），并为后续 sparse-reward RL 提供更好的初始化。额外的 mixed-domain experiments（混合领域实验）进一步表明，ExpRL 可以扩展到原始纯数学设置之外。

#### 研究背景

论文关注 LLM reasoning 中一个很实际的瓶颈：sparse-reward RL 虽然可以用最终答案是否正确来训练模型，但它要求 base model 已经有一定概率采样到正确或接近正确的 reasoning paths（推理路径）。如果 base model 对有效路径的概率质量很低，那么即使采样很多次，也很少得到正确或部分正确的轨迹；最终答案的二值奖励几乎不给学习信号，RL 只能强化模型本来就会采样到的行为。

因此，mid-training 的目标不是直接在中间训练集上刷分，而是改善后续 RL 的起点：让模型在 hard problems（难题）上把更多概率质量放到 productive reasoning attempts（有效推理尝试）上。论文把 pass@$k$ 视为 coverage（覆盖度）的可观测 proxy（代理指标）：如果模型对某个问题至少有一条完整有效路径赋予非平凡概率，那么多次采样后应更可能出现正确 rollout；因此 pass@$k$ 的提升说明模型能采样到更多潜在可成功的解法路径。

已有 mid-training 常见做法包括：

1. SFT（supervised fine-tuning，监督微调）：直接模仿 reference solutions。
2. self-distillation（自蒸馏）：用带参考信息的 teacher 产生 token-level targets（逐 token 目标），再训练 student。
3. sparse-reward GRPO：仍然只用最终答案正确与否作为奖励。

论文认为这些方法各有问题。SFT 会把人类参考解答当成离线轨迹强行克隆，但这些轨迹可能与 base model 自身的推理分布差异很大，导致 off-policy mismatch（离策略不匹配）和能力扰动。self-distillation 虽然使用模型自己的 rollout，但监督信号仍来自 privileged teacher（有特权信息的教师），如果 teacher distribution 与 student 可达分布相距太远，也可能损害泛化。sparse-reward RL 则在难题上奖励过稀，难以引导 partial progress。

ExpRL 的核心动机是：不要把 reference solution 暴露给 actor（行动模型）作为模仿目标，而是只让 judge 使用 reference solution 来评估 actor 自己采样出的 on-policy reasoning trace，从而保留 on-policy exploration（在线探索），同时给出比最终答案更密集的训练信号。

#### Method

ExpRL 的问题设定是：给定 mid-training 数据集

$$
\mathcal{D}_{\text{mid}}=\{(x_i,y_i^\star)\}_{i=1}^N,
$$

其中 $x_i$ 是问题，$y_i^\star$ 是 step-by-step reference solution。训练策略记为 $\pi_\theta$，base policy 记为 $\pi_b$。目标是用 $\mathcal{D}_{\text{mid}}$ 改善 $\pi_b$，使其在下游数据集 $\mathcal{D}'$ 上经过 sparse binary outcome reward 训练时更有效。下游奖励为

$$
r(x,y)\in\{0,1\},
$$

表示 rollout 的最终答案是否正确。

ExpRL 的设计原则是：reference solution 不作为 demonstration（示范轨迹）或 hint（提示）给 policy，而只作为 judge 的 reward scaffold。policy 仍然只看原始问题 $x$ 并采样自己的解答 $y$；LLM judge 比较 $(x,y,y^\star)$，判断 $y$ 与 reference solution 的策略、关键步骤和部分进展是否对齐。

论文使用 base model Qwen3-4B-Instruct 作为主实验中的 LLM judge。judge 输出一个五分制 Likert score：

$$
\tilde{s}(x,y,y^\star)\in\{1,2,3,4,5\}.
$$

该分数被归一化为 dense reward：

$$
s(x,y,y^\star)=\frac{\tilde{s}(x,y,y^\star)-1}{4}\in[0,1].
$$

judge 被明确要求只做 verification（验证），不要自己解题、不要补全缺失步骤、不要纠正模型错误。如果 rollout 中没有直接支持某个 reference checkpoint（参考检查点），则不能凭空认为它完成了该步骤。

ExpRL 有两个主要变体。

第一，ExpRL-Outcome。它对完整 rollout $y$ 给一个 dense outcome-level reward：

$$
s(x,y,y^\star).
$$

这不同于 sparse outcome reward，因为即使最终答案不正确，只要推理中包含与 reference solution 对齐的部分进展、正确中间化简或有效策略，也能得到非零奖励。这样可以在正确解很少被采样到时仍然提供学习信号。

第二，ExpRL-Process。它进一步把 rollout 切成若干 prefixes（前缀）：

$$
\{y_{\leq t}\}_{t=1}^T.
$$

论文实现中用模型自然生成的分隔符 “###” 来切分步骤。对每个 prefix 调用同一个 reference-conditioned judge，得到

$$
s_t=s(x,y_{\leq t},y^\star),\quad t=1,\ldots,T.
$$

然后把这些 prefix scores 转换成 segment-level advantages（片段级优势），用于更局部的 credit assignment（信用分配）：

$$
A_t(x,y)=
\begin{cases}
s_t-s_{t-1}, & t>1,\\
s_1-s_T, & t=1.
\end{cases}
$$

当 $t>1$ 时，如果当前片段让 judge 认为与 reference 的对齐程度提高，则 advantage 为正；如果让对齐程度下降，则为负。第一段使用 $s_1-s_T$ 而不是直接用 $s_1$，是为了避免第一段因绝对分数过大而主导训练。

整体优化目标是在 dense reward 下进行 on-policy RL，并加入相对于 reference policy $\pi_0$ 的 KL regularization：

$$
\max_\theta\;
\mathbb{E}_{(x,y^\star)\sim\mathcal{D}_{\text{mid}}}
\left[
\mathbb{E}_{y\sim\pi_\theta(\cdot\mid x)}
\left[
R(x,y,y^\star)
\right]
-\beta\,\mathrm{KL}\left(\pi_\theta(\cdot\mid x)\Vert\pi_0(\cdot\mid x)\right)
\right].
$$

其中 $R$ 可以是 ExpRL-Outcome 的完整轨迹奖励，也可以是 ExpRL-Process 的过程级 reward / advantage。ExpRL-Outcome 使用类似 GRPO 的 group-normalized reward update；ExpRL-Process 使用 REINFORCE-style token/segment advantages，不再做额外 group normalization。

Stage-I 是 ExpRL mid-training，用 dense reference-guided rewards 建立 reasoning coverage。Stage-II 则从 primed policy 初始化，使用普通 sparse outcome reward 继续训练；reference solution 在 Stage-II 完全移除。也就是说，ExpRL 不改变下游 RL 的奖励结构，只改变初始化。

论文还强调，ExpRL 的 reward 不必完美等价于真实任务成功；在 mid-training 阶段，它只需要倾向于鼓励更广泛、更有生产力的 reasoning behaviors 和 solution paths，使后续 sparse-reward RL 更容易采样到可强化的轨迹。

总体实验结论是：ExpRL 在 math reasoning 上同时改善 Stage-I primed policy 和 Stage-II downstream sparse RL。它优于 SFT，说明直接模仿 human-written reference traces 不是好的 coverage-building 方式；它优于 sparse GRPO，说明 hard problems 上最终答案奖励过稀；它也通常优于 self-distillation，说明 privileged teacher 的 token-level supervision 不如 reference-guided on-policy reward 更适合 RL priming。

混合领域实验则说明 ExpRL 不只是数学模板学习：在 math、science、coding 的 mixed-domain setting 中，4B judge 可以为 8B policy 提供有效 dense rewards。不过 coding 是例外领域，因为 execution-based sparse reward 已经非常强，而 reference solution 对 incomplete code 的 partial progress 评估不如数学和科学推理自然。

#### 论文自我评价的缺陷

论文明确列出了 Limitations：ExpRL 需要 auxiliary information（辅助信息），例如 reference solutions，才能在 mid-training 中识别有用技术和 partial progress。这类信息并非总是可获得，尤其是在高质量参考解答难以收集的领域。

此外，Conclusion 和 Appendix 中还透露出若干自我讨论的局限和未来方向：

第一，当前 ExpRL 主要把 judge 用作 scalar process rewards（标量过程奖励）生成器。论文认为未来可以进一步利用 judge 的 natural-language feedback（自然语言反馈），让模型知道哪些步骤缺失、错误或值得继续，而不只是得到一个分数。

第二，ExpRL 可以与 prefix-conditioned generation（前缀条件生成）结合。论文指出，未来可以主动采样或构造 diverse reasoning prefixes，以进一步扩展 productive solution paths 的 coverage，并更好刻画方法的 exploration ceiling（探索上限）。

第三，process reward 仍有 calibration（校准）、length normalization（长度归一化）和 judge design（裁判设计）问题。论文观察到 ExpRL-Process 会带来 response length 增长，并与 length clipping 产生交互，甚至影响 “###” step delimiter 的分布。这说明过程级奖励虽然提升 credit assignment，但训练稳定性和长度控制仍需要系统研究。

第四，ExpRL 依赖 reference-conditioned judge 的可靠性。校准实验显示，4B、8B、14B judge 在正确 reference 下效果较好，但 0.6B judge 不稳定；wrong reference 会显著破坏 reward signal。这说明 ExpRL 并不是任意 judge 都能工作，它需要足够强的 judge 和可靠的问题匹配 reference。

第五，coding 领域的结果显示，reference-guided judging 并不总是优于领域内强 verifier。对于代码任务，execution feedback 可以直接验证功能正确性，而 incomplete code 的 partial progress 很难与 reference solution 对齐；因此 ExpRL 更适合作为 Stage-I priming，而不是替代强环境奖励。

#### 我的分析：优点

1. 问题定位清楚：论文没有泛泛讨论 “RL 提升 reasoning”，而是准确抓住 sparse-reward RL 的 exploration / coverage bottleneck。它把 pass@$k$ 解释为 productive reasoning path coverage 的 proxy，这个视角比单看 pass@1 更适合分析 RL 初始化问题。

2. 方法设计符合 on-policy learning 的直觉：ExpRL 不让 policy 直接看 reference solution，也不强迫模型模仿离自己分布很远的人类解答，而是只用 reference 来评价 policy 自己采样出的轨迹。这避免了 SFT 和 off-policy distillation 的核心问题，同时保留了 reference 的监督价值。

3. Dense reward 的使用方式比较克制：论文没有声称 LLM judge 的分数就是最终正确性，而是把它定义为 mid-training 阶段的 shaping signal（塑形信号）。只要它能把概率质量推向更有希望的 reasoning regions，就能为后续 sparse RL 提供更好初始化。

4. Outcome reward 和 process reward 的区分有实际意义：ExpRL-Outcome 提供更密集的完整轨迹评分，ExpRL-Process 则进一步解决 credit assignment。实验中两者各有优势：Outcome 在多个最终 benchmark 上更稳，Process 在 AIME26 和 Stage-I coverage 上表现很强。

5. 实验设计比较完整：论文不仅报告 Stage-II downstream RL 后的结果，也报告 Stage-I primed policy 的 pass@1 / pass@16、pass@$k$ curves、training dynamics、entropy、KL、behavior annotation、mixed-domain scaling 和 judge calibration。这些分析共同支持 “ExpRL 改善的是初始化和 coverage，而不仅是多训练了一段” 的主张。

6. 对 SFT 退化的结果很有价值：SFT 在主表中显著低于 base model，这强烈说明在 hard reasoning mid-training 中，人类 reference traces 并不天然适合直接 imitation。这个结果对很多依赖高质量解答做 SFT 的 reasoning pipeline 有警示意义。

7. Judge calibration 实验增强了可信度：论文专门比较 correct reference、no reference 和 wrong reference，证明 dense reward 主要来自 problem-matched reference 的 scaffold，而不是 generic LLM confidence。这一点对 LLM-as-judge reward modeling 很关键。

8. 混合领域实验扩展了方法边界：4B judge 给 8B policy 打分仍有效，说明 ExpRL 的核心不是 judge 和 policy 同规模，而是 reference-conditioned verification 比 open-ended generation 更容易。这对低成本 reward model / judge 设计有启发。

#### 我的分析：缺点 / 风险

1. 对 reference solution 的依赖是最大限制。ExpRL 的优势来自 reference-conditioned dense rewards，但许多真实任务没有高质量 step-by-step reference solution。即使有最终答案，也未必有足够详细、可靠、可用于 partial-progress 判断的 reference path。

2. Reward hacking 和 judge bias 仍然可能存在。虽然论文强调 judge 不要补全缺失步骤，但 LLM judge 仍可能偏好与 reference 表面相似的推理风格，而不是真正等价的 alternative solution。这样可能把模型推向 “看起来像 reference” 的轨迹，而不是更广泛的正确解法空间。

3. Process reward 的实现依赖 “###” delimiter，工程上较脆弱。论文自己也发现 ExpRL-Process 训练后 delimiter 数量分布会变化，并与 length clipping 交互。这说明 process-level credit assignment 的切分方式可能影响训练动态；换模型、换格式或换领域后不一定稳定。

4. 数学任务中 reference path 通常较清晰，但开放式任务更难。对于 science QA 或 coding，正确解可能有多种风格；特别是 coding 中，正确程序可以与 reference implementation 差异很大。论文结果也显示 coding 上 execution-based GRPO 更强，说明 ExpRL 的适用性依赖任务是否适合 reference-path comparison。

5. LLM judge 成本可能较高。ExpRL-Process 需要对多个 prefixes 调用 judge，训练时每个 prompt 又采样多个 rollouts。这会带来显著计算和推理开销。论文虽然展示了 4B judge 可服务 8B policy，但没有充分展开成本-收益曲线。

6. Stage-II 与 Stage-I 使用相近 prompt family，泛化边界仍需更多验证。主数学实验中 downstream RL 使用 InT+POPE prompt mixture，同属 ExpRL priming 的问题族。虽然 held-out benchmarks 是标准评测集，但更强的跨分布测试会更能说明 ExpRL 是否真的学到通用 exploration prior。

7. pass@$k$ 作为 coverage proxy 合理但不完美。pass@$k$ 提升可能来自更长回答、更高采样多样性或某些 benchmark-specific heuristics，而不一定完全代表 productive solution path coverage。论文通过 entropy、behavior annotation 和 calibration 做了补充，但 coverage 本身仍是间接测量。

8. SFT baseline 的实现细节可能影响结论。论文显示 SFT 大幅退化，这支持作者观点；但实际 SFT 的数据格式、学习率、训练步数、loss mask、是否混合原始数据等都会影响结果。如果 SFT baseline 没有充分调优，ExpRL 相对 imitation 的优势可能被放大。

9. Dense reward 与最终 correctness 之间仍有错配风险。ExpRL 奖励 partial progress，但某些局部看似正确的步骤可能引导到死路，或者与 reference 对齐却不是解决该问题最稳健的路径。尤其在 process reward 中，局部 advantage 可能鼓励短期对齐，而不是全局解题成功。

10. 方法可能强化可被 judge 识别的 reasoning behaviors。论文观察到 verification、self-correction、backtracking 增加，这是优点；但如果模型学会显式写出这些行为来获得高分，而不是实质性改善推理，可能出现 verbosity（冗长化）或 performative reasoning（表演式推理）。论文中 response length 增长现象已经提示了这一风险。

#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy distillation。

#### Abstract 原文

> On-policy distillation (\textsc{OPD}) has recently become a prominent post-training recipe by combining two desirable ingredients: on-policy student trajectories and dense teacher supervision. However, how this hybrid changes a model's parameters remains unclear. Across several language and vision-language model pairs and \textsc{OPD} use cases, our analysis yields two main findings. On sparsity, \textsc{OPD} updates are small and coordinate-sparse. They are distributed across layers, with the largest relative movement usually appearing in FFN modules. This sparse structure is operationally useful: training only the discovered subnetwork nearly recovers full-training performance. The sparse support does not remove the need for adaptive optimization: SGD, previously reported to be competitive in \textsc{RLVR}, underperforms AdamW in our \textsc{OPD} optimizer ablation, suggesting that dense teacher supervision preserves useful momentum structure and heterogeneous second-moment scales. On geometry, the updates are numerically full-rank but spectrally concentrated; they lie mostly away from the principal singular subspaces of the source weights and fall disproportionately on coordinates where the source weights are close to zero. These findings suggest that dense teacher supervision does not turn \textsc{OPD} into ordinary dense parameter rewriting; instead, \textsc{OPD} retains important geometric signatures of on-policy post-training.

#### 摘要中文翻译

On-policy distillation (OPD) 近年来成为一种重要的 post-training（后训练）方法，因为它结合了两个理想要素：on-policy student trajectories（学生模型当前策略生成的轨迹）和 dense teacher supervision（密集教师监督）。然而，这种混合机制如何改变模型参数仍不清楚。本文在多个语言模型与视觉语言模型配对、以及多种 OPD 使用场景上进行分析，得到两个主要发现。

关于 sparsity（稀疏性），OPD 的更新幅度很小，并且在坐标层面是稀疏的。这些更新分布在各层中，其中相对移动幅度最大的通常出现在 FFN modules（前馈网络模块）。这种稀疏结构在操作上有用：只训练被发现的 subnetwork（子网络）几乎可以恢复完整训练的性能。稀疏支持集并不意味着不需要 adaptive optimization（自适应优化）：此前在 RLVR 中被报告为有竞争力的 SGD，在本文 OPD optimizer ablation（优化器消融）中弱于 AdamW，这表明 dense teacher supervision 仍保留了有用的 momentum structure（动量结构）和异质的 second-moment scales（二阶矩尺度）。

关于 geometry（几何结构），这些更新在数值上是 full-rank（满秩）的，但谱能量高度集中；它们大多避开源权重的 principal singular subspaces（主奇异子空间），并且不成比例地落在源权重接近零的坐标上。这些发现表明，dense teacher supervision 并不会把 OPD 变成普通的 dense parameter rewriting（密集参数重写）；相反，OPD 保留了 on-policy post-training 的重要几何特征。

#### 研究背景

OPD 的核心位置介于传统 SFT / offline sequence-level knowledge distillation（离线序列级知识蒸馏）和 RLVR 之间。SFT 与 offline distillation 提供 dense token-level supervision（密集 token 级监督），但训练数据是固定轨迹，容易与测试时模型自身生成分布产生 exposure mismatch（暴露偏差）和 distribution shift（分布偏移）。RLVR 使用当前 policy（策略）采样，具有 on-policy 特性，但 reward 通常是稀疏的 sequence-level signal（序列级信号），存在 credit assignment（信用分配）困难。

OPD 结合了二者：样本来自 student policy（学生策略）的当前生成，因此是 on-policy；监督来自 teacher model（教师模型）对 student-generated trajectory（学生生成轨迹）的 token-level feedback，因此是 dense supervision。这使 OPD 成为理解 post-training parameter dynamics（后训练参数动态）的关键中间形态：如果 RLVR 的稀疏参数更新主要来自 sparse reward，那么 OPD 的 dense teacher signal 可能导致更密集的参数改写；如果关键因素是 on-policy data distribution，那么 OPD 即使用密集监督，也应保留类似 RLVR 的 sparse / off-principal 更新几何。

本文的问题正是：OPD 的参数变化更像 dense supervised fine-tuning、更像 sparse RLVR，还是形成一种独立的更新机制？作者通过最终 checkpoint delta（检查点差分）分析其 sparsity、spectral geometry、source-weight alignment（与源权重几何的对齐关系），并进一步用 masked subnetwork training（掩码子网络训练）和 optimizer ablation 验证这些结构是否具有功能意义。

#### Method

论文不提出新的 OPD objective，而是对已有 OPD / RLVR / distillation checkpoint 进行参数更新分析与干预验证。核心对象是源模型与训练后模型之间的 checkpoint delta：

$$
\Delta W = W_{\text{trained}} - W_{\text{src}}
$$

OPD 形式化为 student policy 在当前 prompt 上生成 response，然后 teacher 在 student-generated prefixes 上提供 dense token-level divergence supervision：

$$
\mathcal{L}_{\text{OPD}}(\theta)
=
\mathbb{E}_{x \sim D,\ y \sim \pi_{\theta}(\cdot \mid x)}
\left[
\sum_{t=1}^{|y|}
D\left(
\pi_T(\cdot \mid x, y_{<t})
\|
\pi_{\theta}(\cdot \mid x, y_{<t})
\right)
\right]
$$

其中 $y_{<t}$ 来自 student，而不是固定 teacher trace。这一点区分了 OPD 与 offline KD / SFT。

作者从三类角度分析 $\Delta W$：

1. Scale and coordinate sparsity（规模与坐标稀疏性）。使用 relative delta norm 衡量更新大小：

$$
r(W) =
\frac{\|\Delta W\|_F}{\|W_{\text{src}}\|_F + \epsilon_0}
$$

并使用 threshold sparsity 衡量坐标层面的可见更新：

$$
s_{\epsilon}(W)
=
\frac{|\{i : |\Delta W_i| \le \epsilon\}|}{|\Delta W|}
$$

主表使用 $\epsilon = 10^{-5}$。

2. Spectral structure（谱结构）。对每个二维矩阵更新做 SVD，分析 top-$k$ singular-value energy、stable rank 和 numerical rank：

$$
e_k(W)
=
\frac{\sum_{i=1}^{k} \sigma_i^2}
{\sum_{i=1}^{m} \sigma_i^2 + \epsilon_0}
$$

$$
\operatorname{srank}(\Delta W)
=
\frac{\|\Delta W\|_F^2}
{\|\Delta W\|_2^2 + \epsilon_0}
$$

这用于区分“数值满秩”与“能量谱集中”：OPD 更新可以 rank 很高，但主要能量集中在少数 singular directions（奇异方向）。

3. Source geometry alignment（源权重几何对齐）。对源权重 $W_{\text{src}} = U \Sigma V^{\top}$，计算更新在 source principal singular subspace（源权重主奇异子空间）中的投影：

$$
p_k(W)
=
\frac{
\|U_k^{\top} \Delta W V_k\|_F^2
}
{
\|\Delta W\|_F^2 + \epsilon_0
}
$$

同时构造 source-principal coordinate mask 与 low-magnitude source-weight mask，比较 visible update coordinates（可见更新坐标）落在哪些区域。若更新避开 principal coordinates、偏向 source weights 接近零的位置，就说明它更像 off-principal on-policy editing，而不是沿源权重主方向做密集重写。

除静态分析外，论文还做了两个功能性干预：

- Subnetwork-masked OPD：从源模型重新训练，只允许 full OPD checkpoint 中可见更新的坐标被训练，并与 random mask、RLVR / GRPO mask 比较。
- AdamW versus SGD：在 JustRL-teacher OPD 设置下比较 AdamW 和 vanilla SGD，检验稀疏最终更新是否意味着不再需要 adaptive optimization。

| OPD source geometry | Six OPD-style pairs | 10% source-principal mask coverage 仅 $4.39\%$ 到 $5.46\%$；10% low-magnitude mask coverage 为 $24.99\%$ 到 $48.57\%$ | 更新避开 principal coordinates，偏向 source weights 接近零的坐标 |
| Offline distillation contrast | Qwen-Math Distill | Principal coverage 为 $9.95\%$，low-magnitude coverage 为 $10.10\%$，接近随机 $10\%$ baseline | 与 OPD / RLVR 的 off-principal bias 明显不同 |

整体实验支持三个结论。第一，OPD 更新虽然由 dense teacher supervision 驱动，但在最终 checkpoint 上仍是 small、coordinate-sparse、distributed-across-layers 的。第二，这些更新不是严格 low-rank，而是 full-rank with spectral concentration，因此 LoRA 等 low-rank 方法可能捕获重要能量，但不应被期待精确复现完整 delta。第三，OPD 的更新显著避开 source-principal coordinates，并富集在 source weights 低幅值区域，这使它更接近 sparse on-policy editing，而不是 dense supervised rewriting。

#### 论文自我评价的缺陷

论文未单列 limitations，但从全文可见，作者在 Discussion and Future Work 中明确承认了若干限制。

首先，实验主要是 static final-checkpoint analysis（静态最终检查点分析）。论文比较源模型与训练后模型的 $\Delta W$，但没有完整追踪 OPD training trajectory（训练轨迹）。因此，最终 sparse task vector 是如何在训练过程中形成的、哪些阶段决定了支持集、谱集中和 off-principal movement，目前仍未被揭示。

其次，论文重点分析 weight space（权重空间）的变化，而不是 behavior space（行为空间）的变化。作者指出，需要互补的 behavioral analysis 来解释 on-policy training 如何改变模型输出。换言之，本文能说明“参数怎么变”，但不能完全说明“模型具体行为为什么这样变”。

第三，规模和覆盖范围有限。观测实验包含的是相对 small-scale LLM 和 VLM checkpoints；干预实验主要限制在 DS-Qwen 和 Qwen2.5-VL，并集中于 math reasoning / Geo3K 等任务。作者明确指出，需要 larger models、更多 domains，包括 agentic tasks（智能体任务）和 embodied tasks（具身任务），来验证 sparsity、module structure、off-principal movement 和 optimizer sensitivity 是否是 OPD 的稳定性质，而不是当前训练 recipe 的产物。

第四，论文提出了未来 OPD-native adaptation and optimization 的方向，但尚未验证。例如，谱集中启发 LoRA variants，off-principal geometry 启发 orthogonal fine-tuning，optimizer side 则提示 vanilla Muon 可能不适配而 high-pass variants 值得测试。这些都还只是 future work，而非本文已经完成的实证结论。

#### 我的分析：优点

1. 问题切入非常清晰。论文抓住了 OPD 的关键中间地位：它既是 on-policy，又有 dense teacher supervision。因此它可以检验 RLVR 稀疏更新到底来自 sparse reward，还是来自 on-policy distribution。这个问题对 post-training 机制理解很重要。

2. 不只看 benchmark accuracy，而是分析 parameter dynamics。论文从 relative norm、coordinate sparsity、spectral concentration、source-principal alignment、mask overlap 等多个层面刻画 $\Delta W$，比单纯报告性能更能解释 OPD 与 SFT / RLVR 的差异。

3. 观测分析和功能性干预结合较好。作者不仅发现 OPD delta 稀疏，还重新从源模型出发，只训练 discovered OPD subnetwork，并证明它几乎恢复 full OPD 性能。这使“稀疏支持集”不只是 post-hoc 描述，而有 operational usefulness。

4. 对 optimizer 的结论有实际价值。论文没有简单套用 RLVR 中“SGD 也可以”的结论，而是在 OPD 中做 AdamW versus SGD ablation，发现 dense teacher supervision 可能保留 gradient heterogeneity，使 AdamW 的 adaptive scaling 仍然重要。这对 OPD 训练 recipe 设计很有参考意义。

5. 对 low-rank adaptation 的判断比较审慎。论文区分了 numerical full-rank 和 spectral concentration，避免把“top singular values 能量集中”误读为“更新就是 low-rank”。这对理解 LoRA / low-rank OPD adaptation 的适用边界很有帮助。

6. 结果与 on-policy post-training 的几何图景一致。OPD 与 RLVR 在 sparse support、off-principal movement、low-magnitude coordinate enrichment 上相近，而与 offline distillation 明显不同，支持“on-policy distribution 是决定 post-training update geometry 的关键因素之一”这一解释。

#### 我的分析：缺点 / 风险

1. checkpoint sparsity 对数值阈值和存储精度敏感。论文主要使用 bfloat16 checkpoint subtraction，并以 $10^{-5}$ 作为主阈值。虽然附录给了多阈值统计，但“visible sparsity”仍可能受 checkpoint format、rounding、optimizer state、weight decay 和保存流程影响。它不完全等价于真实训练过程中梯度或参数更新的稀疏性。

2. mask-from-full-run 有 hindsight leakage（后验泄漏）。Subnetwork experiment 先用完整 OPD run 发现 mask，再从源模型只训练这个 mask。这证明了 full-run support 有功能意义，但它不是一个无需完整训练即可获得的 practical PEFT recipe。真正部署时如何提前预测这些 coordinates，仍是开放问题。

3. 对因果机制的解释仍偏间接。论文认为 OPD 的 sparse / off-principal geometry 主要来自 on-policy distribution，而不是 reward sparsity。这个解释合理，但实验主要是 checkpoint-level correlation 与对比；要更强地证明因果，需要控制 teacher quality、trajectory distribution、loss granularity、reward density 等变量的系统实验。

4. AdamW versus SGD 的结论可能依赖调参范围。SGD 使用 learning rate $10^{-2}$，AdamW 使用 $10^{-6}$，结果 AdamW 明显更好。但 SGD 是否经过充分 tuning、是否需要 momentum SGD、schedule、weight decay 或 layer-wise learning rate，并未完全展开。因此“OPD 需要 AdamW”可以作为本文设置下的结论，但不宜过度推广到所有 OPD recipe。

5. 模型规模和任务范围仍有限。论文覆盖多个 public checkpoint 和 LLM / VLM 设置，但主要仍是 $1$B 到 $4$B 级别，以及 math reasoning / Geo3K 等任务。更大模型、代码、agentic、long-context、多模态复杂任务中的 OPD geometry 可能不同。

6. public checkpoint heterogeneity 可能混入混杂因素。不同模型对比中，训练数据、teacher、训练步数、optimizer、checkpoint 来源和 post-training pipeline 不完全一致。虽然作者有 controlled DS-Qwen 和 Qwen2.5-VL 实验，但全局表中的跨模型比较仍应视作 evidence aggregation，而不是严格同变量控制实验。

7. spectral concentration 不直接等于 LoRA 可行。OPD 更新 top-16 SVD energy 较高，但 numerical rank 仍为 full-rank，且坐标稀疏与 off-principal geometry 并不必然被标准 LoRA 捕获。未来 OPD-native adaptation 可能需要同时处理 sparsity、low-rank energy concentration 和 source-orthogonal constraints，而不是简单套用普通 LoRA。

8. 行为层面的解释不足。参数更新落在 low-magnitude coordinates 或避开 principal singular subspace，并不直接告诉我们模型推理策略、错误模式、校准或 teacher imitation 行为如何变化。对于 OPD / OPSD 中 teacher supervision 是否可靠、是否引入 hallucination 或 over-imitation，本文没有深入分析。
