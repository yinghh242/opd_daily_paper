#### Abstract 原文

> Sparse reward reinforcement learning (RL) has become a standard tool for improving LLM reasoning, but its success depends critically on the coverage present in the base model. In practice, models are often primed for RL through \emph{mid-training} on curated reasoning traces that teach useful primitive skills such as decomposition, verification, or self-correction. Although effective, this strategy requires manually specifying what the model should learn, and it remains unclear whether such primitive coverage is enough for much harder problems, which require combining these skills into broader solution strategies. We study a more automated approach: \emph{RL-based mid-training} using large corpora of human-written question-answer data. Rather than treating reference solutions as targets to imitate, our method, ExpRL, uses them as \emph{reward scaffolds}: references are hidden from the policy and used only to construct problem-specific grading rubrics for judging on-policy reasoning traces. The policy samples from the original problem prompt, while an LLM judge compares the sampled reasoning trace against the reference solution and assigns outcome-level or process-level dense rewards. This lets ExpRL reinforce partial progress, useful intermediate reductions, and productive reasoning behaviors that sparse final-answer rewards often fail to upweight. On challenging math reasoning tasks, ExpRL yields stronger RL priming than SFT, sparse-reward GRPO, and self-distillation, and provides a better initialization for subsequent sparse-reward RL. Additional mixed-domain experiments further suggest that ExpRL can extend beyond the original math-only setting.

#### 摘要中文翻译

稀疏奖励 reinforcement learning（RL，强化学习）已经成为提升 LLM reasoning（大语言模型推理能力）的标准工具，但其成功高度依赖 base model（基座模型）中已有的 coverage（覆盖度）。实践中，模型通常会先通过 curated reasoning traces（人工筛选的推理轨迹）进行 mid-training（中间训练），以学习 decomposition（分解）、verification（验证）或 self-correction（自我纠错）等有用的 primitive skills（基础技能）。尽管这种策略有效，但它需要人工指定模型应该学习什么，而且仍不清楚这些基础技能覆盖是否足以应对更难的问题；这些问题往往需要把多种技能组合成更广泛的 solution strategies（解题策略）。本文研究一种更自动化的方法：使用大规模 human-written question-answer data（人工撰写的问答数据）进行基于 RL 的 mid-training。本文方法 ExpRL 并不把 reference solutions（参考解答）当作要模仿的目标，而是把它们用作 reward scaffolds（奖励脚手架）：reference solutions 对 policy（策略模型）隐藏，只用于构造面向具体问题的 grading rubrics（评分准则），以评判 on-policy reasoning traces（在线策略采样的推理轨迹）。policy 仅从原始 problem prompt（问题提示）中采样，而 LLM judge（大语言模型裁判）会将采样得到的 reasoning trace 与 reference solution 对比，并分配 outcome-level（结果级）或 process-level（过程级）dense rewards（稠密奖励）。这使 ExpRL 能够强化 partial progress（部分进展）、有用的 intermediate reductions（中间化简）以及 productive reasoning behaviors（有效推理行为），而这些通常是 sparse final-answer rewards（稀疏最终答案奖励）无法有效提升权重的。在具有挑战性的数学推理任务上，ExpRL 相比 SFT、sparse-reward GRPO 和 self-distillation 产生了更强的 RL priming（RL 预热/初始化效果），并为后续 sparse-reward RL 提供了更好的初始化。额外的 mixed-domain experiments（混合领域实验）进一步表明，ExpRL 可能扩展到原始纯数学设定之外。

#### 研究背景

本文关注的问题是：为什么在难题上，直接用 sparse reward RL 往往不够。作者认为，RL 能否提升 LLM reasoning，关键取决于 base model 在训练开始前是否已经对“可能成功的推理路径”分配了足够概率质量。如果 base model 很少采样到正确或部分正确的 reasoning trajectories，那么 binary final-answer reward 几乎不给学习信号，RL 只能强化模型本来就会产生的模式。

论文把这个瓶颈表述为 coverage over productive reasoning paths（对有效推理路径的覆盖）。作者用 $\mathrm{pass}@k$ 作为可观测代理指标：如果模型对某个问题至少有一条可成功的推理路径分配了非平凡概率，那么多次采样应更可能得到正确答案。因此，提升 $\mathrm{pass}@k$ 表示模型的可探索解法空间变宽，而不仅是单次采样精度变高。

形式化设定中，mid-training 数据为：

$$
\mathcal{D}_{\mathrm{mid}}=\{(x_i,y_i^\star)\}_{i=1}^N
$$

其中 $x_i$ 是问题，$y_i^\star$ 是 step-by-step reference solution。目标是从 base policy $\pi_b$ 出发训练 policy $\pi_\theta$，使其在后续数据 $\mathcal{D}'$ 上用 sparse binary outcome reward：

$$
r(x,y)\in\{0,1\}
$$

进行 downstream RL 时更容易成功。

已有做法包括：

- SFT：直接模仿 reference solutions，但人类解答可能与模型自身分布差异很大，导致 off-policy mismatch（离策略分布不匹配），甚至破坏模型原有推理能力。
- self-distillation：使用 reference-conditioned teacher 给学生提供 token-level targets，虽然比纯 SFT 更接近 on-policy，但如果 teacher distribution 离 student 太远，仍可能损害泛化。
- sparse-reward GRPO：只根据最终答案正确性更新，在难题上信号过稀疏，容易 entropy collapse（熵塌缩）和 under-exploration（探索不足）。

ExpRL 的核心动机是：reference solutions 不一定要作为 imitation targets（模仿目标），也可以作为 reward scaffolds，用来评估模型自己的 on-policy rollouts 中哪些部分体现了有用进展。

#### Method

ExpRL 是一种用于 LLM mid-training 的 on-policy RL 方法。它的关键设计是：reference solution 不暴露给 actor policy，也不作为教师轨迹让模型模仿；reference solution 只提供给 LLM judge，用于构造 problem-specific rubric，并给模型自己采样出的 reasoning trace 打 dense reward。

整体流程分为两阶段：

1. Stage-I：ExpRL mid-training
   在 $\mathcal{D}_{\mathrm{mid}}$ 上，用 reference-guided dense rewards 训练 base policy，使其增加对 productive reasoning paths 的覆盖。
2. Stage-II：downstream sparse-reward RL
   从 Stage-I 得到的 primed policy 初始化，在目标任务上只使用 binary final-answer reward 继续训练。

ExpRL 有两个主要变体。

ExpRL-Outcome 对完整 rollout 给一个 dense outcome-level reward。LLM judge 读取问题 $x$、模型生成的解答 $y$ 和 reference solution $y^\star$，输出 Likert 分数：

$$
\tilde{s}(x,y,y^\star)\in\{1,2,3,4,5\}
$$

再归一化为：

$$
s(x,y,y^\star)=\frac{\tilde{s}(x,y,y^\star)-1}{4}\in[0,1]
$$

这个分数衡量 generated reasoning 与 reference solution 在策略、关键步骤和结论上的对齐程度。与 sparse reward 不同，即使最终答案错误，只要 reasoning trace 展现了有用的中间进展，也能得到非零反馈。

ExpRL-Process 进一步把完整 rollout 切成多个 prefixes（前缀）：

$$
\{y_{\le t}\}_{t=1}^{T}
$$

论文实现中使用 `###` 作为 step delimiter，因为 Qwen3-4B-Instruct 默认经常用这个符号分隔推理步骤。对每个 prefix，judge 给出分数：

$$
s_t=s(x,y_{\le t},y^\star)
$$

然后构造 segment-level advantage：

$$
A_t(x,y)=
\begin{cases}
s_t-s_{t-1}, & t>1 \\
s_1-s_T, & t=1
\end{cases}
$$

其中 $t>1$ 时，某个 segment 只有在相对前一 prefix 提高了 reference alignment 时才获得正 advantage；如果使对齐程度下降，则获得负 advantage。第一个 segment 使用 $s_1-s_T$ 进行中心化，避免早期分数过度主导训练。

训练目标带有 KL regularization：

$$
\max_{\theta}
\mathbb{E}_{(x,y^\star)\sim\mathcal{D}_{\mathrm{mid}}}
\left[
\mathbb{E}_{y\sim\pi_\theta(\cdot|x)}
\left[
R(x,y,y^\star)
\right]
-
\beta\,\mathrm{KL}
\left(
\pi_\theta(\cdot|x)\,\|\,\pi_0(\cdot|x)
\right)
\right]
$$

其中 $R$ 可以是 ExpRL-Outcome 的完整轨迹 dense reward，也可以是 ExpRL-Process 的 process-level rewards。ExpRL-Outcome 使用 GRPO-style normalized reward update；ExpRL-Process 使用 REINFORCE-style token/segment advantages，不再做 group normalization。

LLM judge 的原则是 verify rather than solve（验证而非求解）。judge 被明确要求不要补全缺失步骤、不要自行纠错、不要推断模型未写出的中间结论。如果某个 rubric item 没有被 generated reasoning 明确支持，就不能算作已满足。

#### 论文自我评价的缺陷

论文明确列出的 limitation 是：ExpRL 需要 auxiliary information（辅助信息），尤其是 reference solutions，来识别哪些技术、步骤或 partial progress 是有用的。这类信息并不总是可获得，特别是在高质量 reference solution 难以收集的领域。

论文在 conclusion 和 appendix 中还指出了若干开放问题：

- 当前 ExpRL 主要使用 judge 产生 scalar process rewards，没有利用 judge 的 natural-language feedback。作者认为未来可以让模型从 judge 的文字反馈中学习哪些步骤缺失、错误或值得继续。
- ExpRL 可以与 prefix-conditioned generation 结合，在 post-training 中主动采样或构造多样化 reasoning prefixes，以进一步扩展 productive solution paths 的覆盖，并更好刻画 exploration ceiling。
- reward calibration、length normalization 和 judge design 仍需系统研究。尤其是 process rewards 与长度增长、裁剪惩罚之间存在交互，可能影响训练稳定性。
- ExpRL-Process 使用 `###` 作为 step delimiter 是基于 Qwen3-4B-Instruct 的经验启发；appendix 显示，delimiter count collapse 主要来自 process-level training 与 length clipping 的交互，这仍是实现层面的开放问题。
- judge 必须达到最低能力门槛。Qwen3-0.6B judge 在 calibration test 中不稳定，说明 ExpRL 不能任意依赖很弱的 judge。
- coding 领域中，reference-solution scaffolding 的作用有限。论文观察到 LiveCodeBench 上 no-reference judging 与 correct-reference judging 的 misplacement rate 接近，且 sparse GRPO 更强，说明在 execution reward 很可靠的领域，ExpRL 未必是最佳奖励来源。

#### 我的分析：优点

- 论文的核心视角清晰：它没有把 mid-training 简化为“让模型学会某些人工指定技能”，而是把目标定义为扩大 productive reasoning paths 的 coverage，并用 $\mathrm{pass}@k$ 作为可观测代理指标。这比只看 $\mathrm{pass}@1$ 更贴近 RL exploration 问题。
- ExpRL 对 reference solutions 的使用方式很有价值：reference 不是 imitation target，而是 reward scaffold。这避免了直接 SFT 人类解答带来的强 off-policy mismatch，也比 token-level self-distillation 更符合 on-policy RL 的学习动态。
- 方法相对简单：Outcome 版本只需 full-rollout dense score，Process 版本只需 prefix scores 和 segment-level advantage。它没有引入复杂架构，主要创新在 reward construction 和 training protocol。
- 实验设计比较完整：包含 Stage-I priming 后的直接评估、Stage-II sparse-reward RL 后评估、$\mathrm{pass}@k$ curves、training dynamics、behavior analysis、mixed-domain scale study 和 judge calibration stress test。
- 论文没有只报告最终 benchmark 分数，还分析了 entropy、response length、unlocked prompts、teacher-student KL 和 reasoning behavior changes，因此对“为什么 ExpRL 比 self-distillation 或 sparse GRPO 更适合 priming”给出了较可信的机制解释。
- 对 unreliable teacher supervision（不可靠教师监督）问题有启发：ExpRL 不强迫学生匹配 teacher 的 token distribution，而是只把 reference-conditioned teacher/judge 作为偏好或进展评估器，降低了错误 token-level 监督的伤害。
- 8B policy + 4B judge 的实验说明，reference-conditioned verification 可以让较小 judge 为较大 policy 提供有用 dense rewards，这对降低训练成本有现实意义。
- calibration 实验区分了 correct reference、no reference 和 wrong reference，证明 reward signal 不是单纯来自 generic judge confidence，而确实依赖 problem-matched reference。

#### 我的分析：缺点 / 风险

- ExpRL 的 reward 本质上偏向 reference-aligned reasoning。对于数学中存在多种正确解法的问题，如果模型走的是不同但有效的路径，judge 可能低估其 partial progress，从而把策略推向 reference-style solutions。
- LLM judge 的 1–5 scalar score 仍然粗糙且有噪声。Process reward 对每个 prefix 打分，噪声会被放大到 token/segment-level updates；如果 judge calibration 不稳定，可能引入系统性偏差。
- 主实验主要集中在 Qwen3-4B-Instruct、InT+POPE 数学数据和 same-distribution Stage-II 设置。虽然有 mixed-domain 扩展，但跨模型、跨数据分布、跨更开放任务的泛化还不充分。
- ExpRL-Process 的实现依赖 `###` delimiter，这是一种模型特定启发。appendix 中 delimiter collapse 和 length clipping 的交互说明，process-level credit assignment 在实际训练中可能很脆弱。
- 论文虽然使用 KL regularization 和 overflow reward 为 $0$ 抑制过长输出，但 dense process reward 仍可能诱导 reward hacking，例如模型学会生成更像“有步骤”的文本，而不一定真正提高推理质量。
- ExpRL 需要 reference solutions 和额外 judge compute。对于没有高质量参考解、或参考解昂贵的领域，方法的适用性会下降；对于大规模 mid-training，judge 调用成本也不可忽略。
- Coding 实验暴露了方法边界：当环境提供强 execution reward 时，reference-guided dense judging 不如直接 sparse GRPO。这说明 ExpRL 更适合“最终答案稀疏但中间推理可由 reference scaffold 验证”的任务，而不是所有 reasoning domains。
- $\mathrm{pass}@k$ 是 coverage 的合理代理，但不是直接证明模型学到了更广泛的解题策略。更严格的因果证据可能需要对 solution strategy diversity、novel problem transfer 和 reference-path dependence 做更细粒度评估。
- self-distillation baseline 的 teacher 是 reference-conditioned base model；如果换成更强但校准更好的 teacher，或使用更保守的 token filtering，结论可能会有所变化。因此“ExpRL 优于 self-distillation”的范围仍依赖具体 teacher 和蒸馏方式。

#### Abstract 原文

> We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. #### 摘要中文翻译

我们介绍 Nemotron 3 Ultra，这是一个总参数 550B、每 token 激活参数 55B 的 Mixture-of-Experts Hybrid Mamba-Attention 语言模型。我们先在 20T 文本 tokens 上预训练 Nemotron 3 Ultra，随后将 context length 扩展到 1M tokens，并使用 Supervised Fine Tuning (SFT)、Reinforcement Learning (RL) 和 Multi-teacher On-Policy Distillation (MOPD) 进行 post-training。Nemotron 3 Ultra 是我们迄今能力最强的模型，采用了多项关键技术：LatentMoE、Multi Token Prediction (MTP)、NVFP4 pre-training、multi-environment RLVR、MOPD 和 reasoning budget control。Nemotron 3 Ultra 在达到同等 accuracy 的同时，相比公开可用的 SOTA LLMs 最高可实现约 6 倍 inference throughput。其 SOTA accuracy、高 inference throughput 和 1M token context length，使 Nemotron 3 Ultra 非常适合 long-running autonomous agentic tasks。我们在 HuggingFace 开源 base、post-trained 和 quantized checkpoints，以及 training data 和 recipe。

#### 研究背景

随着 LLM 应用从简单 chatbot 转向 long-running agents，例如自主写代码、研究、使用终端和工具完成复杂任务，模型不仅需要高 accuracy，还需要高 inference throughput、低 KV cache 成本、长上下文和稳定的 agentic reasoning 能力。Nemotron 3 Ultra 的核心目标是在 accuracy-throughput frontier 上前移：通过 Mixture-of-Experts (MoE，专家混合) 提升每个 active parameter 的能力，通过 Hybrid Mamba-Attention (混合 Mamba-注意力) 降低 attention cost 和 KV cache footprint，再结合 1M context、MTP speculative decoding、NVFP4 训练/量化和面向 agent 的 post-training pipeline。

论文把该模型定位为 Nemotron 3 系列中最大、最强的开放模型：550B total parameters、55B active parameters，开放 base、post-trained、NVFP4 quantized checkpoints，以及部分 training data、recipe 和 RL environments。研究背景中的关键问题不是单一 benchmark 追分，而是如何在超大模型规模上同时实现：长上下文、agentic tool use、reasoning、throughput、训练稳定性、可部署性和开放复现资源。

#### Method

- **模型架构**：Nemotron 3 Ultra 使用 Hybrid Mamba-Attention MoE 架构，共 108 层，model dimension 为 8192，每层 512 个 experts，Top-$k$ activated experts 为 22，MoE latent size 为 2048，总参数 550B、每 token 激活 55B。它沿用 Nemotron 3 Super 的 LatentMoE，并使用 shared-weight Multi-Token Prediction (MTP) heads 支持 speculative decoding。

- **NVFP4 pre-training**：模型在 20T text tokens 上预训练，采用 Warmup-Stable-Decay learning rate schedule。前 15T tokens 偏向 diversity 和 broad domain coverage，后 5T tokens 偏向 high-quality data。学习率 warmup 200B tokens 到峰值 $2.5 \times 10^{-4}$，最后 5T tokens decay 到 $2.5 \times 10^{-6}$。NVFP4 使用 E2M1 datatype、two-dimensional block quantization、Random Hadamard Transforms 和 stochastic rounding；论文保留最后 15% 网络、Mamba output projections、latent projections、QKV/attention projections、MTP layers 和 embedding layers 为更高精度，以维持稳定性。

- **预训练数据**：数据混合覆盖 web crawl、synthetic web、code、math、academic、legal、multilingual、Wikipedia、finepdfs 和 SFT-style data 等 19 类。论文新增或刷新了 GitHub code data、benchmark-oriented synthetic Q&A、fact-seeking data、moral scenarios、legal datasets 等。数据 curriculum 从 phase 1 的 diversity-biased mixture 转向 phase 2 的 quality-biased mixture。

- **Long-context extension**：预训练结束后做 33B tokens 的 long-context continuous pretraining。训练 blend 中 46% 是 long-context data，54% 是 phase 2 data；92% iterations 使用 1,048,576 token context，8% 使用 4K context 以维持短 benchmark 能力。论文明确未在 blend 中使用 RULER-style data。

- **SFT**：post-training 从 base model 开始做两阶段 SFT。Stage 1 使用 294,912 token packed sequence、global batch size 64、204,800 samples；Stage 2 扩展到 515,000 token packed sequence，并加入 512K long-context data。SFT 保留 shared-weight MTP objective，两个 MTP layers 的 auxiliary-loss scaling factor 为 0.1。SFT 数据覆盖 long context、reasoning efficiency/control、safety、多语言 safety、search、terminal use、conversational tool use、SWE issue resolution、math/proof、science、chat、code、CUDA、RTL 和 multilingual translation。

- **RLVR**：随后进行 unified RLVR，覆盖 terminal usage、office/productivity workflows、software engineering、search、tool-calling、math、code、STEM、safety、chat、instruction following、long-context QA、structured outputs 和 general usability。算法主要沿用 asynchronous GRPO，并使用 global batch size 8192，每个 sample 生成 16 rollouts；maximum generation length 从 48K tokens 增加到 64K tokens。

- **MOPD**：论文认为 mixed-environment RLVR 在大量 domains 混合时会稀释单域学习信号，因此训练十多个 specialized teacher models，并通过 Multi-teacher On-Policy Distillation 将能力合并到 student。MOPD 在 student-generated rollouts 上用对应 teacher 提供 dense token-level guidance，核心思想是在 prefix $s_t=(q,y_{<t})$ 上让 student 最小化 $D_{\mathrm{KL}}(\pi_\theta(\cdot \mid s_t)\|\pi_{T_i}(\cdot \mid s_t))$。异步实现中，rollout generation、teacher scoring 和 learner optimization fully pipelined；为了处理 stale behavior policy，使用 behavior-to-proximal ratio $c_t=\mathrm{sg}[\pi_{\mathrm{prox}}(y_t\mid s_t)/\pi_{\mathrm{behav}}(y_t\mid s_t)]$ 和 proximal-to-current ratio $r_t(\theta)=\pi_\theta(y_t\mid s_t)/\pi_{\mathrm{prox}}(y_t\mid s_t)$，并对 $r_t(\theta)$ 做 PPO-style clipping。MOPD 使用 192K maximum generation length、batch size 1024 prompts，每 prompt 一个 rollout；论文表示多 rollout 在 ablation 中没有额外收益。

- **MOPD warmup**：由于不同 teachers 和 student 的 SFT 数据/训练路径差异会造成 distribution mismatch，论文在 MOPD 前引入 very light SFT warmup，让 student 的 reasoning trajectories 与 teacher-supported distributions 更接近，从而提高 teacher supervision 可靠性。该 warmup 对 agentic domains 明显有效，对 HLE 这类 self-contained reasoning benchmark 收益很小。

- **Specialized teachers**：教师包括 SWE teacher、office/workplace task teacher、search teacher、terminal-use teacher、conversational tool-use teacher、model usability teacher、agentic safety teacher、chat teacher、instruction-following and factuality teacher、STEM/general reasoning teacher、competitive coding teacher 等。它们分别通过 SFT、PivotRL、RLVR、GenRM/RLHF 或 domain-specific data 训练。

- **MTP Boosting**：模型原生支持 MTP speculative decoding。论文指出 teacher-forced MTP training 与 autoregressive MTP inference 存在 train-inference mismatch：越深的 draft positions 会条件化在越来越多 MTP-generated hidden states 上。MTP Boosting 固定 backbone，只更新 MTP head，用 on-policy rollouts 和 temperature-scaled forward-KL loss 让 MTP head 匹配 backbone logits；使用 $T=2$、$N_{\mathrm{mtp}}=7$，训练 12K steps，global batch size 64，sequence cap 8K。

- **Reasoning efficiency/control**：Nemotron 3 Ultra 支持 reasoning-off、regular、medium-effort 三种 reasoning modes，并支持 inference-time budget control。medium-effort 在 SFT 中引入，在 RLVR 中优化，用于在 accuracy 和 token cost 之间做可控折中。

- **训练与推理基础设施**：论文还系统描述了大规模 RL/MOPD infrastructure：MTP 加速 rollout generation、Ray GCS/Slurm 启动优化、NVLink topology-aware placement、NUMA binding、async checkpointing、JIT cache warm-start、multi-node vLLM 稳定性修复、container/storage I/O 优化等。这部分强调 Nemotron 3 Ultra 不只是模型 recipe，也包括超大规模训练工程。

#### 论文自我评价的缺陷

- **MOPD 对 student support 依赖强**：论文明确指出，MOPD 最有效的场景是 student-generated trajectories 已经落在 teacher support 内，此时 teacher 能对 token-level preferences 给出可靠监督。若 teacher 的优势来自额外 off-policy SFT/RL 数据暴露，而 student 很少采样到相关 reasoning paths，则 MOPD 信号会变弱。这解释了 HLE no tools 的 recovery 只有 16.9%，LiveCodeBench v6 也只有 32.0%。

- **Teacher-student distribution mismatch**：不同 specialized teachers 与 student 往往由不同 SFT pipeline 训练，输出分布和 reasoning behaviors 不一致。论文因此引入 MOPD warmup，但 warmup 不是根本解决方案，对 HLE 这类 reasoning benchmark 收益很小。

- **Logit matching 未奏效**：论文尝试了 top-$k$ 或 full-vocabulary distribution-level distillation，但 preliminary experiments 中没有提升 MOPD，且在 Terminal Bench 等 agentic benchmark 上弱于 sampled-token objective。作者推测，在 student prefix 对 teacher 来说 off-support 时，teacher logits 可能校准较差，full-distribution matching 会放大噪声。

- **MOPD foundation 尚未系统研究**：论文提出两个可能方向：让 teacher 和 student 共享统一 SFT stage，或先用 specialized teachers 生成 SFT data 训练 student，再做 RL/MOPD。但由于 time and resource constraints，本文未系统评估这些替代方案。

- **Long-horizon MOPD 仍是开放问题**：agentic workflows 往往包含很多 tool calls 和环境交互，而 reasoning tasks 通常是 single-turn。混合 end-to-end agentic environments 与 reasoning environments 会导致 rollout time 差异巨大、训练效率低。论文实践中多数 agentic tasks 使用类似 PivotRL 的 single-turn rollouts，但是否 end-to-end rollouts 能进一步提升、如何避免 distribution mismatch，仍未解决。

- **预训练稳定性未完全解释**：论文报告两次 training divergence。第一次约 8T tokens 的 divergence 被归因于 output layer local gradient accumulation precision 从 FP32 降到 BF16，导致 MTP blocks 对 shared output layer 的 wgrad contribution 基本丢失；回滚并恢复 FP32 gradient reduction 后稳定。第二次约 16T tokens 的 divergence 原因未确定，只通过提前 learning-rate annealing 缓解，并最终把总预训练 horizon 缩短到 20T tokens。作者观察到 expert routing imbalance、dead experts 和 residual activation norm spikes 与不稳定性相关，但没有找到明确因果。

- **基础设施仍需改进**：论文的 future work 指向 fail-fast fault isolation、component-level recovery、generation worker 或 sandbox instance 独立重启、sandbox/tool-calling infrastructure disaggregation，以及对 in-flight rollouts、KV cache、conversation state 做 fine-grained checkpointing，说明现有大规模 RL/MOPD 系统仍有明显工程脆弱点。

#### 我的分析：优点

- **工程与方法整体性强**：论文不是只提出一个模型结构，而是完整覆盖 pretraining、long-context extension、SFT、RLVR、MOPD、MTP Boosting、reasoning budget control、serving throughput 和 RL infrastructure，展示了训练 agentic large model 所需的系统栈。

- **架构选择服务于部署效率**：Hybrid Mamba-Attention 降低 attention cost 和 KV cache footprint，MoE 提升 active parameter efficiency，MTP 支持 speculative decoding，NVFP4 进一步提升训练/推理效率。最终在 8K input / 64K output 场景下，相比 GLM、Kimi、Qwen 有明显 throughput 优势，这是 agentic long-running tasks 的关键瓶颈。

- **MOPD 的定位清晰**：论文没有把 MOPD 描述成万能蒸馏，而是明确其适用条件：当 student 能生成 teacher 可评价的轨迹时，dense token-level guidance 很有效。这与实验吻合：agentic/tool-use、abstention、instruction following 的 recovery 高，而 HLE 这种需要新 reasoning paths 的 benchmark recovery 低。

- **开放程度较高**：论文声称开放 base、post-trained、NVFP4 quantized checkpoints，以及 training data、recipes 和 RL environments。对 550B 级别模型而言，开放 base 与 post-trained checkpoints 的价值很高，也便于社区研究 Mamba-Attention MoE、NVFP4 training 和 MOPD。

- **对失败经验披露较多**：训练 divergence、BF16/FP32 gradient reduction 问题、MaxVio/expert imbalance、residual norm spikes、logit matching 失败、MOPD warmup 局限、RL infrastructure failure breakdown 等都写得比较具体。这些负结果对复现和后续研究很有价值。

- **agentic 安全问题有针对性**：Agentic Safety Teacher 专门针对 indirect prompt injection，通过读工具返回的恶意内容诱导敏感写工具调用，并用 deterministic verifier 判断是否抵抗攻击。这比只做普通 chat safety 更贴近 tool-using agents 的真实风险。

#### 我的分析：缺点 / 风险

- **评测可比性存在风险**：throughput 对比中 Nemotron 3 Ultra 使用 TRT-LLM，其他模型使用 vLLM；虽然论文说明选择各模型最佳 speculative decoding 设置，但推理栈不同会影响公平性。部分 benchmarks 还使用 internal scaffolding 或尚未完全 open-source 的实现，复现难度较高。

- **MOPD 依赖强教师和大量专用数据**：多个 teachers 使用 DeepSeek、MiniMax、GLM、GPT-OSS、internal strong model 或 vendor-curated data 生成/训练。最终能力可能高度依赖 teacher quality、数据许可、过滤策略和内部环境，开源权重不等于完整训练能力可复现。

- **on-policy distillation 不能替代能力注入**：MOPD 对 agentic 行为模式很有效，但对 HLE、hard reasoning、competitive coding 的 teacher gap recovery 较低。这意味着如果 student 从未学会某类推理路径，仅靠 teacher 对 student rollouts 打 token-level 分数可能不足，需要 off-policy SFT data、exploration 或更强的 curriculum。

- **训练稳定性仍有未解问题**：第二次 divergence 没有明确 root cause，expert imbalance 和 residual norm spikes 只是相关现象。550B MoE + Mamba + MTP + NVFP4 的组合在更长训练、更大规模或不同数据分布下是否稳定，仍有风险。

- **agentic benchmark 可能存在 harness overfitting**：论文投入大量 search、terminal、SWE、office、tool-use 专用数据和 teachers，这对目标 benchmarks 很有效，但也增加了对特定 harness、verifier、timeout、tool format 的适配风险。真实开放环境中的鲁棒性仍需要更多外部验证。

- **安全覆盖仍偏窄**：Agentic Safety Teacher 主要用 deterministic verifier 检查是否调用攻击者指定敏感工具和参数，但真实 indirect prompt injection 可能涉及更隐蔽的数据泄露、权限升级、跨工具组合攻击和长期状态污染。该机制是重要起点，但不能视为完整 agent safety 解决方案。

- **部署门槛高**：即使 active parameters 只有 55B，模型总参数 550B，且最佳 throughput 依赖 GB200、NVFP4、TRT-LLM、MTP 和复杂缓存/拓扑优化。对多数研究者或企业来说，使用和复现实验都需要高端硬件与工程能力。

- **长上下文结果需谨慎解读**：RULER @ 1M 分数很强，且论文强调未用 RULER-style data 做 LC-Phase，但 RULER 仍是合成式 long-context benchmark。真实 long-horizon agentic tasks 还涉及检索噪声、上下文压缩、工具错误和多轮状态管理，不能仅由 RULER 代表。
