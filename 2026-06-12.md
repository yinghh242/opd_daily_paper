#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy distillation。

#### Abstract 原文

> On-policy distillation (OPD) trains a student on its own trajectories with dense per-token supervision from a stronger teacher, and often outperforms off-policy distillation and standard reinforcement learning. However, we find that its effectiveness implicitly relies on two assumptions that frequently break in practice: trajectory-level alignment between the student and the teacher, and uniform token-level reliability of the teacher's preferences. We therefore propose Sign-Gated On-Policy Distillation (SG-OPD), which uses a binary verifier as a trust signal for the teacher at two complementary granularities: phased teacher sampling mixes in verifier-endorsed teacher rollouts at cold-start, and a sign-consistency gate extrapolates the distillation update on tokens where the teacher agrees with the verifier-correct direction and interpolates it where it disagrees. Experiments on competition-level mathematical reasoning benchmarks show that SG-OPD consistently outperforms standard OPD, with average gains of 1.98 and 7.50 at the per-sample and per-question levels, respectively.

#### 摘要中文翻译

On-policy distillation (OPD) 在学生模型自己的 trajectories（轨迹）上训练，并从更强的教师模型获得 dense per-token supervision（密集逐 token 监督），通常优于 off-policy distillation 和标准 reinforcement learning。然而，作者发现 OPD 的有效性隐含依赖两个在实践中经常失效的假设：学生与教师之间的 trajectory-level alignment（轨迹级对齐），以及教师偏好在 token-level（逐 token 层面）上的一致可靠性。因此，论文提出 Sign-Gated On-Policy Distillation (SG-OPD)，它使用 binary verifier（二元验证器）作为教师可信度信号，并在两个互补粒度上发挥作用：phased teacher sampling 在 cold-start（冷启动）阶段混入 verifier-endorsed teacher rollouts（验证器认可的教师轨迹）；sign-consistency gate 则在教师与 verifier-correct direction（一致的验证器正确方向）一致的 token 上 extrapolate（外推）distillation update，在二者不一致的 token 上 interpolate（插值/缓和）该 update。竞争级数学推理 benchmarks 上的实验表明，SG-OPD 持续优于标准 OPD，在 per-sample 和 per-question 层面分别带来平均 1.98 和 7.50 的提升。

#### 研究背景

本文关注 strong-to-weak on-policy distillation：用更强的 teacher model 指导较弱的 student model 学习数学推理能力。传统 off-policy distillation 通常训练学生模仿教师生成的 trajectories，但推理时学生会落在自己的分布上，导致 exposure bias。OPD 通过让学生在自己的 on-policy trajectories 上采样，并最小化与教师分布的 reverse KL，缓解了训练-推理分布不匹配问题。

论文认为，OPD 的成功隐含依赖两个假设，但这两个假设在强教师到弱学生的设置中并不稳固：

1. trajectory-level alignment 假设：学生早期生成的轨迹应与教师支持的轨迹足够接近。实际 cold-start 阶段弱学生经常生成教师也不认可的轨迹，使 reverse-KL 信号被噪声主导。
2. token-level teacher reliability 假设：教师对每个 token 的偏好都可靠。实际即使一个 rollout 的最终答案被 verifier 判为正确，教师仍可能在部分中间 token 上给出与正确轨迹方向相反的偏好。

与 G-OPD / ExOPD 使用全局 extrapolation factor 不同，本文的核心思想是：不要无差别放大教师信号，而是用 verifier 判断教师信号在不同 trajectory 和 token 上是否可信。SG-OPD 因此引入两个机制：sample-level 的 Phased Teacher Sampling (PTS) 解决 early-stage trajectory mismatch；token-level 的 Sign-Consistency Gate 解决教师逐 token 信号可靠性不均匀的问题。

#### Method

SG-OPD 将 binary verifier 作为 teacher trust signal，而不是直接把 verifier reward 当作唯一优化目标。设 student 为 $\pi_\theta$，frozen teacher 为 $\pi^*$，reference policy 为 $\pi_{\mathrm{ref}}$，prompt 为 $x$，student rollout 为 $y=(y_1,\ldots,y_T)$，verifiable reward 为 $r(x,y)\in\{0,1\}$。

首先，论文定义 verifier-side 的 outcome-level advantage：

$$
a_1(t)=\frac{r(x,y)-\mu_x}{\sigma_x+\epsilon}
$$

其中 $\mu_x$ 和 $\sigma_x$ 是同一 prompt 的 $G$ 个 rollouts 上的 reward 均值和标准差。由于 reward 是 binary，$a_1(t)$ 在同一条 trajectory 内是常数；作者只把它当作 teacher trust signal，而不是单独的优化目标。

OPD 的逐 token reverse-KL advantage 为：

$$
a_2(t)=\log \pi_\theta(y_t)-\log \pi^*(y_t)
$$

它对应 OPD 的 dense per-token policy gradient 信号。SG-OPD 的关键是比较 verifier-side signal 与 teacher-side OPD signal 的符号是否一致：

$$
g_t=\mathbf{1}[a_1(t)\cdot a_2(t)>0]
$$

当 $g_t=1$ 时，称为 consensus token，表示 verifier 和 teacher 在该 token 上方向一致；当 $g_t=0$ 时，称为 conflict token，表示 teacher 的逐 token 偏好可能与 verifier-correct direction 冲突。

在 consensus tokens 上，SG-OPD 使用更强的 G-OPD / ExOPD extrapolation：

$$
A_t^{\mathrm{cons}}=A_t^{\mathrm{G\text{-}OPD}}(\lambda_{\mathrm{high}})
$$

其中 $\lambda_{\mathrm{high}}>1$。论文使用的 G-OPD advantage 形式为：

$$
A_t^{\mathrm{G\text{-}OPD}}(\lambda)
=
a_2(t)+(\lambda-1)(\log \pi_{\mathrm{ref}}(y_t)-\log \pi^*(y_t))
$$

在 conflict tokens 上，默认采用 softened OPD fallback：

$$
A_t^{\mathrm{conf}}=\beta a_2(t), \quad \beta\in[0,1]
$$

最终 routed advantage 为：

$$
A_t^{\mathrm{SG}}
=
g_t A_t^{\mathrm{cons}}+(1-g_t)A_t^{\mathrm{conf}}
$$

直观上，SG-OPD 在 teacher 和 verifier 一致时大胆 extrapolate，在二者冲突时降低 teacher 信号权重，从而避免盲目放大错误或不可靠的 token-level supervision。

另一个机制是 Phased Teacher Sampling (PTS)。在 mini-batch 中，SG-OPD 保留一部分比例 $\rho$ 的 prompts 交给 teacher 采样，得到 teacher rollouts；只保留被 verifier 判为正确的 teacher trajectories，并用 cross-entropy 作为 auxiliary teacher-anchor loss：

$$
L_{\mathrm{SLT}}
=
\sum_{(x,y^T)\in B_T,\ r(x,y^T)=1}
\sum_t
L_{\mathrm{CE}}(\pi_\theta,y_t^T)
$$

该 anchor 只在 cold-start 阶段起作用，权重 $\alpha(t)$ 通过三阶段 cosine schedule 逐渐退火到 0。这样早期用 verified teacher rollouts 把学生拉到更合理的 trajectory neighborhood，后期则恢复 fully on-policy training，避免 teacher trajectories 长期污染学生自己的分布。

整体目标为：

$$
L_{\mathrm{SG\text{-}OPD}}(\theta)
=
L_{\mathrm{TLT}}(\theta)+\alpha(t)L_{\mathrm{SLT}}(\theta)
$$

其中 token-level loss 使用 routed advantage $A_t^{\mathrm{SG}}$，并引入 detached stability weight $\phi_t=\min(1,\tau/|a_2(t)|)$ 来抑制极端 OPD outliers。实现上，作者在 verl framework 中基于 GRPO、PPO-clipped actor loss、teacher sampling path 和 sign-gated reverse-KL advantage 完成训练。

#### 论文自我评价的缺陷

论文单列了 Limitations，主要包括以下几点。

第一，验证范围局限于 math-only binary verifiable reward。所有实验都在 competition-level math reasoning 上进行，并使用 binary trajectory-level verifier。虽然作者认为 token-level gate 可以自然扩展到 code with unit tests、tool use、reward-model verification 等 verifier-style tasks，但本文没有验证这些场景。

第二，当前 $a_1(t)$ 来自 trajectory-level binary reward，因此在同一 rollout 内是常数。这会带来一个风险：如果最终答案错误，整条 trajectory 的 verifier sign 可能为负，gate 可能抑制某些其实有价值的 intermediate reasoning tokens。作者认为未来可以用 process-level 或 step-level certificate 替代 trajectory-level sign。

第三，scale 和 schedule transfer 尚未验证。主要实验只使用一个 Qwen3-1.7B student / Qwen3-4B-RL-Math teacher pair，并采用固定的 100-step schedule。conflict fraction、safe extrapolation range、PTS phase boundaries 是否能稳定迁移到更大模型、更长训练、更不同的任务，仍是开放问题。

第四，compute footprint 方面，sign-consistency gating 相比 OPD 几乎不增加计算，但 PTS 在 warm-up 阶段需要 teacher rollouts。尽管 auxiliary term 会退火到 0，使后期成本接近 OPD，但早期仍需要额外 teacher inference。

第五，reproducibility 上作者承认主表每行来自 single training seed。avg@32 虽然在 evaluation 时平均 32 个 sampling seeds，但没有充分刻画 training seed variance。作者基于 50+ run sweep 认为 +1.98 AVG 超过噪声，但 formal multi-seed study 留待未来工作。

#### 我的分析：优点

1. 问题定位清晰：论文没有简单宣称 OPD 有效，而是把 OPD 的失败拆成 trajectory-level mismatch 和 token-level teacher unreliability 两个层次，对应提出 PTS 和 Sign-Consistency Gate，方法结构与 failure modes 对齐较好。

2. 方法与用户当前关注的 unreliable teacher supervision 很相关：SG-OPD 的核心不是盲目信任 stronger teacher，而是用 verifier 判断 teacher signal 何时可信；这与 token-level filtering / calibration 的方向高度一致。

3. token-level sign gating 设计直观且轻量：只比较 $a_1(t)$ 和 $a_2(t)$ 的符号，不需要额外训练 reward model 或 process reward model，也不需要 full-vocabulary KL，因此实现成本相对低。

4. 结果不仅报告主指标，还做了比较完整的 ablations：包括 Sign-Gate only、PTS only、full SG-OPD、uniform ExOPD collapse、time-separated SFT control、fallback modes、PTS schedule、teacher correctness filter 等，能较好支持两个机制互补这一主张。

5. 对 aggressive extrapolation collapse 的解释有说服力：uniform $\lambda=1.8$ 会崩到 AVG 24.71，而 sign-gated $\lambda_{\mathrm{high}}=1.8$ 能恢复到 28.78 或 29.53，说明 token-level routing 确实扩大了安全 extrapolation 区间。

6. 训练动态分析比较重要：SG-OPD 同时提高 training reward 和 policy entropy，说明它不只是通过降低熵、收缩到少数模板答案来提升 pass@32，而可能保留了更好的 exploration。

7. 论文诚实报告了 HMMT25-Feb 上不如 ExOPD 的情况，并尝试解释 gate 可能抑制 benign tokens。这比只强调平均提升更可信。

#### 我的分析：缺点 / 风险

1. 最大风险是 verifier granularity 不足。当前 $a_1(t)$ 是 trajectory-level binary signal，却被用于 token-level gating。最终答案正确不代表每个中间 token 都好，最终答案错误也不代表每个中间 token 都坏。因此该 gate 可能把 trajectory-level credit assignment 问题转移到 token-level，存在误抑制或误放大的风险。

2. single-seed 主结果削弱了结论强度。SG-OPD 相比 OPD 的 avg@32 提升是 +1.98，虽然作者说 sweep 噪声小于约 0.5，但正式 multi-seed 仍缺失；对于数学 benchmark，训练和采样方差可能影响结论。

3. 方法对 schedule 和超参可能敏感。PTS 的 $P_1/P_2=30/35$ 表现最好，短窗口、长窗口、较大 teacher sampling ratio 都会退化。这说明方法虽然概念简单，但实际迁移时可能需要重新调 phase boundaries、$\rho$、$\lambda_{\mathrm{high}}$ 和 $\beta$。

4. 当前只验证 Qwen3-1.7B 到 Qwen3-4B 的一个 teacher-student pair。strong-to-weak gap 更大、teacher 更强但风格更不同、student/tokenizer 不同、或者 teacher 本身不稳定时，$a_2(t)$ 的含义和 conflict fraction 可能明显变化。

5. 数学任务具有天然 binary verifier，推广到开放式 reasoning、instruction following、long-form generation 或 subjective preference alignment 并不直接。若 verifier 是 reward model 而非确定性判题器，$a_1(t)$ 的噪声会直接影响 sign gate。

6. PTS 依赖 teacher rollouts 被 verifier 正确筛选。若 teacher 在目标任务上正确率不够高，或 verifier 有 false positives / false negatives，sample-level anchor 可能引入系统性偏差。

7. HMMT25-Feb 上落后 ExOPD 暗示 gate 对 multi-stage reasoning chain 可能过度保守。复杂推理中某些看似与最终 reward sign 冲突的 token 可能是探索必要步骤，简单 sign consistency 可能不足以判断 token 的真实贡献。

8. 论文的若干文字和排版存在小问题，例如局部句子重复、语法错误、公式抽取显示异常等；虽然不影响核心思想，但会影响可读性和正式发表质量。
