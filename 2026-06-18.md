#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy self-distillation。

#### Abstract 原文

> On-policy self-distillation (OPSD) trains a model on its own rollouts and uses a frozen copy to provide dense token-level targets conditioned on a reference target. This works well for LLM reasoning, but a direct extension to multimodal large language models (MLLMs) can create a shortcut: the privileged target may guide tokens mainly based on the text reference target rather than the image. We propose ViGOS, a visually grounded OPSD framework for MLLM post-training. The student first writes a visual description and then reasons toward the final answer. For valid rollouts, an image-only perception teacher supervises the description, while a privileged reasoning teacher supervises the reasoning and final answer on the same student prefix. A reference teacher is used only for invalid rollouts to recover the output format. Across general vision-language, expert reasoning, visual math, spatial grounding, and visual-language-prior benchmarks, ViGOS keeps the main benefits of OPSD and improves image-grounded behavior in shortcut-prone settings.

#### 摘要中文翻译

On-policy self-distillation（OPSD，on-policy 自蒸馏）让模型在自己的 rollouts（采样轨迹）上训练，并使用一个 frozen copy（冻结副本）在给定 reference target（参考目标）的条件下提供 dense token-level targets（密集 token 级目标）。这种方法在 LLM reasoning（大语言模型推理）中效果很好，但直接扩展到 multimodal large language models（MLLMs，多模态大语言模型）时可能产生一种 shortcut（捷径）：privileged target（特权目标）可能主要基于文本 reference target 而不是图像来引导 token。本文提出 ViGOS，一个用于 MLLM post-training（后训练）的 visually grounded OPSD（视觉扎根 OPSD）框架。学生模型首先写出 visual description（视觉描述），然后再推理得到最终答案。对于 valid rollouts（有效轨迹），一个 image-only perception teacher（仅图像感知教师）监督描述部分，而 privileged reasoning teacher（特权推理教师）在相同 student prefix（学生前缀）上监督推理和最终答案。reference teacher（参考教师）只用于 invalid rollouts（无效轨迹），以恢复输出格式。在 general vision-language、expert reasoning、visual math、spatial grounding 和 visual-language-prior benchmarks 上，ViGOS 保留了 OPSD 的主要收益，并在容易出现 shortcut 的设置中提升了 image-grounded behavior（基于图像的行为）。

#### 研究背景

OPSD 的核心优势是缓解普通 distillation（蒸馏）中的 train-test gap（训练-测试分布差异）：学生不是只学习固定 teacher trajectory（教师轨迹），而是在自己生成的 on-policy prefixes（当前策略前缀）上接受 dense token-level supervision（密集 token 级监督）。在文本 LLM reasoning 中，frozen teacher 可以看到 reference answer 或 reference solution，从而在学生自己的推理状态上提供更细粒度的纠偏信号。

但论文指出，将这一机制直接搬到 MLLM 会引入新的 multimodal shortcut（多模态捷径）。学生在训练和推理时只能看到 image $I$ 和 question $x$，而 privileged teacher 在训练监督时还能看到 reference answer $a^\star$。对于 MLLM，文本信号往往比图像信号更容易被利用，因此 teacher 可能在学生尚未真正检查图像证据前，就根据 $a^\star$ 推动其生成与答案一致的 rationale（理由）。这会形成类似：

$$
a^\star \to r \to a
$$

的 answer-driven path（答案驱动路径），导致模型学会“看似合理但视觉 grounding 不足”的推理。

为量化这一问题，论文提出 Privileged Answer Leakage Rate（PALR，特权答案泄漏率）。它固定学生 rollout 和 prefixes，只改变 teacher context，比较 dense correction 中有多少来自 privileged answer 而不是 image。PALR 越高，说明 token-level correction 越受 reference answer 驱动。论文在 Qwen2.5-VL 诊断子集上观察到 vanilla OPSD 的 reasoning-answer segment 存在明显 answer leakage：3B 的 $PALR(T_{ra})$ 为 17.26%，7B 为 26.01%。

这说明问题不是 OPSD 本身无效，而是 vanilla OPSD 把 answer-conditioned teacher 用在整个 trajectory 上，导致 visual grounding 和 answer-guided reasoning 被混在同一个监督信号中。ViGOS 的目标就是保留 answer-guided reasoning 的好处，同时控制 $a^\star$ 进入训练路径的位置。

#### Method

ViGOS（Visual Grounding On-Policy Self-Distillation）的基本思想是将 perception（感知）和 reasoning（推理）解耦。学生模型不再直接生成普通答案，而是被要求生成结构化序列：

$$
y = (d, r, a)
$$

其中 $d$ 是 visual description，$r$ 是 reasoning process，$a$ 是 final answer。输出格式为：

`<description> d </description>  a`

学生在 rollout 时仍然只接收 image $I$、question $x$ 和 output-format prompt，不接收 reference answer $a^\star$。训练时从学生当前策略采样：

$$
y \sim p_\theta(\cdot \mid c_{\text{stu}})
$$

其中：

$$
c_{\text{stu}} = (I, x, \pi_{\text{out}})
$$

解析器随后提取三个 token set：description tokens $T_d$、reasoning tokens $T_r$、answer tokens $T_a$，并定义 $T_{ra} = T_r \cup T_a$。如果 delimiter 缺失、description 或 reasoning 为空、final answer 不可解析，则该 rollout 被标记为 invalid：

$$
m_{\text{inv}}(y) = 1[y \notin Y_{\text{valid}}]
$$

ViGOS 使用同一个 frozen MLLM 的三个 teacher roles，但它们的 external context 不同：

1. Image-only perception teacher：只接收 image 作为外部证据，监督 description segment。
2. Privileged reasoning teacher：接收 image、question、reference answer 和 reasoning prompt，监督 reasoning 与 answer segment。
3. Reference teacher：只在 invalid rollout 时启用，用于恢复结构化输出格式。

对应 context 为：

$$
c_{\text{img}} = I
$$

$$
c_{\text{rea}} = (I, x, a^\star, \pi_{\text{rea}})
$$

$$
c_{\text{ref}} = (I, x, a^\star, \pi_{\text{out}})
$$

在同一个 student prefix $h_t = (y_1, \ldots, y_{t-1})$ 上，学生与教师分布为：

$$
p_{\theta,t}(\cdot) = p_\theta(\cdot \mid c_{\text{stu}}, h_t)
$$

$$
q_{\text{img},t}(\cdot) = p_{\bar{\theta}}(\cdot \mid c_{\text{img}}, h_t)
$$

$$
q_{\text{rea},t}(\cdot) = p_{\bar{\theta}}(\cdot \mid c_{\text{rea}}, h_t)
$$

$$
q_{\text{ref},t}(\cdot) = p_{\bar{\theta}}(\cdot \mid c_{\text{ref}}, h_t)
$$

对于 valid rollout，description tokens 使用 perception loss：

$$
L_{\text{perc}} =
\mathbb{E}_{D,p_\theta}
\left[
(1 - m_{\text{inv}}(y))
\sum_{t \in T_d}
D_{\mathrm{KL}}(q_{\text{img},t} \parallel p_{\theta,t})
\right]
$$

由于 $q_{\text{img}}$ 的外部上下文只有图像，不包含 question、answer options 或 $a^\star$，它不会直接把答案信号注入 description segment。

对于 reasoning 和 answer tokens，使用 reasoning loss：

$$
L_{\text{rea}} =
\mathbb{E}_{D,p_\theta}
\left[
(1 - m_{\text{inv}}(y))
\sum_{t \in T_r \cup T_a}
D_{\mathrm{KL}}(q_{\text{rea},t} \parallel p_{\theta,t})
\right]
$$

该 teacher 可以看到 $a^\star$，但只在学生已经生成 description prefix 之后监督后续推理，因此训练路径从 vanilla OPSD 的：

$$
a^\star \to r \to a
$$

变成 ViGOS 希望的：

$$
I \to d \to r \to a
$$

对于 invalid rollout，ViGOS 不使用不可靠的 segment masks，而是启用 reference fallback loss，并采用 reverse KL：

$$
L_{\text{ref}} =
\mathbb{E}_{D,p_\theta}
\left[
m_{\text{inv}}(y)
\sum_{t \in T_y}
D_{\mathrm{KL}}(p_{\theta,t} \parallel q_{\text{ref},t})
\right]
$$

最终目标为：

$$
L_{\text{ViGOS}} =
\lambda_{\text{perc}} L_{\text{perc}}
+
\lambda_{\text{rea}} L_{\text{rea}}
+
\lambda_{\text{ref}} L_{\text{ref}}
$$

推理阶段所有 teachers、reference solutions 和 segment masks 都被移除，最终模型只接收 image、question 和 prompt，并生成结构化输出。

#### 论文自我评价的缺陷

论文未单列 Limitations 小节，但在 Conclusion 中明确承认了方法限制：

1. Generated description 可能不完整或错误。ViGOS 依赖学生先生成 visual description，如果该 description 漏掉关键视觉证据、OCR 错误、空间关系误读，后续 reasoning teacher 即使只在 $d$ 之后介入，也可能基于错误前缀继续推理。
2. Image-only perception teacher 可能生成 generic descriptions（泛化、模板化描述）。由于 perception teacher 不接收 question $x$ 和 reference answer $a^\star$，它可能只给出通用图像描述，而不一定突出解题所需的 question-relevant evidence。
3. 训练需要额外 teacher forward passes。ViGOS 至少涉及 image-only perception teacher、privileged reasoning teacher，以及 invalid rollout 时的 reference teacher，因此训练成本高于普通 SFT 或更简单的 OPSD。
4. 论文还隐含承认 ViGOS 没有消除所有 shortcut。PALR 只是 counterfactual diagnostic，不是完整因果归因；$PALR(T_d)=0$ 也是由 image-only teacher 设计保证的，并不意味着训练后的 student 在所有情况下都没有 prior shortcut。
5. 一些主 benchmark 上 Avg@5 仍会下降。例如 7B 的 MathVista、MMSI、CV-Bench 上，ViGOS 的 Avg@5 不一定优于 Baseline，说明方法提升 sampled success 的同时，回答稳定性和复杂推理一致性仍需优化。

#### 我的分析：优点

1. 问题定义清楚，抓住了 MLLM OPSD 与 text-only OPSD 的关键差异。文本 LLM 中 reference answer 作为 privileged context 较自然，但 MLLM 中图像与文本存在竞争路径，答案文本更容易成为 shortcut。论文没有简单否定 OPSD，而是指出“answer guidance 有用，但必须控制进入位置”，这是一个合理且细致的判断。

2. ViGOS 的设计很直接：先看图，再推理。通过强制学生生成 $d$，并用 image-only teacher 监督 $T_d$，再让 privileged reasoning teacher 监督 $T_r \cup T_a$，方法结构与目标高度一致。其训练路径：

$$
I \to d \to r \to a
$$

比 vanilla OPSD 的潜在路径：

$$
a^\star \to r \to a
$$

更符合视觉推理任务的因果顺序。

3. PALR diagnostic 是论文较有价值的分析工具。它固定 student rollout，只改变 teacher context，从 token-level correction 角度衡量 answer-driven support 与 image-driven support。虽然不是完整归因，但比单纯报告 benchmark accuracy 更能解释 OPSD 在 MLLM 中为什么会产生 shortcut。

4. 实验覆盖面较完整。论文不仅报告 MM-Vet、MMMU、MathVerse、MathVista、MMSI、RealWorldQA、CV-Bench 等常规 benchmark，还加入 ViLP 这种 prior-conflict benchmark，并做 PALR、ablation、same-prompt control 和 qualitative analysis。尤其 same-prompt experiment 很重要，因为 ViGOS 的 structured prompt 本身就可能提升表现；论文用同 prompt 下 Baseline / OPSD / ViGOS 对比，说明收益不只是 prompt engineering。

5. 对 invalid rollout 的处理务实。结构化输出方法常见问题是格式崩坏；ViGOS 不把 reference teacher 用作默认全程教师，而只在 invalid rollout 中用 reverse KL fallback，避免重新引入完整 trajectory 的 privileged answer leakage。这一点设计比较克制。

6. 结果支持核心论点。ViGOS 在 ViLP 上同时提高 Score 并保持 Prior，说明它不是简单削弱 prior knowledge，而是更好地区分何时应相信图像、何时可保留常识 prior。qualitative examples 也显示 OPSD 常常“看到了视觉线索但最终答案仍回到 prior”，这与论文的 shortcut 叙事一致。

#### 我的分析：缺点 / 风险

1. Description bottleneck 可能成为新的误差源。ViGOS 把视觉证据显式压缩到 $d$，但如果 $d$ 错误，后续 $r$ 和 $a$ 会被错误 grounding 牵引。对于复杂图表、多对象空间关系、细粒度 OCR 或医学/科学图像，description 可能不足以承载所有必要证据。

2. Image-only perception teacher 不看 question，可能导致监督不够 task-relevant。论文这样做是为了避免 question text、answer options 和 $a^\star$ 泄漏，但代价是 teacher 可能监督大量与问题无关的视觉细节，而没有突出真正关键的证据。对于需要问题引导注意力的图像任务，这可能降低效率。

3. Structured prompt 增加输出长度和推理成本。ViGOS 推理时也要求生成 description 和 reasoning，再给 final answer。虽然不需要 teachers，但生成长度明显增加，可能影响实际部署中的 latency、cost 和 context budget。

4. Benchmark 上并非全面优于 OPSD 或 Baseline。3B 上部分主 benchmark Pass@5 低于 OPSD；7B 上部分 Avg@5 低于 Baseline 或 OPSD。这说明 ViGOS 更像是为 shortcut-prone / image-grounded 场景做 tradeoff，而不是无条件提升所有 multimodal reasoning 能力。

5. PALR 是有用但有限的诊断。它依赖 wrong-answer counterfactuals、mismatched image counterfactuals 和 observed token likelihood，对 counterfactual 构造方式敏感。它衡量的是特定 teacher correction 中 answer-vs-image sensitivity 的比例，不等价于模型真实决策因果机制的完整解释。

6. 训练成本较高。相比 vanilla OPSD，ViGOS 需要不同 teacher contexts 的额外 forward passes，还要解析 segment masks 和处理 invalid rollouts。对于大规模 MLLM post-training，额外成本可能明显。

7. 方法依赖格式解析与 prompt compliance。若模型在早期训练中频繁输出 malformed structure，fallback 会更常启用；如果 fallback 权重或 KL 方向设置不佳，可能重新放大 reference target 的影响，或者让模型过度学习格式而非视觉 grounding。

8. 可能存在“描述看似 grounded、推理仍走 prior”的残余风险。即使 description segment 由 image-only teacher 监督，后续 privileged reasoning teacher 仍能看到 $a^\star$，并监督 $T_r \cup T_a$。论文显示 $PALR(T_{ra})$ 降低但并非为零，这意味着 answer guidance 仍可能在后续推理中过强，尤其当 $d$ 模糊或不完整时。

### 2. Trust the Right Teacher: Quality-Aware Self-Distillation for GUI Grounding

- HF Daily date: 2026-06-18
- HF Paper: https://huggingface.co/papers/2606.18101
- arXiv: https://arxiv.org/abs/2606.18101
- PDF: https://arxiv.org/pdf/2606.18101
- Authors: Jingyuan Huang, Zuming Huang, Yucheng Shi, Tianze Yang, Xiaoming Zhai, Wei Chu, Ninghao Liu
- 作者单位：University of Georgia; INFLY Tech; Tencent AI Lab; The Hong Kong Polytechnic University。脚注说明：Jingyuan Huang 的工作在 INFLY Tech 实习期间完成。
- arXiv published: 2026-06-16
- arXiv updated: 2026-06-17
- Categories: cs.AI

#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy self-distillation。

#### Abstract 原文

> Graphical user interface (GUI) grounding requires vision-language models (VLMs) to identify small target elements in high-resolution screenshots and predict precise screen coordinates. On-policy self-distillation (OPSD) is a promising post-training approach for this coordinate-sensitive task, since it provides dense token-level teacher signals beyond hard coordinate labels. However, naive OPSD is not well suited to GUI grounding: OPSD evaluates the teacher on student-generated prefixes, the quality of coordinate-token teacher signals can degrade when the prefix has already deviated from the target coordinate, leading to unreliable teacher signal. To mitigate this, We propose quality-aware self-distillation for VLM-based GUI grounding, which improves coordinate-token teacher-signal quality through soft correctness-aware gating and teacher-probability scaling. The soft correctness-aware gate checks whether the teacher's current coordinate-token prediction can still be completed into the ground-truth box under the student-generated prefix. If not, the corresponding teacher signal is down-weighted. Teacher-probability scaling then uses the teacher's confidence as a lightweight factor to further calibrate the strength of the gated supervision. A key empirical finding is that neither component alone improves overall performance, whereas combining them consistently improves performance. This suggests that the two mechanisms play complementary roles: correctness-aware gating suppresses unreliable coordinate-token supervision, while teacher-probability scaling calibrates the strength of the remaining signals. Experiments across six GUI grounding benchmarks show that our method consistently improves the base model and outperforms strong baselines.

#### 摘要中文翻译

Graphical user interface (GUI) grounding 要求 vision-language models (VLMs，视觉语言模型) 在高分辨率截图中识别小的目标元素，并预测精确的屏幕坐标。On-policy self-distillation (OPSD，在线策略自蒸馏) 是一种很有前景的 post-training 方法，适用于这种对坐标敏感的任务，因为它能提供超过硬坐标标签的 dense token-level teacher signals（密集 token 级教师信号）。然而，naive OPSD 并不适合 GUI grounding：OPSD 会在 student-generated prefixes（学生生成前缀）上评估 teacher，当前缀已经偏离目标坐标时，coordinate-token teacher signals（坐标 token 教师信号）的质量可能下降，从而产生不可靠的 teacher signal。

为缓解这一问题，本文提出用于 VLM-based GUI grounding 的 quality-aware self-distillation（质量感知自蒸馏），通过 soft correctness-aware gating（软正确性感知门控）和 teacher-probability scaling（教师概率缩放）提升 coordinate-token teacher-signal quality。soft correctness-aware gate 会检查在 student-generated prefix 下，teacher 当前的 coordinate-token prediction 是否仍能被补全为 ground-truth box 内的坐标。如果不能，相应 teacher signal 会被降权。随后，teacher-probability scaling 使用 teacher confidence（教师置信度）作为轻量因子，进一步校准 gated supervision（门控后的监督）强度。

一个关键经验发现是：两个组件单独使用都不能提升整体性能，但二者结合后能稳定提升性能。这说明两个机制具有互补作用：correctness-aware gating 抑制不可靠的 coordinate-token supervision，而 teacher-probability scaling 校准剩余信号的强度。六个 GUI grounding benchmarks 上的实验表明，本文方法能持续提升 base model，并优于强 baseline。

#### 研究背景

GUI grounding 是 GUI agents、computer-use agents 和 mobile/web automation 中的基础能力：给定截图和自然语言指令，模型需要定位目标界面元素并输出点击坐标。其难点在于高分辨率 GUI 中目标元素往往很小、相似元素密集、视觉差异细微，因此最终坐标 token 的细粒度学习信号非常关键。

现有 post-training 方法存在局限：

- SFT 使用 annotated instruction-coordinate pairs，将坐标作为 hard target（硬标签）学习，训练稳定但只提供最终答案，缺少 teacher uncertainty 和 “dark knowledge”（暗知识），难以表达多个 plausible locations（可能位置）之间的偏好。
- GRPO / RL 方法使用 outcome-level reward（结果级奖励），需要多次 rollouts，监督稀疏，训练成本高，对坐标这种细粒度输出的 token-level guidance 不足。
- Naive OPSD 相比 SFT 和 RL 有优势，因为它在 student-generated trajectories 上用 teacher distributions 提供 dense token-level supervision，并更接近 autoregressive inference 过程。但 GUI 坐标是自回归生成的，如果学生前缀已经偏离正确坐标，teacher 在这个错误前缀条件下给出的后续 logits 可能只是“错误坐标的合理延续”，而不是指向真实目标的可靠信号。

本文的核心动机是：GUI grounding 具有 spatial verifiability（空间可验证性）。训练时有 ground-truth bounding box，因此可以直接判断某个 coordinate-token teacher signal 在当前前缀下是否仍有可能补全到真实框内。相比 entropy、perplexity、teacher probability 等间接 proxy，这是一种更直接的 teacher-signal reliability 判断依据。

#### Method

本文提出 quality-aware self-distillation，用于提升 OPSD 中 coordinate-token teacher signals 的可靠性。方法主要包含 privileged teacher input、soft correctness-aware gating、teacher-probability scaling 和 weighted reverse-KL objective。

首先，对每个训练样本，给定 GUI instruction $x$、原始截图 $I$ 和 ground-truth bounding box $B$。student 只看到原始输入 $(x,I)$。teacher 在训练时看到 privileged input $(x^+, I^+)$：目标区域被保留并高亮，其他区域通过 layout-preserving regional masking / Gaussian soft mask 被弱化，同时 prompt 中加入 “answer is located within the green rectangle” 这类提示。推理时不使用 privileged information，只有 student 使用原始截图和原始指令。

on-policy self-distillation 的分布定义为：

$$
y \sim \pi_\theta(\cdot \mid x,I), \quad
P_S^t = \pi_\theta(\cdot \mid x,I,y_{<t}), \quad
P_T^t = \mathrm{sg}\left[\pi_\theta(\cdot \mid x^+,I^+,y_{<t})\right].
$$

其中，$y$ 是 student sampled response，$P_S^t$ 是 student 在当前 student-generated prefix 上的分布，$P_T^t$ 是 privileged teacher 在同一前缀上的分布，$\mathrm{sg}[\cdot]$ 表示 stop-gradient。distillation 只更新 student，teacher signal 不反传。

对 coordinate-token position $t$，设 $D$ 为 coordinate digit tokens 集合，teacher 的 top coordinate digit 为：

$$
d_t^\star = \arg\max_{d \in D} P_T^t(d).
$$

然后进行 prefix-aware correctness check。设 $a(t) \in \{x,y\}$ 表示当前位置属于横坐标还是纵坐标，$B_{a(t)}$ 表示 ground-truth box 在对应轴上的区间。方法将 $d_t^\star$ 追加到当前 axis-specific prefix 后，判断剩余 digit 是否仍能补全为落在 $B_{a(t)}$ 内的合法坐标。如果可以，则：

$$
h_t = 1.
$$

否则：

$$
h_t = 0.
$$

为了避免 hard filtering 过于激进，本文将二值兼容性转为 soft correctness-aware gate：

$$
g_t = \alpha + (1-\alpha)h_t.
$$

当 teacher 当前 coordinate token 与目标框兼容时，$g_t=1$；当不兼容时，$g_t=\alpha$。主实验中设置 $\alpha=0.5$，即 failed-gate teacher signals 不被完全丢弃，而是降权一半。

仅有 correctness-aware gating 仍不足，因为它不刻画 teacher uncertainty。两个都能通过 gate 的 token，teacher confidence 可能差异很大。因此本文再使用 teacher-probability scaling：

$$
p_t = P_T^t(d_t^\star).
$$

$p_t$ 越大，表示 teacher 对当前 top coordinate token 越有信心，对应 distillation term 权重越大；$p_t$ 越小，则监督被进一步软化。

最终 token-level distillation weight 为：

$$
w_t = (1-r_t) + r_t g_t \lambda p_t.
$$

其中，$r_t$ 表示位置 $t$ 是否为 coordinate token。若不是 coordinate token，则 $r_t=0$ 且 $w_t=1$，正常蒸馏；若是 coordinate token，则由 gate、teacher probability 和 scaling coefficient 共同控制。主实验设置 $\lambda=3$。等价地：

$$
w_t =
\begin{cases}
1, & r_t=0, \\
\lambda p_t, & r_t=1 \text{ and } h_t=1, \\
\alpha \lambda p_t, & r_t=1 \text{ and } h_t=0.
\end{cases}
$$

训练目标是 weighted reverse-KL objective：

$$
\mathcal{L}_{\mathrm{ours}}(\theta)=
\mathbb{E}_{y\sim\pi_\theta(\cdot\mid x,I)}
\left[
\frac{1}{|R(y)|}
\sum_{t\in R(y)}
w_t D_{\mathrm{KL}}\left(P_S^t \parallel P_T^t\right)
\right].
$$

其中，$R(y)$ 是 response-token positions，prompt tokens 不计入 loss。整体设计的作用是：普通 response tokens 正常蒸馏；可靠 coordinate tokens 由 $\lambda p_t$ 加强；不可靠 coordinate tokens 由 $\alpha\lambda p_t$ 降权但不完全丢弃，从而减少 blind imitation（盲目模仿）不可靠 teacher signals，同时保留可能有用的纠错信息。

#### 论文自我评价的缺陷

论文在 Discussion and Limitations 中明确指出：

1. correctness-aware gate 依赖训练阶段的 ground-truth bounding boxes，因此最直接适用于有 spatial annotations 的场景。如果没有框标注，就无法直接进行“当前 coordinate token 是否仍可补全到目标框内”的验证。
2. 当前 reliability criterion 是为 GUI grounding 的 coordinate-token prediction 设计的。对于没有显式 spatial coordinates 的任务，不能直接复用，需要设计其他形式的 verifiable teacher-signal assessment。
3. 未来仍需研究类似 reliability-aware self-distillation 是否能迁移到不同 model scales，以及其他 visually grounded agent tasks。

结合全文还能看到一些隐含限制：实验主要围绕 Qwen3.5-9B 和 GUI-SD / ScaleCUA 风格的数据构造展开；核心超参数 $\alpha=0.5$ 和 $\lambda=3$ 虽有消融，但是否跨模型、跨数据规模稳定仍需进一步验证。

#### 我的分析：优点

1. 问题定义精准：论文抓住了 naive OPSD 在 GUI grounding 中的关键失败模式，即 teacher 在 student-generated wrong prefix 上可能产生“错误但自洽”的 coordinate continuation。
2. 方法利用了 GUI grounding 的结构性优势：ground-truth bounding box 让 teacher-signal reliability 可以被 spatially verified，而不是只依赖 entropy、perplexity、confidence 这类间接 proxy。
3. soft gating 的设计合理：hard gating 会完全丢弃错误前缀后的 teacher signal，但 OPD / OPSD 的价值之一正是让 student 从自己生成的错误状态中学习。降权而非删除是更稳健的折中。
4. gating 与 probability scaling 的互补性通过消融实验被清楚验证：单独使用都不提升，组合后提升明显，这比只报告主结果更有说服力。
5. 实验覆盖六个 GUI grounding benchmarks，并与 base model、GRPO、SFT、Naive-OPSD、GUI-SD 对比，说明该方法不是只针对单一 benchmark 的局部优化。
6. 训练目标仍然简单：本质上只是对 token-level reverse KL 加权，没有引入复杂 rollout 或额外 reward model，工程实现成本相对可控。
7. 论文与 OPD / OPSD 中“teacher signal 不总是可靠”的研究方向高度相关，对 on-policy distillation 的 token-level filtering / calibration 提供了一个清晰实例。

#### 我的分析：缺点 / 风险

1. 方法强依赖 bounding box annotations。对于真实 GUI agent 数据，精确框标注可能昂贵；如果框噪声较大，gating 也可能错误降权有用信号或保留坏信号。
2. reliability 判断只看 teacher 的 top-1 coordinate digit $d_t^\star$ 是否可补全到目标框内，可能忽略 teacher distribution 中其他高概率 digit 的结构。例如 top-1 不兼容但次高概率兼容时，当前 gate 可能过于粗糙。
3. teacher-probability scaling 假设 teacher confidence 与 signal quality 正相关，但 neural model confidence 可能 miscalibrated。尤其在 GUI 中相似元素密集时，高置信错误信号仍可能存在。
4. $\alpha$ 和 $\lambda$ 对结果有明显影响。虽然论文做了 $\lambda$ 消融，但主设置是否对不同 backbone、不同坐标 tokenization、不同数据分布都稳定，还需要更多证据。
5. privileged teacher input 使用高亮框和 textual hint，这与真实推理条件差异较大。虽然这是训练时 teacher-only 设计，但如果 teacher 过度依赖 privileged cue，可能导致 teacher distribution 与 student inference distribution 存在偏移。
6. 实验主要报告 accuracy，缺少更细粒度的 error analysis，例如 failed-gate token 比例、哪些 benchmark 上错误前缀最多、gating 对坐标位数不同位置的影响等。
7. 方法针对 coordinate-token supervision 很自然，但对需要更复杂 action sequence、multi-step GUI navigation 或 non-coordinate reasoning 的 agent task，迁移并不直接。
8. 从 OPD 角度看，本文处理的是“坐标是否还能补全进框”的局部 token reliability，但没有显式分析 teacher signal 的 gradient direction、student uncertainty 或 token-level update 是否真的朝向更优 policy，这些仍是后续研究空间。
