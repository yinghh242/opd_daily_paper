#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy self-distillation。

#### Abstract 原文

> On-policy self-distillation (OPSD) trains a model on its own rollouts and uses a frozen copy to provide dense token-level targets conditioned on a reference target. This works well for LLM reasoning, but a direct extension to multimodal large language models (MLLMs) can create a shortcut: the privileged target may guide tokens mainly based on the text reference target rather than the image. We propose ViGOS, a visually grounded OPSD framework for MLLM post-training. The student first writes a visual description and then reasons toward the final answer. For valid rollouts, an image-only perception teacher supervises the description, while a privileged reasoning teacher supervises the reasoning and final answer on the same student prefix. A reference teacher is used only for invalid rollouts to recover the output format. Across general vision-language, expert reasoning, visual math, spatial grounding, and visual-language-prior benchmarks, ViGOS keeps the main benefits of OPSD and improves image-grounded behavior in shortcut-prone settings.

#### 摘要中文翻译

On-policy self-distillation（OPSD，在线策略自蒸馏）使用模型自身的 rollout（生成轨迹）进行训练，并使用一个 frozen copy（冻结副本）在 reference target（参考目标）条件下提供 dense token-level targets（密集 token 级目标）。这种方法在 LLM reasoning（大语言模型推理）中效果很好，但直接扩展到 multimodal large language models（MLLMs，多模态大语言模型）时可能产生 shortcut（捷径）：privileged target（特权目标）可能主要基于文本 reference target 而非图像来指导 token。本文提出 ViGOS，一个用于 MLLM post-training（后训练）的 visually grounded OPSD（视觉扎根 OPSD）框架。学生模型首先写出 visual description（视觉描述），然后推理得到最终答案。对于 valid rollouts（有效轨迹），image-only perception teacher（仅图像感知教师）监督描述部分，而 privileged reasoning teacher（特权推理教师）在同一个学生前缀上监督推理和最终答案。reference teacher（参考教师）只用于 invalid rollouts（无效轨迹），以恢复输出格式。在 general vision-language、expert reasoning、visual math、spatial grounding 和 visual-language-prior benchmarks 上，ViGOS 保留了 OPSD 的主要收益，并在容易产生 shortcut 的设置中改善了 image-grounded behavior（基于图像的行为）。

#### 研究背景

这篇论文关注的问题是：OPSD 在纯文本 LLM reasoning 中有效，但直接迁移到 MLLM 时，reference answer（参考答案）可能成为比图像更容易利用的监督信号，从而让模型学习“答案兼容但视觉证据不足”的 rationale（推理过程）。

传统 knowledge distillation（知识蒸馏）通常训练学生模型模仿固定 teacher trajectory（教师轨迹）或 reference trajectory（参考轨迹），会带来 train-test gap（训练-测试状态分布差异）：学生训练时没有充分见到自己推理时会产生的状态。OPD（on-policy distillation）通过让学生自己采样 rollout，再由 teacher 对这些学生实际访问的 prefix（前缀状态）提供 dense token-level supervision（密集 token 级监督），缓解这个问题。OPSD 进一步不依赖更强外部教师，而是使用学生模型的 frozen copy，并让该教师看到 reference answer 或 reference solution 来提供监督。

在 MLLM 场景中，问题变复杂了。学生推理时只能看到 image $I$ 和 question $x$，但 OPSD teacher 在训练监督时还能看到 reference answer $a^\star$。由于 MLLM 往往容易受文本信号影响，teacher 可能在学生尚未真正利用图像证据前，就根据 $a^\star$ 推动其生成与答案一致的 reasoning 和 final answer。论文将这种风险概括为答案驱动路径：

$$
a^\star \to r \to a
$$

其中 $r$ 是 reasoning，$a$ 是 final answer。这与 VQA 中的 language prior（语言先验）和 shortcut learning（捷径学习）问题一致：模型可能知道或猜到常见答案，却没有真正根据当前图像判断。

为诊断这一问题，论文提出 PALR（Privileged Answer Leakage Rate，特权答案泄漏率）。PALR 固定学生 rollout 和 prefix，只改变 teacher context：一方面用 wrong-answer counterfactual（错误答案反事实）测量监督信号对 reference answer 的依赖，另一方面用 mismatched image counterfactual（错配图像反事实）测量监督信号对图像的依赖。PALR 越高，说明 dense correction（密集校正）中更大比例来自 privileged answer，而不是图像。作者在 Qwen2.5-VL 的 1,000 样本诊断子集上发现，vanilla OPSD 在 reasoning-answer segment 上有明显答案泄漏：3B 的 $PALR(T_{\mathrm{ra}})$ 为 17.26%，7B 为 26.01%。

因此，本文的核心动机是：保留 OPSD 的 on-policy dense supervision 优点，但控制 privileged answer 进入训练轨迹的位置，避免它直接塑造早期视觉描述。

#### Method

ViGOS（Visual Grounding On-Policy Self-Distillation）的核心思想是 decouple perception and reasoning（解耦感知与推理）：先让学生模型写出视觉描述，再基于该描述和图像进行推理。训练集中每个样本为：

$$
D = \{(I_i, x_i, a_i^\star)\}_{i=1}^{N}
$$

其中 $I_i$ 是图像，$x_i$ 是问题或指令，$a_i^\star$ 是 privileged reference target。学生模型训练和推理时都不能直接看到 $a^\star$。

vanilla OPSD 的 MLLM 目标可以写作：

$$
\mathcal{L}_{\mathrm{OPSD}}
=
\mathbb{E}_{D,p_\theta}
\left[
\sum_{t \in T_y}
D_{\mathrm{KL}}(q_{\mathrm{priv},t} \| p_{\theta,t})
\right]
$$

其中：

$$
q_{\mathrm{priv},t}(\cdot)
=
p_{\bar{\theta}}(\cdot \mid I, x, a^\star, h_t)
$$

这里 $p_\theta$ 是 trainable student，$p_{\bar{\theta}}$ 是 frozen teacher，$h_t=(y_1,\ldots,y_{t-1})$ 是学生自己生成的 prefix。问题在于 $q_{\mathrm{priv},t}$ 对所有 token 都能看到 $a^\star$，因此早期视觉声明也可能被答案条件化。

ViGOS 要求学生生成结构化序列：

$$
y=(d,r,a)
$$

其中 $d$ 是 visual description，$r$ 是 reasoning process，$a$ 是 final answer。输出格式为：

$$
`<description> d </description>  \boxed{a}`
$$

parser（解析器）会提取三个 token 集合：description tokens $T_d$、reasoning tokens $T_r$、answer tokens $T_a$，并定义 $T_{\mathrm{ra}}=T_r \cup T_a$。如果 delimiter 缺失、description 或 reasoning 为空、final answer 无法解析，则 rollout 被标为 invalid：

$$
m_{\mathrm{inv}}(y)=\mathbf{1}[y \notin Y_{\mathrm{valid}}]
$$

ViGOS 使用三个 frozen teacher roles，它们来自同一个初始 MLLM 的冻结副本，但外部上下文不同：

1. image-only perception teacher：只接收图像作为外部证据，用于监督 $d$。
2. privileged reasoning teacher：接收图像、问题和 reference target，用于监督 $r$ 和 $a$。
3. reference teacher：只在 invalid rollout 时启用，用于恢复结构化输出格式。

三个 teacher distribution 为：

$$
q_{\mathrm{img},t}(\cdot)=p_{\bar{\theta}}(\cdot \mid c_{\mathrm{img}}, h_t)
$$

$$
q_{\mathrm{rea},t}(\cdot)=p_{\bar{\theta}}(\cdot \mid c_{\mathrm{rea}}, h_t)
$$

$$
q_{\mathrm{ref},t}(\cdot)=p_{\bar{\theta}}(\cdot \mid c_{\mathrm{ref}}, h_t)
$$

其中 $c_{\mathrm{img}}=I$，$c_{\mathrm{rea}}=(I,x,a^\star,\pi_{\mathrm{rea}})$，$c_{\mathrm{ref}}=(I,x,a^\star,\pi_{\mathrm{out}})$。所谓 image-only 并不是 teacher 不看 prefix，而是它没有额外的问题文本、选项或 reference answer；它仍然在学生已经生成的 $h_t$ 上打分。

对于 valid rollout，perception loss 只作用于 description tokens：

$$
\mathcal{L}_{\mathrm{perc}}
=
\mathbb{E}_{D,p_\theta}
\left[
(1-m_{\mathrm{inv}}(y))
\sum_{t \in T_d}
D_{\mathrm{KL}}(q_{\mathrm{img},t} \| p_{\theta,t})
\right]
$$

reasoning loss 只作用于 reasoning 和 answer tokens：

$$
\mathcal{L}_{\mathrm{rea}}
=
\mathbb{E}_{D,p_\theta}
\left[
(1-m_{\mathrm{inv}}(y))
\sum_{t \in T_r \cup T_a}
D_{\mathrm{KL}}(q_{\mathrm{rea},t} \| p_{\theta,t})
\right]
$$

对于 invalid rollout，ViGOS 不使用不可靠的 segment masks，而使用 reference fallback loss，并采用 reverse KL：

$$
\mathcal{L}_{\mathrm{ref}}
=
\mathbb{E}_{D,p_\theta}
\left[
m_{\mathrm{inv}}(y)
\sum_{t \in T_y}
D_{\mathrm{KL}}(p_{\theta,t} \| q_{\mathrm{ref},t})
\right]
$$

总体目标为：

$$
\mathcal{L}_{\mathrm{ViGOS}}
=
\lambda_{\mathrm{perc}}\mathcal{L}_{\mathrm{perc}}
+
\lambda_{\mathrm{rea}}\mathcal{L}_{\mathrm{rea}}
+
\lambda_{\mathrm{ref}}\mathcal{L}_{\mathrm{ref}}
$$

ViGOS 的关键路径从 vanilla OPSD 的答案先导：

$$
a^\star \to r \to a
$$

变为先视觉描述、后答案引导：

$$
I \to d \to r \to a
$$

这并不是取消 answer-guided reasoning，而是把 reference answer 的作用限制在学生已经生成视觉描述之后。推理时所有 teachers、reference solutions 和 segment masks 都被移除，最终模型只接收图像、问题和输出格式提示。

#### 论文自我评价的缺陷

论文未单列独立的 limitations 章节，但在 Conclusion 和 Appendix caveats 中明确承认了若干限制。

第一，generated description（生成的视觉描述）可能不完整或错误。ViGOS 依赖 $d$ 作为 grounding interface（扎根接口），后续 reasoning teacher 在学生 prefix 上继续监督。如果 $d$ 漏掉关键视觉证据，或错误描述图像内容，后续推理可能会被错误 prefix 锚定。

第二，image-only perception teacher 可能产生 generic descriptions（泛化、空泛描述）。虽然它不接收 question text 或 reference answer，因此可以避免直接答案泄漏，但也可能因为缺少问题上下文而无法突出与当前问题最相关的视觉细节。论文将其看作一种 trade-off：减少 answer leakage，但可能牺牲 task-relevant perception。

第三，训练需要额外 teacher forward passes。相比 vanilla OPSD，ViGOS 对 valid rollout 要分别查询 image-only perception teacher 和 privileged reasoning teacher，对 invalid rollout 还要使用 reference teacher fallback，因此训练成本更高。

第四，PALR 只是 shortcut diagnostic（捷径诊断），不是完整归因方法。论文承认 PALR 使用 wrong-answer counterfactual 和 mismatched-next image counterfactual，只衡量在这种反事实设定下 answer-driven support 与 image-driven support 的相对比例，不能覆盖所有可能 shortcut。

第五，PALR 在报告中把 reasoning 和 final answer 合并为 $T_{\mathrm{ra}}$，而训练 parser 实际区分 $T_r$ 和 $T_a$。这会降低诊断粒度：它能说明 reasoning-answer 区域整体的 answer leakage，但不能精确区分 reasoning token 与 final answer token 各自的问题。

第六，ViGOS 在 $T_d$ 上的 $PALR(T_d)=0.00\%$ 是 by construction（由设计决定）：因为 description segment 的 active teacher 不接收 $a^\star$。这说明直接 privileged-answer path 被切断，但不等于模型完全不存在其他视觉 shortcut 或语言先验 shortcut。

#### 我的分析：优点

1. 问题定义很清楚：论文不是泛泛地说 MLLM 有 shortcut，而是指出 OPSD 在多模态场景下的特殊风险，即 reference answer 进入 dense token-level supervision 后，可能在视觉证据尚未建立前塑造 reasoning。

2. 方法设计简单且针对性强：ViGOS 没有引入复杂外部 reward model 或额外标注，而是通过 segment-wise teacher contexts（分段教师上下文）控制 $a^\star$ 的作用位置。核心改动可以概括为先 $I \to d$，再 $d \to r \to a$。

3. 保留了 OPSD 的主要优势：它仍然是 on-policy，teacher 对学生自己生成的 prefix 打分，而不是训练在固定参考轨迹上；同时保留 dense token-level supervision，比纯 outcome reward 更细。

4. PALR 是一个有用的诊断工具。虽然不是完整因果归因，但它把“答案泄漏”从直觉变成了可度量信号，并且结果与 ViLP shortcut-sensitive evaluation 相互印证。

5. 实验覆盖面较广：主实验包括 general VL、expert/academic reasoning、visual math、spatial grounding 和 vision grounding；同时加入 ViLP 专门测试 visual-language prior conflict，能够更直接验证“是否更依赖图像”。

6. same-prompt comparison 很重要。因为 ViGOS 的 structured prompt 本身可能带来收益，论文额外比较了同样结构化 prompt 下的 Baseline、OPSD 和 ViGOS，说明收益不只是“先描述图像”这个 prompt trick，而是 segment-wise supervision 的作用。

7. ablation 设计比较完整。论文分别验证了 perception loss、reasoning loss 和 reference fallback 的必要性，尤其是 w/o reference teacher 在 ViLP 上明显下降，说明 invalid rollout 的格式恢复不能简单交给 segment teachers 混合处理。

8. 定性案例有解释力。ViLP examples 显示 OPSD 有时已经“看见”关键视觉线索，但最终答案仍回到 prior；这支持论文观点：问题不只是 perception failure，也包括 reasoning / final decision 阶段对视觉证据赋权不足。

#### 我的分析：缺点 / 风险

1. description bottleneck 风险明显。ViGOS 把 $d$ 作为视觉证据接口，但如果 $d$ 错了，后续 reasoning teacher 仍在这个错误 prefix 上监督，可能让模型学会围绕错误描述进行更流畅的推理。

2. image-only teacher 缺少问题上下文，可能监督出不够 task-relevant 的描述。很多视觉问答不需要完整图像描述，而需要识别与问题高度相关的区域、文本或关系；完全去掉 question text 可能导致 description 冗长但不聚焦。

3. privileged answer leakage 并没有完全消失。ViGOS 只是推迟 $a^\star$ 的进入位置，reasoning-answer segment 仍由 privileged reasoning teacher 监督；PALR 降低但不为零。因此如果 $d$ 很弱，$a^\star$ 仍可能主导后续 reasoning。

4. 输出更长，推理成本更高。ViGOS 推理时要求生成 description、reasoning 和 final answer，虽然不需要 teachers，但会增加生成长度、延迟和 token 成本；对于生产系统未必总是可接受。

5. 依赖 parser 和格式稳定性。方法假设可以可靠解析 `<description>`、`` 和 `\boxed{}`。一旦模型格式漂移，训练就进入 fallback；如果某些任务天然不适合这种格式，方法可能不稳定。

6. 训练成本比 vanilla OPSD 高。多个 teacher roles 意味着额外 forward passes，尤其在大规模 MLLM post-training 中成本显著。论文报告使用 8$\times$A100，对资源较少的实验环境不友好。

7. 实验主要围绕 Qwen2.5-VL 3B/7B 和 Vision-SR1-47K。虽然结果较强，但还不能充分证明该方法对其他 MLLM 架构、不同数据分布、更大模型或真实用户任务同样有效。

8. PALR 的 wrong-answer counterfactual 依赖 Gemini 生成错误答案并过滤，这引入了额外模型和构造策略。不同 wrong-answer quality、数量或 image counterfactual 选择可能影响 PALR 结论。

9. Avg@5 在部分任务上仍有下降或不稳定。论文自己也指出，ViGOS 支持 image-grounded multimodal reasoning，但 harder symbolic reasoning 和 response stability 可能仍需要额外优化。

10. 该方法可能鼓励“看起来很扎根”的解释。由于训练显式要求先写 description，模型可能生成更有视觉细节的 rationale，但这不必然保证所有细节都真实来自图像；如果缺少更强的视觉事实验证，仍可能出现 grounded-sounding hallucination。

#### 与 OPD / OPSD 的相关性依据

标题或摘要中明确出现强相关短语：on-policy self-distillation。

#### Abstract 原文

> Graphical user interface (GUI) grounding requires vision-language models (VLMs) to identify small target elements in high-resolution screenshots and predict precise screen coordinates. On-policy self-distillation (OPSD) is a promising post-training approach for this coordinate-sensitive task, since it provides dense token-level teacher signals beyond hard coordinate labels. However, naive OPSD is not well suited to GUI grounding: OPSD evaluates the teacher on student-generated prefixes, the quality of coordinate-token teacher signals can degrade when the prefix has already deviated from the target coordinate, leading to unreliable teacher signal. To mitigate this, We propose quality-aware self-distillation for VLM-based GUI grounding, which improves coordinate-token teacher-signal quality through soft correctness-aware gating and teacher-probability scaling. The soft correctness-aware gate checks whether the teacher's current coordinate-token prediction can still be completed into the ground-truth box under the student-generated prefix. If not, the corresponding teacher signal is down-weighted. Teacher-probability scaling then uses the teacher's confidence as a lightweight factor to further calibrate the strength of the gated supervision. A key empirical finding is that neither component alone improves overall performance, whereas combining them consistently improves performance. This suggests that the two mechanisms play complementary roles: correctness-aware gating suppresses unreliable coordinate-token supervision, while teacher-probability scaling calibrates the strength of the remaining signals. Experiments across six GUI grounding benchmarks show that our method consistently improves the base model and outperforms strong baselines.

#### 摘要中文翻译

Graphical user interface (GUI) grounding（图形用户界面定位）要求 vision-language models (VLMs) 在高分辨率截图中识别很小的目标元素，并预测精确的屏幕坐标。On-policy self-distillation (OPSD) 是一种很有前景的 post-training（后训练）方法，适合这种对坐标敏感的任务，因为它能提供超越硬坐标标签的密集 token-level teacher signals（逐 token 教师信号）。然而，朴素的 OPSD 并不适合 GUI grounding：OPSD 会在 student-generated prefixes（学生生成的前缀）上评估 teacher；当该前缀已经偏离目标坐标时，coordinate-token teacher signals（坐标 token 教师信号）的质量可能下降，从而产生不可靠的 teacher signal。

为缓解这一问题，本文提出用于 VLM-based GUI grounding 的 quality-aware self-distillation（质量感知自蒸馏），通过 soft correctness-aware gating（软正确性感知门控）和 teacher-probability scaling（教师概率缩放）提升 coordinate-token teacher-signal quality。soft correctness-aware gate 会检查：在当前 student-generated prefix 下，teacher 当前预测的 coordinate token 是否仍然可以被补全为 ground-truth box（真实框）内的坐标。如果不能，相应的 teacher signal 会被降权。teacher-probability scaling 随后使用 teacher 的 confidence（置信度）作为轻量因子，进一步校准经过 gating 后的监督强度。一个关键经验发现是：单独使用任一组件都不能提升整体性能，而将二者结合会稳定提升性能。这说明两种机制具有互补作用：correctness-aware gating 抑制不可靠的 coordinate-token supervision，而 teacher-probability scaling 校准剩余信号的强度。在六个 GUI grounding benchmarks 上的实验表明，该方法稳定改进 base model，并优于强 baselines。

#### 研究背景

GUI grounding 是让 VLMs 和 GUI agents 操作电脑、移动设备、网页应用的基础能力：给定 screenshot 和 instruction，模型需要定位目标界面元素并输出屏幕坐标。困难在于高分辨率 GUI 场景中目标元素可能很小、外观相似、排列密集，坐标预测又是高度细粒度的任务。

已有 post-training 方法各有不足。SFT 简单稳定，但把标注坐标当作 hard target（硬标签），几乎只提供最终答案，无法利用 teacher 对其他可能坐标 token 的 uncertainty（不确定性）或 dark knowledge（暗知识）。GRPO 等 reinforcement learning 方法可以优化任务结果，但需要多次 rollouts，并依赖 sparse rewards（稀疏奖励），对精细坐标定位的 token-level guidance 较弱且成本较高。

OPSD 的优势是让 student 在自己生成的 trajectories 上学习 teacher distributions，可以提供 dense token-level supervision，并更贴近 autoregressive inference（自回归推理）时模型使用自身前缀的场景，从而缓解 exposure bias（暴露偏差）。但 GUI grounding 中的坐标是逐 token 生成的：如果 student-generated prefix 已经进入错误的空间假设，teacher 在该错误前缀上的后续 logits 可能只是“合理延续错误坐标”，而不是把 student 拉回真实目标区域的可靠监督。因此，naive OPSD 在 GUI grounding 中会遇到 teacher-signal reliability（教师信号可靠性）问题。

相关工作已经认识到 teacher signals 并非同等可信。例如 entropy-based self-distillation、perplexity-based OPD、基于 outcome correctness 的 routing 或 weighting、Vision-OPD、GUI-SD 等，都试图通过 uncertainty、teacher probability、perplexity 或 privileged teacher input 改善信号质量。但这些方法大多使用间接 proxy，不能直接保证被强调的 teacher signal 在空间上确实可靠。本文的关键观察是：GUI grounding 具有 spatial verifiability（空间可验证性），因为 coordinate prediction 可以直接与 ground-truth bounding box 对齐检查。因此，本文用 ground-truth box 作为训练时的直接可靠性标准，对 coordinate-token teacher signals 进行质量感知校准。

#### Method

本文方法是 quality-aware self-distillation for GUI grounding，核心是只对 coordinate tokens 调整 distillation loss 的权重；formatting tokens 或其他 non-coordinate text tokens 仍正常蒸馏。这样可以集中处理真正决定 GUI grounding accuracy 的坐标 token。

首先，训练样本包含 GUI instruction $x$、screenshot $I$ 和 ground-truth bounding box $B$。student 只接收原始输入 $(x, I)$。teacher 在训练时接收 privileged input $(x^+, I^+)$：其中 $I^+$ 通过 layout-preserving regional masking 构造，目标区域被保留和高亮，任务无关区域被抑制；teacher prompt 还会提示答案位于绿色矩形中。推理时不使用 privileged information，只有 student 和原始输入。

附录进一步说明了 visual privileged information 的构造。给定原始截图 $I$ 和目标框 $b$，令 $d_b(u,v)$ 表示像素 $(u,v)$ 到 bounding box $b$ 的欧氏距离，框内像素距离为 $0$。Gaussian soft mask 写作：

$$
\alpha(u,v)=\exp\left(-\frac{d_b(u,v)^2}{2\sigma^2}\right),
\quad
I_{\mathrm{mask}}(u,v)=\alpha(u,v)I(u,v).
$$

然后在 target region 周围画绿色矩形，并附加 textual hint。student 始终只看原始 GUI screenshot 和原始 instruction。

在 self-distillation 中，student 先采样 on-policy response $y$：

$$
y \sim \pi_\theta(\cdot \mid x, I).
$$

student 和 teacher 都在同一个 student-generated prefix $y_{<t}$ 上评估分布：

$$
P_S^t = \pi_\theta(\cdot \mid x, I, y_{<t}),
$$

$$
P_T^t = \mathrm{sg}\left[\pi_\theta(\cdot \mid x^+, I^+, y_{<t})\right].
$$

其中 $\mathrm{sg}[\cdot]$ 表示 stop-gradient，teacher distribution 只作为 teacher signal 使用。附录还说明训练使用 EMA teacher：teacher 从 legacy model 初始化，并在每次成功 optimizer step 后更新：

$$
\theta_{\mathrm{teacher}} \leftarrow 0.95\theta_{\mathrm{teacher}} + 0.05\theta_{\mathrm{student}}.
$$

soft correctness-aware gating 的目标是判断当前 teacher 的 top coordinate-token prediction 在空间上是否仍可行。令 $D$ 是 coordinate digit tokens 的集合，teacher 在位置 $t$ 的 top coordinate digit token 为：

$$
d_t^{\star}=\arg\max_{d \in D} P_T^t(d).
$$

对 coordinate-token position $t$，令 $a(t)\in\{x,y\}$ 表示其对应坐标轴，$B_{a(t)}$ 表示 ground-truth box 在该轴上的区间。方法会把 $d_t^{\star}$ 接到当前 axis-specific prefix 后，检查剩余 digits 是否仍能补全出落在 $B_{a(t)}$ 内的合法坐标。于是定义 compatibility indicator：

$$
h_t =
\begin{cases}
1, & \text{if such a completion exists},\\
0, & \text{otherwise}.
\end{cases}
$$

如果 teacher 当前 coordinate token 与目标框仍兼容，$h_t=1$；否则 $h_t=0$。但本文不采用 hard gating 直接丢弃失败信号，而是使用 soft correctness-aware gate：

$$
g_t=\alpha+(1-\alpha)h_t.
$$

兼容信号得到 $g_t=1$；不兼容信号得到 $g_t=\alpha$。主实验设置 $\alpha=0.5$，即 failed-gate coordinate-token signals 被降权一半，而非完全丢弃。

teacher-probability scaling 进一步利用 teacher confidence。定义 teacher 对 top coordinate token 的概率：

$$
p_t=P_T^t(d_t^{\star}).
$$

$p_t$ 越大，说明 teacher 对当前 coordinate token 的偏好越清晰，distillation term 应更强；$p_t$ 越小，则说明 teacher 更不确定，即使通过空间兼容性检查，也应降低监督强度。

最终的 token-level distillation weight 为：

$$
w_t=(1-r_t)+r_t g_t \lambda p_t.
$$

其中 $r_t$ 表示位置 $t$ 是否为 coordinate token。若 $r_t=0$，则 $w_t=1$，普通 response token 正常蒸馏；若 $r_t=1$，则 coordinate-token loss 由 $g_t$、$p_t$ 和全局系数 $\lambda$ 控制。主实验设置 $\lambda=3$，用于避免 reliability-based weighting 后 coordinate tokens 的总 loss mass 过低。

等价地：

$$
w_t =
\begin{cases}
1, & r_t=0,\\
\lambda p_t, & r_t=1 \text{ and } h_t=1,\\
\alpha\lambda p_t, & r_t=1 \text{ and } h_t=0.
\end{cases}
$$

最终训练目标是 weighted reverse-KL objective：

$$
\mathcal{L}_{\mathrm{ours}}(\theta)
=
\mathbb{E}_{y\sim\pi_\theta(\cdot\mid x,I)}
\left[
\frac{1}{|R(y)|}
\sum_{t\in R(y)}
w_t
D_{\mathrm{KL}}\left(P_S^t \parallel P_T^t\right)
\right].
$$

其中 $R(y)$ 是 response-token positions，prompt tokens 不参与 loss。训练目标格式是结构化 tool call，例如输出 left click 的 coordinate，而不是自然语言推理过程。

#### 论文自我评价的缺陷

论文在 Discussion and Limitations 中明确给出局限：

第一，correctness-aware gate 依赖训练时的 ground-truth bounding boxes，因此最直接适用于有空间标注的 GUI grounding 场景。如果训练数据没有 bounding box 或可验证坐标区间，就难以直接使用该 gating 标准。

第二，当前 reliability criterion 专门为 GUI grounding 的 coordinate-token prediction 设计。它利用了坐标 token 可以根据当前 prefix 检查是否仍能补全到真实目标框内这一结构。对于没有显式空间坐标的任务，或输出不是坐标序列的 visually grounded agent tasks，需要设计其他形式的 verifiable teacher-signal assessment。

第三，论文指出未来还需要研究类似 reliability-aware self-distillation 策略是否能迁移到不同 model scales 和其他 visually grounded agent tasks。也就是说，当前实验证据主要集中在 Qwen3.5-9B 和 GUI grounding benchmarks 上，跨模型规模、跨任务的泛化仍未充分验证。

#### 我的分析：优点

本文的最大优点是问题定义很准确：它没有泛泛地说 teacher confidence 更高就更可信，而是指出 OPSD 在 GUI grounding 中的具体失效模式：teacher 被迫在 student-generated prefix 上给出后续 token 分布，而错误坐标前缀会使 teacher logits 变成“错误空间假设下的合理延续”。这个分析非常贴合 autoregressive coordinate generation 的特点。

第二，方法利用了 GUI grounding 的 spatial verifiability，把 teacher-signal reliability 从 indirect proxy 推进到 direct verification。相较于 entropy、perplexity、teacher probability 等间接指标，检查 coordinate token 是否仍能补全到 ground-truth box 内更直接，也更符合任务结构。

第三，soft gating 的设计比 hard filtering 更合理。完全丢弃 failed-gate signals 会损失 teacher 在错误 student states 上可能提供的 corrective guidance；而 $\alpha=0.5$ 的 soft down-weighting 在“避免强化错误坐标”和“保留潜在有用分布信息”之间做了折中。Table 3 中 soft gating 的 Avg 72.23 高于 hard gating 的 71.46，也支持这一判断。

第四，本文强调 gating 和 probability scaling 的互补性，并通过 ablation 验证。单独 gating 可能压低仍有用的 teacher signal；单独 probability scaling 可能放大空间上不可靠的 teacher signal。二者组合后，先用 correctness-aware gating 限制错误信号，再用 confidence scaling 校准强度，逻辑上比较闭环。

第五，方法只改变训练时 coordinate-token KL loss 的权重，不增加推理成本。teacher 的 privileged visual cue、ground-truth box、EMA teacher 都只在训练时使用，推理时 student 仍只接收原始 screenshot 和 instruction，这对实际部署友好。

第六，实验覆盖六个 GUI grounding benchmarks，并且包含主结果、高层组件分析、gating strength、teacher-probability scaling、$\lambda$ 系数等多组 ablation。尤其是“单组件无效、组合有效”的现象比较关键，说明提升不只是来自额外 teacher information，而是来自 teacher-signal quality calibration 的组合机制。

#### 我的分析：缺点 / 风险

第一，方法强依赖 ground-truth bounding boxes。对于真实 GUI agent 数据，很多交互轨迹可能只有点击点、最终成功信号或弱标注，而没有完整 target bounding box。如果没有高质量 box annotation，correctness-aware gating 的直接适用性会下降。

第二，可靠性判断只看 teacher top-1 coordinate digit token 是否可补全到目标区间，而不是评估完整 teacher distribution。若 teacher 的 top-1 token 不可行但分布中仍有较大概率质量落在可行 token 上，当前 gating 可能过度惩罚；反过来，top-1 token 可行也不代表整个 distribution 高质量。

第三，teacher-probability scaling 默认 teacher confidence 与 signal quality 正相关，但 VLMs 的 probability calibration 并不一定可靠。尤其在高分辨率 GUI、相似元素密集、遮挡或 OCR 不稳定时，teacher 可能对错误局部模式过度自信，$p_t$ 仍可能放大错误信号。论文用 gating 缓解了这一点，但没有彻底解决 calibration risk。

第四，超参数敏感性仍存在。主方法依赖 $\alpha=0.5$ 和 $\lambda=3$，Table 5 显示 $\lambda=4$ 已经使 Avg 从 72.23 降到 71.80。不同 backbone、tokenizer、坐标格式、benchmark 分布下，最佳 $\alpha$ 和 $\lambda$ 可能变化，需要重新调参。

第五，实验主要是单一 backbone Qwen3.5-9B，且没有报告多随机种子、置信区间或显著性检验。Avg 提升 2.16 points 相对 GUI-SD 是有意义的，但不同 benchmark 上的方差、训练随机性和数据构造影响仍不清楚。

第六，训练成本可能不低。虽然推理无额外成本，但训练需要 privileged teacher input、full-logit distillation、EMA teacher、full vocabulary KL，以及 8 张 H800。论文强调比 GRPO 的多 rollout 和 sparse reward 更适合，但没有系统给出 wall-clock、显存、吞吐或总训练成本对比。

第七，方法绑定 coordinate-token 生成范式。如果 GUI agent 改用 continuous coordinate regression、heatmap prediction、region proposal、DOM element selection 或 action sequence planning，当前 digit-token prefix completion 的 gating 逻辑需要重写，不能直接迁移。

第八，privileged teacher input 使用绿色框和 hint，本质上让 teacher 在训练时获得非常强的目标定位信息。虽然 student 推理时不使用该信息，但 teacher signal 的质量和形态可能与真实推理分布差异较大；如果 privileged cue 过强，teacher distribution 可能过于依赖标注区域，未必能完全反映 student 在无提示场景下应学习的视觉判别过程。
